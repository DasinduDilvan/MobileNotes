🍪 PHP Cookies

🎯 Learning Objectives

After the successful completion of this lecture, students should be able to:

  • Identify what cookies are in PHP
  • Use cookies in web development

📌 What is an HTTP Cookie?

A cookie is a small file stored on the client's hard drive which contains data that the server has asked to store. Cookies introduce state into the otherwise stateless HTTP protocol.
  • Cookies are transferred between the server and the client according to HTTP
  • PHP fully supports HTTP cookies
  • A cookie is often used to identify a user
  • Each time the same computer requests a page, it will send its stored cookies along with the request
  • With PHP, you can both create and retrieve cookie values
⚠️ Cookies can be removed by the user at any time — never use them to store anything critically important.

🧱 Cookie Structure

Name–Value Pairs Cookies store data as key-value pairs (e.g., name=John).
Expiration Date Each cookie comes with an expiry. Expired cookies are no longer accessed by the browser.
Server-Side Management Cookies can be managed by server-side scripts (e.g., PHP).
Automatic Submission Relevant cookies are automatically sent from the client to the server with every HTTP request.
$_COOKIE Superglobal PHP stores all received cookie information in the $_COOKIE superglobal array.

⚙️ How Cookies Work

1 Server sends cookies: A server-side script sends a set of cookies to the browser in the HTTP response header.
2 Browser stores them: The browser stores this information on the local machine for future use.
3 Cookie is sent back: On the next request to the same web server, the browser automatically sends those cookies back to the server.
4 Server identifies the user: The server uses the cookie information to identify the user and personalize the response.

📂 Types of Cookies

🔄 Session Cookie (Transient Cookie)
  • Erased when the web browser is closed
  • Stored in temporary memory — not retained after closing
  • Does not collect information from your computer
  • Typically stores information in the form of a session identification
💾 Persistent Cookie (Permanent / Stored Cookie)
  • Stored on your hard drive until it expires or is deleted
  • Set with expiration dates
  • Used to collect identifying information about the user
  • Examples: web surfing behavior, user preferences for a specific website

🛠️ Uses of Cookies

1. Session Management
Cookies are widely used to manage user sessions. For example, when using an online shopping cart, items are added across multiple pages. This is maintained using cookies — all items remain in the cart until checkout.
2. User Identification
Once a user visits a webpage, cookies can "remember" them. On subsequent visits, content relevant to the user's search/visit patterns is served. This is the basis of Retargeting — a concept used in online marketing where advertisements for products the user is likely to buy are shown based on their browsing history.
3. Tracking / Analytics
Cookies are used to track users and gather analytical data such as:
  • Number of visits per user
  • Time of first visit, previous visit, and current visit
  • Technologies used (e.g., browser, operating system)
  • Time spent on various pages

✏️ Creating Cookies in PHP

PHP provides the setcookie() function to set a cookie. This function must be called before the <html> tag. For each cookie, the function has to be called separately. 
setcookie(name, value, expire, path, domain, security);

Parameters of setcookie()

Parameter Description Status
name The name of the cookie. Required
value The value (content) you actually want to store in the cookie. Required
expire A future time in seconds (e.g., time()+3600 for 1 hour). If not set, the cookie expires when the browser closes. Optional
path The server path of the cookie. A single / makes it valid for all directories. Optional
domain Specifies the domain name. Must contain at least two periods to be valid. Cookies are only valid for the host/domain that created them. Optional
security Set to 1 (TRUE) to only transmit over secure HTTPS. Default is 0 (FALSE). Optional

Example: Setting Cookies

<?php
setcookie("name", "John Watson", time()+3600, "/", "", 0);
setcookie("age",  "36",          time()+3600, "/", "", 0);
?>
<html>
<head>
    <title>Setting Cookies with PHP</title>
</head>
<body>
    <?php echo "Set Cookies"; ?>
</body>
</html>

Example: Using Variables for Cookie Parameters

<?php
$cookiename   = "test";
$cookievalue  = "test value";
$cookieExpire = time() + 3600;  // after an hour
$cookieDomain = "127.0.0.1";

setcookie($cookiename, $cookievalue, $cookieExpire, "/", $cookieDomain, 0);
?>
<html>
<body>
    <?php echo "Set Cookies"; ?>
</body>
</html>

📥 Retrieving Cookies in PHP

⚠️ Important: When a browser accepts a cookie, you cannot extract its value until the next HTTP request is made. If you set a cookie on page 1, you can only retrieve it starting from page 2 onwards.

Use the $_COOKIE superglobal array with the cookie name as the key:

Read a Single Cookie

<?php
echo $_COOKIE["name"] . "<br />";
?>

Retrieve All Cookies

Iterate through the $_COOKIE array to list all stored cookies:

<?php
echo "<p>There are " . count($_COOKIE) . " cookies stored for this domain.</p>";

for ($i=0; $i < count($_COOKIE); $i++) {
    echo "Cookie name: " . key($_COOKIE) . " Cookie value: " . current($_COOKIE) . "<br>";
    next($_COOKIE);
}
?>

Check if a Cookie Exists — isset()

Use isset() to check whether a cookie has been set before accessing it:

<?php
if (isset($_COOKIE["name"])) {
    echo "Welcome " . $_COOKIE["name"] . "<br />";
} else {
    echo "Sorry... Not recognized" . "<br />";
}
?>

🔄 Altering & Deleting Cookies

Alter a Cookie

To change a cookie's value, simply call setcookie() again with the same name but new parameters:

// Old cookie:
// setcookie("test", "teststring!", time()+3600, "/examples", "localhost", 1);

// Updated cookie (new value, extended expiry):
setcookie("test", "anothervalue", time() + (3600 * 24 * 365), "/examples", "localhost", 1);

Delete a Cookie

To destroy a cookie, call setcookie() again but set the expiration time to a time in the past:

<?php
setcookie("name", "", time() - 60, "/", "", 0);
setcookie("age",  "", time() - 60, "/", "", 0);
?>

⚖️ Cookies vs. Sessions

🍪 Cookies
  • Stored in the user's browser
  • Can keep information until manually deleted or expired
  • Can store login details so users don't re-login on every visit
  • Users can block or delete cookies at any time
  • If cookies are blocked, cookie-dependent features (e.g., shopping cart) won't work
📋 Sessions
  • Stored server-side (not in the browser)
  • Session data is lost when the browser is closed
  • Cannot persist login state across browser sessions
  • PHP passes session data from one page to another temporarily
  • Session info is usually deleted shortly after the user leaves the site
Summary: Use cookies for long-term storage of user information on the client's computer. Use sessions for temporary, secure data passing between pages during a single visit.

📝 Summary

  • A cookie is a small file stored on the client's computer by the web server to maintain state in HTTP
  • Cookies contain name-value pairs and come with expiration dates
  • Two types: Session cookies (temporary, lost on browser close) and Persistent cookies (stored on disk until expiry/deletion)
  • Common uses: session management, user identification, and tracking/analytics
  • PHP uses setcookie() to create and modify cookies — must be called before any HTML output
  • PHP reads cookies via the $_COOKIE superglobal array
  • Use isset() to safely check if a cookie exists before accessing it
  • Delete a cookie by setting its expiration date to the past using time() - 60
  • Sessions pass data between pages temporarily; cookies store data on the user's computer for later visits
