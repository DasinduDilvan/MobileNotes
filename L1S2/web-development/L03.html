⚡ Arrow Functions (ES6 Feature)

What is an Arrow Function?
  • Introduced in ES6 (2015)
  • Provides a shorter syntax for writing functions
  • Often used for callbacks and concise one-liner functions
  • Also known as "fat arrow functions" because of the => symbol
Basic Syntax
const functionName = (parameters) => expression;
  • parameters → zero, one, or multiple parameters
  • expression → automatically returned (if single-line)
  • const is recommended for the variable

Syntax Variations

No Parameters 
const greet = () => "Hello!";
console.log(greet()); // Hello!
Single Parameter 
const square = x => x * x;
console.log(square(4)); // 16
Multiple Parameters 
const add = (a, b) => a + b;
console.log(add(2, 3)); // 5

Multi-line Body (with { } and return)

const multiply = (a, b) => {
    const result = a * b;
    return result;
};
console.log(multiply(2, 4)); // 8
💡 Key Rule: If the body is a single expression, you can omit { } and return — the value is returned automatically. For multi-line logic, curly braces and an explicit return are required.

🔄 Asynchronous JavaScript

Direct vs Indirect Function Execution

Direct Execution (Synchronous)

Functions are called one after another and executed in order. Each function must complete before the next one starts.

function one()   { console.log("one"); }
function two()   { console.log("two"); }
function three() { console.log("three"); }

one();
three();
two();
// Output: one, three, two
Indirect Execution (Asynchronous)

Functions can go outside the sequence and do not depend on the strict order of execution. Associated with asynchronous programming.

☕ Real-World Analogy: Ordering Coffee at a Café

  • You order a coffee → Starting an async task
  • While coffee is being prepared, you talk to a friend → Other JS code continues running
  • The barista calls your name → Callback / event fires
  • You collect your coffee → Async task completes

📞 Callback Functions

Definition: A callback function is a function passed as an argument to another function, which is then executed later — usually after a task is completed.

Basic Example

function displayMessage() {
    console.log("Task completed!");
}

function performTask(callback) {
    callback(); // calling the callback function
}

performTask(displayMessage);

Real-World Callback Example (Food Order)

function order(food, id, callback) {
    console.log(`Order received for the ${food}`);
    // simulating preparation...
    callback(id); // order is ready, call the customer
}

function callCustomer(orderNum) {
    console.log(`Food for order #${orderNum} is ready`);
}

// New customer approached
order("pasta", "987", callCustomer);

⏱️ setTimeout() in JavaScript

What is setTimeout? A built-in JavaScript function that executes a callback after a specified delay (in milliseconds).
Basic Syntax: setTimeout(function, delay);
  • function → the callback to execute after the delay
  • delay → time in milliseconds (1000 ms = 1 second)

Simple Example

setTimeout(function () {
    alert("Hello!");
}, 2000); // Displays an alert after 2 seconds

setTimeout with Out-of-Order Execution

function one()   { console.log("one"); }
function two()   { console.log("two"); }
function three() { console.log("three"); }

one();
setTimeout(three, 2000); // three executes after 2 seconds
two();

// Output order: one → two → (2s later) → three
💡 Note: two() runs before three() even though it was called after — because three is delayed by setTimeout. This is the essence of non-blocking asynchronous behavior.

📦 Objects in JavaScript

What is an Object?
  • An object is a collection of key-value pairs
  • Keys are called properties
  • Values can be: primitive types (string, number, boolean), arrays, functions (called methods), or other objects

Object Creation — 3 Ways

1. Using new Object() Constructor 
const student = new Object();
student.name = "Alice";
student.age = 20;
student.program = "CS";
2. Using Object Literal 
const student = {
    name: "Alice",
    age: 20,
    program: "CS"
};
3. ES6 Shorthand
(when variable name = property name) 
const name = "Alice";
const age = 20;

const student = { name, age };

🔍 Accessing Object Properties

Dot Notation
  • Most common and readable
  • Property name must be a valid identifier
  • Preferred when property name is known in advance
console.log(student.name);    // Alice
console.log(student.age);     // 20
console.log(student.program); // CS
Bracket Notation
  • Property name written as a string
  • Useful when property name contains spaces or is dynamic
console.log(student["name"]); // Alice
console.log(student["age"]);  // 20

Dynamic Property Access

const key = "program";
console.log(student[key]); // Computer Science
// JS first evaluates `key`, then accesses student["program"]
⚠️ When Dot Notation Fails — If a property key contains spaces, dot notation is invalid. Use bracket notation instead. 
const person = { "full name": "Alice Smith" };

console.log(person["full name"]); // Alice Smith ✅
// person.full name  ❌ invalid

✏️ Adding, Updating, and Deleting Properties

Adding a Property

JavaScript allows properties to be added at runtime. Objects are dynamic.

student.program = "CS";
student.year = 2;
Updating a Property

Assign a new value to an existing key. No special syntax required.

student.age = 21;
console.log(student.age); // 21
Deleting a Property

delete removes the property completely. Accessing it afterwards returns undefined.

delete student.year;

Checking Property Existence — in Operator

console.log("program" in student); // true
console.log("year"    in student); // false (was deleted)

🛠️ Functions Inside Objects — Methods

Definition: When a function is stored as a value inside an object, it is called a method.
Standard Method Syntax 
const person = {
    name: "Alice",
    age: 25,
    greet: function() {
        console.log("Hello, my name is " + this.name);
    }
};

person.greet(); // Hello, my name is Alice
ES6 Shorthand Method Syntax (omit the function keyword) 
const person = {
    name: "Bob",
    age: 30,
    greet() {
        console.log(`Hi, I'm ${this.name} and I'm ${this.age} years old`);
    }
};

person.greet(); // Hi, I'm Bob and I'm 30 years old
this refers to the object itself. In the example above, this.name refers to the name property of the same object.

🔑 The this Keyword

What is this?
  • this is a special keyword in JavaScript
  • It refers to the object that is currently "calling" the function
  • The value of this depends on how the function is called

this Inside an Object Method

const student = {
    name: "Alice",
    greet: function() {
        console.log("Hello, " + this.name);
    }
};

student.greet(); // Hello, Alice
greet() is called by
the object student
this points to
the object student
Therefore this.name
equals "Alice"

🏭 Factory Functions

Definition: A factory function is a regular function that creates and returns an object. The object is created at the return { ... } statement inside the function.
General Pattern 
function createObject(param1, param2) {
    return {
        key1: param1,
        key2: param2
    };
}
Practical Example 
function createStudent(name, age) {
    return {
        name,
        age,
        greet() {
            return `Hi, I am ${name}`;
        }
    };
}

Creating and Using Objects via Factory Function

const s1 = createStudent("Alice", 20);

console.log(s1.name);    // Alice
console.log(s1.age);     // 20
console.log(s1.greet()); // Hi, I am Alice
createStudent("Alice", 20)
↓
New Object Created
↓
{ name: "Alice", age: 20, greet() }
↓
Assigned to s1

📝 Practice Questions — Arrow Functions

Q1: checkGrade — Arrow Function with Conditional

Write an arrow function named checkGrade that accepts one parameter marks and returns "Pass" if marks ≥ 40, otherwise "Fail". Use a multi-line arrow function with { } and return.

const checkGrade = (marks) => {
    if (marks >= 40) {
        return "Pass";
    } else {
        return "Fail";
    }
};

// Sample calls
alert("Marks: 50 → " + checkGrade(50)); // Pass
alert("Marks: 35 → " + checkGrade(35)); // Fail
alert("Marks: 80 → " + checkGrade(80)); // Pass

Q2: countDown — Arrow Function with while Loop

Write an arrow function named countDown that accepts parameter n and uses a while loop to print numbers from n down to 1 using document.write().

const countDown = (n) => {
    while (n >= 1) {
        document.write(n + "<br>"); // write number with line break
        n--;                         // decrement
    }
};

// Call the function
countDown(5);

📝 Practice Questions — Objects

Q1: Student Record System (Factory Function)

Write a factory function createStudent accepting name, rollNumber, and department. Return an object with those properties and a displayInfo() method. Create two student objects and call displayInfo() on each.

function createStudent(name, rollNumber, department) {
    return {
        name,
        rollNumber,
        department,
        displayInfo() {
            return `Name: ${this.name}, Roll No: ${this.rollNumber}, Department: ${this.department}`;
        }
    };
}

// Creating student objects
const student1 = createStudent("Alice", 101, "Computer Science");
const student2 = createStudent("Bob",   102, "Information Technology");

console.log(student1.displayInfo());
console.log(student2.displayInfo());
  • name, rollNumber, department → are properties of the object being returned
  • this refers to the object itself when displayInfo() is called
  • This ensures the method always refers to the object's own properties, not the outer function parameters

Q2: Employee Salary System

Create an employee object, write a calculateNetSalary function (adds 20% allowance), update the basic salary, and display all details with a for...in loop.

// Employee object
const employee = {
    name: "John",
    id: 5001,
    basicSalary: 30000
};

// Function to calculate net salary
function calculateNetSalary(emp) {
    const allowance = emp.basicSalary * 0.20;
    emp.netSalary = emp.basicSalary + allowance; // adds new property
}

calculateNetSalary(employee);

// Updating basic salary directly
employee.basicSalary = employee.basicSalary + 5000;

// Displaying employee details using for...in
for (let key in employee) {
    console.log(key + ": " + employee[key]);
}
