📂 PHP File Handling

Lecture 6 · ICT1232

🎯 Learning Objectives

After completing this lecture, you should be able to:

  • Identify all types of file manipulations in PHP
  • Handle files using PHP scripts — create, open, read, write, append, and delete

💡 Introduction

PHP can create, open, read, write, and delete files stored on the server. File operations are grouped as:

🔧 Basic Operations Create · Open · Close
📋 Other Operations Read · Write · Append · Truncate · Upload · Delete
⚠️ Important Warning: When manipulating files you must be very careful — doing something wrong can cause a lot of damage (overwriting or deleting important data permanently).

Typical File Operation Flow:

1. Open / Create
fopen()
→
2. Read / Write
fread() / fwrite()
→
3. Close
fclose()

🔓 fopen() Modes — Types of Access

When opening a file with fopen(), you must specify a mode that defines how the file will be accessed.

ModeAccess TypeFile Exists?Pointer Starts At
r Read only Must exist Beginning
w Write only — erases existing content Creates if not exists Beginning
a Write only — preserves existing content Creates if not exists End (appends)
x Write only — new file only Returns FALSE if exists Beginning
r+ Read & Write Must exist Beginning
w+ Read & Write — erases existing content Creates if not exists Beginning
a+ Read & Write — preserves existing content Creates if not exists End (appends)
x+ Read & Write — new file only Returns FALSE if exists Beginning
Quick Memory Aid:
r = read  |  w = write (wipes)  |  a = append (adds to end)  |  x = exclusive new file
Add + to any mode to also allow the opposite operation (e.g., r+ = read AND write).

📄 Create & Open a File — fopen()

fopen( filename, mode )
Binds a named file to a stream (opens it for use). Returns a file pointer resource on success, or FALSE on error. When opened with mode 'w', it implicitly creates the file if it does not exist.
$handle = fopen($filename, 'mode')  or  die('Cannot open file: ' . $filename);

📝 Example — Create a new file

<?php
$my_file = 'file.txt';

// Mode 'w' creates the file if it doesn't exist
$handle = fopen($my_file, 'w') or die('Cannot open file: ' . $my_file);

echo "File created/opened successfully!";
?>
Output:File created/opened successfully!
The or die() pattern: If fopen() fails and returns FALSE, die() immediately stops the script and displays the error message. This is a safe way to handle file errors.

✏️ Write to a File — fwrite()

fwrite( handle, string )
Writes the given string to the file pointed to by handle. Returns the number of bytes written, or FALSE on failure.

📝 Example — Write data to a file

<?php
$filename = "/home/user/guest/newfile.txt";
$file = fopen($filename, "w");   // Open in write mode

if ($file == false) {
    echo ("Error in opening new file");
    exit();
}

fwrite($file, "This is a simple test\n");  // Write a line of text
fwrite($file, "Second line of text\n");    // Write another line
fclose($file);                             // Always close after writing

echo "Data written successfully!";
?>
Output:Data written successfully!

newfile.txt will contain:
This is a simple test
Second line of text
⚠️ Mode 'w' overwrites! Opening a file with 'w' erases all existing content. To add data without erasing, use mode 'a' (append) instead.

🔒 Close a File — fclose()

fclose( handle )
Closes the file pointed to by handle. Returns TRUE on success or FALSE on failure. Always close files after you are done with them to free up system resources.

📝 Example

<?php
$my_file = 'file.txt';
$handle = fopen($my_file, 'w') or die('Cannot open file: ' . $my_file);

fwrite($handle, "Hello, file!");   // Write some data

fclose($handle);                    // Close the file — free resources
echo "File closed successfully.";
?>
✅ Best Practice: Always call fclose() after finishing with a file. Leaving files open wastes server memory and can cause unexpected behavior.

📖 Read a File — fread()

fread( file, length )
Reads up to length bytes from the open file. Stops at end-of-file or when the specified length is reached — whichever comes first. Returns the read string, or FALSE on failure.
ParameterRequired?Description
file✅ RequiredThe open file resource (handle)
length✅ RequiredMaximum number of bytes to read
Tip — Read entire file: Use filesize($filename) as the length to read the whole file in one call.

📝 Example — Read a whole file

<html>
<body>
<?php
$filename = "tmp.txt";
$file     = fopen($filename, "r");   // Open in read-only mode

if ($file == false) {
    echo("Error in opening file");
    exit();
}

$filesize = filesize($filename);         // Get total size in bytes
$filetext = fread($file, $filesize);     // Read the whole file
fclose($file);                             // Close after reading

echo("File size: $filesize bytes");
echo("<pre>$filetext</pre>");            // <pre> preserves formatting
?>
</body>
</html>
Output (example — if tmp.txt contains "Hello World"):File size: 11 bytes
Hello World

🏁 Check End-of-File — feof()

feof( handle )
Tests whether the file pointer has reached the End Of File (EOF). Returns TRUE if at EOF, otherwise FALSE. Especially useful when reading files of unknown size in a loop.
bool feof ( resource $handle )
⚠️ Note: You cannot read from files opened in write-only modes: w, a, or x. Always use a readable mode (r, r+, a+, etc.) before calling feof().

📝 Example

<?php
$file = fopen("name.txt", "r");

if (feof($file)) {
    echo "End of file reached.";
} else {
    echo "Not at end of file yet.";
}

fclose($file);
?>

📃 Read Line by Line — fgets()

fgets( handle [, length] )
Reads one line from the file pointer. Returns up to length - 1 bytes of text. Returns FALSE when there is no more data or on error. If length is omitted, reads until end of line.
string fgets ( resource $handle [, int $length] )

📝 Example — Read all lines until EOF

<?php
$file = fopen("welcome.txt", "r") or exit("Unable to open file!");

// Keep reading one line at a time until end of file
while (!feof($file)) {
    echo fgets($file) . "<br>";   // Read and display each line
}

fclose($file);
?>
Output (if welcome.txt contains 3 lines):Hello, welcome to PHP!
This is line two.
This is line three.
How it works: while(!feof($file)) keeps looping as long as we have not reached the end of the file. Each call to fgets() reads one line and moves the pointer forward automatically.

🔡 Read Character by Character — fgetc()

fgetc( handle )
Reads exactly one character from the file pointer. Returns a single-character string, or FALSE at end-of-file.
string fgetc ( resource $handle )

📝 Example — Read each character one at a time

<?php
$file = fopen("welcome.txt", "r") or exit("Unable to open file!");

while (!feof($file)) {
    echo fgetc($file);   // Print one character, then move to next
}

fclose($file);
?>
Output (if welcome.txt contains "Hi!"):H i !
fgets() vs fgetc() — When to use which?
Use fgets() when you want to process text line by line (e.g., reading a log file).
Use fgetc() when you need to inspect or process each individual character (e.g., parsing custom file formats).

🗑️ Delete a File — unlink()

unlink( filename )
Deletes a file from the server. Returns TRUE on success, or FALSE on failure (e.g., file doesn't exist or no permissions). Note: there is no "recycle bin" — deletion is permanent.

📝 Example

<?php
$my_file = 'file.txt';

if (unlink($my_file)) {
    echo "$my_file deleted successfully.";
} else {
    echo "Error deleting $my_file.";
}
?>
Output:file.txt deleted successfully.
⚠️ Permanent deletion! Unlike moving to a trash folder, unlink() permanently removes the file. Always double-check the filename before calling this function.

➕ Append to a File — fwrite() with mode 'a'

To add content to a file without erasing existing data, open it in mode 'a' (append). The file pointer starts at the end of the file.

📝 Example — Append a new line each time

<?php
$filename = "log.txt";
$file = fopen($filename, "a") or die("Cannot open file!");

// This adds to the END of the file, not overwriting
fwrite($file, "User logged in at 10:00 AM\n");
fwrite($file, "User logged out at 11:30 AM\n");
fclose($file);

echo "Data appended successfully!";
?>
log.txt content after running twice: User logged in at 10:00 AM
User logged out at 11:30 AM
User logged in at 10:00 AM
User logged out at 11:30 AM
Each time the script runs, new data is added to the bottom of the file — nothing is lost. This is perfect for log files or audit trails.

📌 Complete Function Reference

#OperationFunctionKey Detail
1 Create a File fopen($f, 'w') Creates file if it doesn't exist (with 'w', 'a', 'x')
2 Open a File fopen($f, mode) Returns a file handle/resource for further operations
3 Read a File fread($h, $size) Use filesize() as length to read entire file
4 Read Line by Line fgets($h) Returns one line; use with while(!feof()) loop
5 Read Char by Char fgetc($h) Returns a single character; use with while(!feof())
6 Write to a File fwrite($h, $data) Use mode 'w' — overwrites existing content
7 Append to a File fwrite($h, $data) Use mode 'a' — adds to end, preserves data
8 Check End of File feof($h) Returns TRUE when file pointer reaches end
9 Close a File fclose($h) Always close files to free server resources
10 Delete a File unlink($filename) Permanently deletes — no undo!

🧪 Full Practical Example

A complete script that creates, writes, reads, and then deletes a file:

<?php
$filename = "demo.txt";

// --- STEP 1: Write to file ---
$file = fopen($filename, "w") or die("Cannot open!");
fwrite($file, "Line 1: Hello PHP\n");
fwrite($file, "Line 2: File Handling\n");
fclose($file);
echo "Written. <br>";

// --- STEP 2: Append to file ---
$file = fopen($filename, "a") or die("Cannot open!");
fwrite($file, "Line 3: Appended!\n");
fclose($file);
echo "Appended. <br>";

// --- STEP 3: Read the file line by line ---
$file = fopen($filename, "r") or die("Cannot open!");
echo "<strong>File contents:</strong><br>";
while (!feof($file)) {
    echo fgets($file) . "<br>";
}
fclose($file);

// --- STEP 4: Delete the file ---
unlink($filename);
echo "File deleted.";
?>
Output: Written.
Appended.
File contents:
Line 1: Hello PHP
Line 2: File Handling
Line 3: Appended!
File deleted.
