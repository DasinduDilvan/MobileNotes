🔐 PHP Sessions

Lecture 09 — ICT1232 | Server-Side Scripting

📌 Objective

After the successful completion of this lecture, students should be able to:

  • Use sessions with PHP

1. The Need for Persistence

HTTP is a stateless protocol — each page request is independent and the server has no memory of previous requests. This creates a problem whenever data needs to survive across multiple page loads.

Example Problem — Hit Counter:
Counting the number of times a client loads your web page source. Your .php file needs to remember previous instances of it being requested by a client — but by default, it cannot.

Sessions are one of the key solutions to this statefulness problem in web applications.

2. What is a Session?

A session represents a user's interaction with an application — you open it, do some changes, and then close it.

Session Variables Store user information (in the form of variables) to be used across multiple pages — e.g. username, items selected, etc.
Server-Side Storage Unlike cookies, session data is stored on the server, not on the client's browser. This makes it more secure.
Temporary Session data is transient — it is only relevant for a short time and is usually deleted shortly after the user leaves the website.

3. When Should You Use Sessions?

  • When there is a need for data to be stored on the server
  • When you need unique session information for each user
  • For transient data — only relevant for a short time
  • When data does not contain secret information
  • Similar to Cookies, but stored server-side (not on the client)
Sessions are great for storing temporary data about your visitors, particularly when you do not want that data to be accessible from outside of your server.

5. Starting a Session — session_start()

When you call session_start(), PHP will check whether the visitor has already sent a session:

If session exists PHP will load the existing session data for that visitor.
If no session exists PHP will create a new session file on the server and send a session ID back to the visitor.
⚠️ Critical Rule: session_start() must appear at the very top of EVERY page — even before the <html> tag — otherwise you will not have access to session data. 
<?php
session_start(); // Must be FIRST — before any HTML output
?>
<!-- HTML content begins here -->
<!DOCTYPE html>
<html>
...
Each visitor has their own data locked away in their unique session file. You must call session_start() before trying to read session variables.

6. Adding Session Data

All session data is stored in the session super global array $_SESSION. Each session variable is one element in that array, combined with its value.

Adding variables to $_SESSION is done in the same way as adding variables to any PHP array. Session variables will persist when the user navigates to another page. 
<?php
session_start();

// Store session variables
$_SESSION["username"] = "JohnDoe";
$_SESSION["views"]    = 1;
$_SESSION["cart_item"] = "Laptop";

echo "Session variables have been set.";
?>

7. Reading Session Data

Once stored, session data becomes immediately available in the $_SESSION super global array using the key name you assigned.

  • Session variables are not passed individually to each new page.
  • Instead, they are retrieved from the session opened at the beginning of each page via session_start().
<?php
session_start(); // Open session first

// Read previously stored session variables
echo "Username: " . $_SESSION["username"];
echo "Page views: " . $_SESSION["views"];
?>

✏️ Exercise

Write PHP code to print the session variable views that was created by a previous script, and increment it by 1 on each page visit.

<?php
session_start();

if (isset($_SESSION["views"])) {
    $_SESSION["views"]++;
} else {
    $_SESSION["views"] = 1;
}

echo "Page views: " . $_SESSION["views"];
?>

8. Removing Session Data

unset() Frees (removes) a specific session variable from $_SESSION. Other session data remains intact.
session_unset() Removes all global session variables from the current session. The session file still exists on the server.
session_destroy() Completely destroys the session — resets it and removes all stored session data permanently.

Removing a specific session variable with unset():

<?php
session_start();
if (isset($_SESSION['views']))
    unset($_SESSION['views']); // removes only 'views'
?>

Completely destroying a session:

<?php
session_start();
session_unset();   // remove all session variables
session_destroy(); // destroy the session entirely
?>
⚠️ Note: session_destroy() resets your session and you will lose all stored session data. Use it only when you intend to fully log the user out or end their session.

📋 Summary

Purpose PHP sessions pass data from one page to another across a user's visit to a website.
Storage Session data is stored server-side in a unique file per user, identified by a session ID.
Temporary Session information is temporary and is usually deleted quickly after the user leaves the website.
vs. Cookies PHP Cookies store information about the user on the user's own computer, unlike sessions which stay on the server.
