📋 HTML Forms – Basic Structure

An HTML form is used to collect user input and send it to a server-side script for processing.

<html>
<head>
  <title>A BASIC HTML FORM</title>
</head>
<body>
  <FORM NAME="form1" METHOD=" " ACTION="">
    <INPUT TYPE="TEXT" NAME="Name" VALUE="username">
    <INPUT TYPE="Submit" NAME="Submit1" VALUE="Login">
  </FORM>
</body>
</html>

Key Form Attributes

METHOD Tells the browser how the form information should be sent. Most common values: GET and POST.
ACTION Specifies where to send the form data (the URL of the script to process it).
NAME Identifies the form. Every input element also needs a NAME attribute so PHP can retrieve its value.
Syntax examples:
<FORM NAME="form1" METHOD="GET" ACTION="">
<FORM NAME="form1" METHOD="POST" ACTION="">

🔄 GET vs. POST Methods

GET Method
  • Data is visible to everyone (appended to the URL)
  • Use when data is not sensitive
  • Has limits on the amount of data that can be sent
  • PHP variable: $_GET
POST Method
  • Data is invisible to others (sent in request body)
  • No limits on the amount of data being sent
  • Preferred for sensitive data (passwords, personal info)
  • PHP variable: $_POST
How both work: Both GET and POST create an array of key => value pairs, where keys are the names of the form controls and values are the input data entered by the user.

array( key => value, key2 => value2, key3 => value3, ... )
$_GET – an array of variables passed to the current script via the URL parameters.
$_POST – an array of variables passed to the current script via the HTTP POST method.

🌐 Superglobal Arrays

Superglobals are built-in PHP variables that are always available in all scopes — there is no need to use global $variable to access them within functions.

$_SERVER Stores data about the currently running server (headers, paths, script locations).
$_ENV Stores data about the current client's environment.
$_GET Stores data sent to the server using HTTP method GET.
$_POST Stores data sent to the server using HTTP method POST.
$_REQUEST A super global widely used to collect data after submitting HTML forms. Can be used instead of $_GET or $_POST.
Processing Form Data – Key Questions:
  • How does a PHP script get information from a client? → Superglobal arrays
  • How does PHP get information from the server? → $_SERVER
  • How does PHP save information from a session? → Session superglobals

📦 $_REQUEST

$_REQUEST is a superglobal variable widely used to collect data after submitting HTML forms. Instead of using $_GET and $_POST arrays separately, you can use $_REQUEST.

<?php
  $name = $_REQUEST['name'];
  echo $name;
?>

🖥️ $_SERVER

$_SERVER holds information about headers, paths, and script locations.

<?php
  echo $_SERVER['PHP_SELF'];      // Output: /myFile.php
  echo "<br>";
  echo $_SERVER['SERVER_NAME'];   // Output: localhost
  echo "<br>";
  echo $_SERVER['SCRIPT_NAME'];   // Output: /myFile.php
?>

Using PHP_SELF in the Action Field

PHP_SELF is a variable that returns the name and path of the current script being executed. Using it in the form action allows you to write more generic, reusable code.

<form name="form1" method="post"
      action="<?php echo $_SERVER['PHP_SELF']; ?>">
Using $_SERVER['PHP_SELF'] means the form submits to the same page it is on, making the script reusable across multiple pages.

🎛️ Handling Form Controls

Text Box

To get the text that a user entered into a text box, the input element must have a NAME attribute.

<INPUT TYPE="Text" VALUE="username" NAME="uname">

Then, in PHP retrieve the value using:

$username = $_POST['uname'];   // using POST
$username = $_GET['uname'];    // using GET

Radio Buttons

All radio buttons in a group share the same NAME attribute. Each has a unique value. PHP reads whichever was selected.

<!-- HTML Form -->
<FORM name="form1" method="post" action="radioButton.php">
  <Input type='Radio' Name='gender' value='male'>Male
  <Input type='Radio' Name='gender' value='female'>Female
  <Input type="Submit" Name="Submit1" VALUE="Select a Radio Button">
</FORM>
<!-- PHP Processing -->
<?PHP
  $male_status   = 'unchecked';
  $female_status = 'unchecked';

  if (isset($_POST['Submit1'])) {
    $selected_radio = $_POST['gender'];

    if ($selected_radio == 'male') {
      $male_status = 'checked';
      echo "Male Selected";
    }
    else if ($selected_radio == 'female') {
      $female_status = 'checked';
      echo "Female Selected";
    }
  }
?>

Dropdown List (Select)

Use <select> with a name attribute. PHP reads the selected option's value.

<!-- HTML Form -->
<form action="" method="post">
  <select name="clr">
    <option value="Red">Red</option>
    <option value="Green">Green</option>
  </select>
  <input type="submit" name="btnsubmit"/>
</form>

<!-- PHP Processing -->
<?php
  if (isset($_POST['btnsubmit']))
    echo "Selected color is : " . $_POST['clr'];
?>

✅ Form Validation

Client-Side Validation User input validated in the browser using JavaScript before submitting to the server. Fast but can be bypassed.
Server-Side Validation Used when data will be inserted into a database. Performed using PHP. More secure and reliable.

Key Validation Functions

empty() Determines whether a variable is empty.
if (empty($_POST["name"]))
isset() Determines if a variable is set and is not NULL.
if (isset($_POST['submit']))

Form Processing Tips

  • Always confirm that valid information was entered before processing.
  • extract() function – creates variables corresponding to each key-value pair in an array, making it easy to retrieve all POST values. ⚠ Do NOT use extract() on untrusted data like user input with $_GET.
  • Regular expressions are very helpful for validating formats (phone numbers, emails, etc.).
  • die() function – terminates script execution. Remember to close all HTML tags before calling it.

Validating Text (Letters Only)

Check if the name field only contains letters and whitespace using preg_match(). This function searches a string for a pattern and returns true if the pattern exists, false otherwise.

$name = $_POST["name"];

if (!preg_match("/^[a-zA-Z ]*$/", $name)) {
  $nameErr = "Only letters and white space allowed";
}

Validating Email Format

Use PHP's filter_var() function with the FILTER_VALIDATE_EMAIL filter to check whether an email address is well-formed.

  • filter_var() – Filters a variable with a specified filter.
  • FILTER_VALIDATE_EMAIL – Validates an email address format.
$email = $_POST["email"];

if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
  $emailErr = "Invalid email format";
}

Validating Phone Number with Regular Expression

<?php
extract($_POST);

if (isset($_POST['submit'])) {
  // ... form processing ...
}

// Check if phone number is valid using regex
if (!preg_match("/^(\(\+[0-9]{2}\)[0-9]{9})$/", "412278456")) {
  print("<p>INVALID PHONE NUMBER<br />
    A valid phone number must be in the form
    <strong>(+94)412278456</strong>
  </p></body></html>");
  die(); // terminate script execution
} else {
  echo "Valid phone no";
}
?>

🔍 var_dump() Function

var_dump() displays structured information (type and value) about one or more variables. Arrays and objects are explored recursively with values indented to show structure. All public, private, and protected properties of objects are included in the output.

Syntax:
var_dump(variable1, variable2, ....variableN)

Example

Code: 
<?php
$a = array(1, 2, array("a", "b", "c"));
var_dump($a);
?>
Output: 
array(3) {
  [0]=>
  int(1)
  [1]=>
  int(2)
  [2]=>
  array(3) {
    [0]=>
    string(1) "a"
    [1]=>
    string(1) "b"
    [2]=>
    string(1) "c"
  }
}
