⚙️ Rules of Inference

TMS1223 Discrete Mathematics — Building Valid Arguments & Proofs
📘 Valid Arguments & Proofs

Argument: A sequence of statements that ends with a conclusion. The preceding statements are premises (hypotheses).

Valid argument (propositional logic): If all premises are true, then the conclusion must be true. Equivalently, the implication (p₁ ∧ p₂ ∧ ... ∧ pₙ) → q is a tautology.

⭐ Core idea: Rules of inference are templates for constructing valid arguments. They are the building blocks of mathematical proofs.
📌 Instead of huge truth tables (e.g., 1024 rows for 10 variables), we use inference rules to reason step‑by‑step.
📋 Rules of Inference – Propositional Logic
Modus Ponens (MP)
p → q
p
∴ q
Tautology: (p ∧ (p→q)) → q

Example: “If it snows, I study. It snows → I study.”

Modus Tollens (MT)
p → q
¬q
∴ ¬p
Tautology: (¬q ∧ (p→q)) → ¬p

Example: “If snow, study. Not study → not snow.”

Hypothetical Syllogism (HS)
p → q
q → r
∴ p → r
Tautology: ((p→q) ∧ (q→r)) → (p→r)
Disjunctive Syllogism (DS)
p ∨ q
¬p
∴ q
Tautology: ((p∨q) ∧ ¬p) → q
Addition (Add)
p
∴ p ∨ q
Tautology: p → (p∨q)
Simplification (Simp)
p ∧ q
∴ p
Tautology: (p∧q) → p
Conjunction (Conj)
p
q
∴ p ∧ q
Tautology: (p ∧ q) → (p∧q)
Resolution (Res)
¬p ∨ r
p ∨ q
∴ q ∨ r
Tautology: ((¬p∨r) ∧ (p∨q)) → (q∨r)
💡 To identify the rule, match the pattern of premises and conclusion. For example: p, p→q ⇒ q = Modus Ponens.
🔍 Recognizing Rules of Inference – Examples
Example 1
“It is below freezing. Therefore, it is below freezing or raining.”
Rule: Addition (p → p∨q)
Example 2
“It is below freezing and raining. Therefore, it is below freezing.”
Rule: Simplification (p∧q ⇒ p)
Example 3
“If it rains, no barbecue today. If no barbecue today, then barbecue tomorrow. ∴ If it rains, barbecue tomorrow.”
Rule: Hypothetical Syllogism
🏗️ Building Valid Arguments – Step by Step
Example (from premises to conclusion):
Premises: “It is not sunny this afternoon and it is colder than yesterday.”
“We will go swimming only if it is sunny.”
“If we do not go swimming, then we will take a canoe trip.”
“If we take a canoe trip, then we will be home by sunset.”
Conclusion: “We will be home by sunset.”
Solution:
Let p: “It is sunny this afternoon”, q: “It is colder than yesterday”, r: “We will go swimming”, s: “We will take a canoe trip”, t: “We will be home by sunset”.
Hypotheses: ¬p ∧ q, r → p, ¬r → s, s → t Conclusion: t.
  • 1. ¬p ∧ q      (Premise)
  • 2. ¬p      (Simplification, 1)
  • 3. r → p      (Premise)
  • 4. ¬r      (Modus tollens, 2,3)
  • 5. ¬r → s      (Premise)
  • 6. s      (Modus ponens, 4,5)
  • 7. s → t      (Premise)
  • 8. t      (Modus ponens, 6,7) ✓
Another short example: “Randy works hard. If Randy works hard then he is a dull boy. If Randy is a dull boy then he will not get the job. ∴ Randy will not get the job.”
w → d, d → ¬j ⇒ using modus ponens twice gives ¬j.
🌍 Rules of Inference for Quantified Statements (Predicate Logic)
Universal Instantiation (UI)
∀x P(x)
∴ P(c) for any c

If everything has property P, then a specific instance has it.

Universal Generalization (UG)
P(c) for arbitrary c
∴ ∀x P(x)

If any arbitrary element satisfies P, then all do.

Existential Instantiation (EI)
∃x P(x)
∴ P(c) for some c

Introduce a new constant symbol for the witness.

Existential Generalization (EG)
P(c) for some c
∴ ∃x P(x)

From a specific instance, conclude existence.

Example (UI + Modus Ponens): “Everyone in this class has taken CS. Marla is in this class. Therefore Marla has taken CS.”
∀x (D(x)→C(x)), D(Marla) ⇒ C(Marla).
📐 Quantified Arguments – Detailed Proofs
Example 1: Premises: “A student in this class has not read the book.” ”Everyone in this class passed the first exam.” Conclusion: “Someone who passed the first exam has not read the book.”
Let C(x): “x is in this class”, B(x): “x has read the book”, P(x): “x passed the first exam”.
Premises: ∃x (C(x) ∧ ¬B(x)) , ∀x (C(x) → P(x)) . Conclusion: ∃x (P(x) ∧ ¬B(x)).

Proof:
  1. ∃x (C(x) ∧ ¬B(x))    (Premise)
  2. C(a) ∧ ¬B(a)    (Existential instantiation, 1)
  3. C(a)    (Simplification, 2)
  4. ∀x (C(x) → P(x))    (Premise)
  5. C(a) → P(a)    (Universal instantiation, 4)
  6. P(a)    (Modus ponens, 3,5)
  7. ¬B(a)    (Simplification, 2)
  8. P(a) ∧ ¬B(a)    (Conjunction, 6,7)
  9. ∃x (P(x) ∧ ¬B(x))    (Existential generalization, 8) ✓
💡 Note: Universal generalization requires that the element be arbitrary (no special assumptions). Existential instantiation introduces a new constant that hasn't been used before.
⚖️ Truth Tables vs. Inference Rules

A truth table can always verify validity, but for arguments with many variables it becomes impractical (e.g., 210 = 1024 rows). Rules of inference give an efficient, step‑by‑step method to derive conclusions from premises — the foundation of mathematical proofs.

Takeaway: A valid argument guarantees that if all premises are true, the conclusion cannot be false. Using inference rules, we build chains of reasoning: each step is justified by a rule applied to previous statements or premises.
📌 Quick Reference: Inference Rules Summary
Propositional Rules
  • Modus Ponens: p→q, p ⇒ q
  • Modus Tollens: p→q, ¬q ⇒ ¬p
  • Hypothetical Syllogism: p→q, q→r ⇒ p→r
  • Disjunctive Syllogism: p∨q, ¬p ⇒ q
  • Addition: p ⇒ p∨q
  • Simplification: p∧q ⇒ p
  • Conjunction: p, q ⇒ p∧q
  • Resolution: ¬p∨r, p∨q ⇒ q∨r
Quantified Rules
  • ∀xP(x) ⇒ P(c)    (UI)
  • P(c) for arbitrary c ⇒ ∀xP(x) (UG)
  • ∃xP(x) ⇒ P(c) (new c) (EI)
  • P(c) ⇒ ∃xP(x)    (EG)
Validity condition

An argument is valid iff (premise₁ ∧ … ∧ premiseₙ) → conclusion is a tautology.

✅ In any valid argument, the conclusion is a logical consequence of the premises. Inference rules are the tools that make this consequence explicit.
🎯 Why Rules of Inference Matter

Rules of inference are not only for mathematics: they form the logical core of artificial intelligence, program verification, hardware design, and theorem provers. By mastering these rules, you learn to reason rigorously — from simple propositional steps to complex quantified arguments.
