📚 Nested Quantifiers

Predicate logic · Quantifier negation · Order matters · Translation & truth

📖 Predicates & Quantifiers

Propositional function (predicate): a statement containing variables. Example: P(x) = “x is greater than 3”. The variable is the subject, the predicate expresses a property.

🌍 Universal quantifier ∀
“for all”, “every”
∀x P(x) : P(x) true for every x in domain.
🔎 Existential quantifier ∃
“there exists”, “some”
∃x P(x) : P(x) true for at least one x.
Charles Peirce (1839–1914) – Quantifier theory pioneer.
Quantifiers bind the variable. Domain of discourse is essential.
Translation examples:
“Every student in this class has taken Java.”
Domain = all students → ∀x J(x).
Domain = all people → ∀x (S(x) → J(x))
❗ ∀x (S(x) ∧ J(x)) would mean everything is a student and took Java → wrong.

🚫 Negating Quantified Expressions

✍️ De Morgan’s laws for quantifiers:
¬∀x P(x) ≡ ∃x ¬P(x)
¬∃x P(x) ≡ ∀x ¬P(x)
Intuition:
“Not all are” ≡ “Some are not”.
“None exist” ≡ “All are not”.
Example 1: Negation of “Every student passed” → “There is a student who did NOT pass”.
Example 2: “There is an honest politician” (∃x H(x)) negated → ∀x ¬H(x) → “Every politician is dishonest”. 
¬∀x (x² > x)  ≡  ∃x (x² ≤ x)
¬∃x (x² = 2)  ≡  ∀x (x² ≠ 2)
Logical equivalence proof style: ¬∀x P(x) true ↔ ∀x P(x) false ↔ there exists x with ¬P(x) ↔ ∃x ¬P(x) true.
Advanced equivalence: ¬∀x (P(x) → Q(x)) ≡ ∃x (P(x) ∧ ¬Q(x)). Use De Morgan + implication negation.

📌 Quantifiers as Conjunctions / Disjunctions (Finite Domains)

If domain = {1,2,3}:

  • ∀x P(x) ≡ P(1) ∧ P(2) ∧ P(3)
  • ∃x P(x) ≡ P(1) ∨ P(2) ∨ P(3)

Even for infinite domains, this conceptual view helps understand meaning (infinite conjunction/disjunction).

🌱 Translation Practice (English → Logic)

Domain = {Fruit, Vegetable, Plant}; F(x)=“x is fruit”, S(x)=“veg”, T(x)=“plant”.

“Everything is a fruit”
∀x F(x)
“Nothing is a vegetable”
¬∃x S(x) ≡ ∀x ¬S(x)
“All fruits are plants”
∀x (F(x) → T(x))
“Some fruits are plants”
∃x (F(x) ∧ T(x))
“No vegetable is a plant”
¬∃x (S(x) ∧ T(x)) ≡ ∀x (¬S(x) ∨ ¬T(x))
“If any fruit is a vegetable then it is also a plant”
∀x ((F(x) ∧ S(x)) → T(x))

🧩 Nested Quantifiers

Quantifiers inside the scope of other quantifiers. Order matters!

∀x ∃y (x + y = 0) (over reals) → “Every real number has an additive inverse” → true.
∃y ∀x (x + y = 0) → “There exists a y that works for all x” → false (no universal additive constant).
📖 Translation: ∀x∀y ((x > 0) ∧ (y < 0) → (xy < 0))
→ “Product of a positive and a negative real number is always negative”.
Order swapping caution: ∀x∀y P(x,y) ≡ ∀y∀x P(x,y) (always equivalent).
But ∀x∃y P(x,y) is NOT equivalent to ∃y∀x P(x,y) in general.

⚖️ Truth Values & Order – Worked Examples

Example 1: Domain = ℝ, P(x,y): x·y = 0

  • ∀x∀y P(x,y) → False (not every pair multiplies to 0).
  • ∀x∃y P(x,y) → True (for each x, choose y=0 → product 0).
  • ∃x∀y P(x,y) → True (x=0 makes 0·y=0 for all y).
  • ∃x∃y P(x,y) → True (e.g., x=0, y=5).

Example 2: Domain = ℝ, P(x,y): x / y = 1 (division by zero issues - careful)

  • ∀x∀y P(x,y) → False.
  • ∀x∃y P(x,y) → False (for x=0 no y gives 0/y=1).
  • ∃x∀y P(x,y) → False (even for fixed x, cannot hold ∀y).
  • ∃x∃y P(x,y) → True (choose x=5, y=5).
∀x∀y∃z (x+y=z) → true (sum exists).
∃z∀x∀y (x+y=z) → false (no single z works for every x,y).

🌐 Translating Nested Quantifiers to English

∀x ( C(x) ∨ ∃y (C(y) ∧ F(x,y)) )
C(x)=“x has computer”, F(x,y)=“x and y are friends”.
➜ “Every student has a computer or has a friend who has a computer.”
Mathematical statement: “The sum of two positive integers is always positive.”
∀x∀y ( (x > 0 ∧ y > 0) → (x + y > 0) ), domain = all integers.
Complex nested example: “There is a woman who has taken a flight on every airline in the world.”
Let P(w,f)=“w has taken f”, Q(f,a)=“f is a flight on airline a”.
∃w ∀a ∃f ( P(w,f) ∧ Q(f,a) )

⛔ Negating Nested Quantifiers (Moving ¬ inside)

Rule: successively apply De Morgan for quantifiers.

Original:  ∀x ∃y (xy = 1)
Negation:  ¬∀x ∃y (xy = 1)
  ≡ ∃x ¬∃y (xy = 1)
  ≡ ∃x ∀y ¬(xy = 1)
  ≡ ∃x ∀y (xy ≠ 1)
Example with women & flights:
Original: ∃w ∀a ∃f (P(w,f) ∧ Q(f,a))
Negation: ¬∃w ∀a ∃f (P(w,f) ∧ Q(f,a))
→ ∀w ¬∀a ∃f (P(w,f) ∧ Q(f,a))
→ ∀w ∃a ¬∃f (P(w,f) ∧ Q(f,a))
→ ∀w ∃a ∀f ¬(P(w,f) ∧ Q(f,a))
→ ∀w ∃a ∀f (¬P(w,f) ∨ ¬Q(f,a))
English: “For every woman there is an airline such that for all flights, she has not taken that flight or the flight is not on that airline.”

⚠️ Order & Distribution – Important Non-Equivalences

✅ ∀x∀y P(x,y) ≡ ∀y∀x P(x,y)
Swapping universal quantifiers does NOT change meaning.
❌ ∀x∃y P(x,y) ≢ ∃y∀x P(x,y)
Counterexample: x+y=0 (first true, second false).
Distribution over ∧ works: ∀x (P(x) ∧ Q(x)) ≡ ∀x P(x) ∧ ∀x Q(x)
❌ Distribution over → fails: ∀x (P(x) → Q(x)) is not equivalent to ∀x P(x) → ∀x Q(x).
Counterexample: domain animals, P=“x is a fish”, Q=“x has scales”. Left may be false (some fish no scales), right side true vacuously.

📝 Additional Translation Drills

  • “All real numbers have a square root.” (over ℝ) → ∀x ∃y (y² = x) (false for negative numbers requires domain specification).
  • “Every student has at least one friend who is a CS major.” → ∀s ∃f (Friend(s,f) ∧ CS(f))
  • “There is a number that is greater than every other number.” → false statement; logically ∃x ∀y (y ≠ x → y < x).
Tip: When translating “every A is B” use implication: ∀x (A(x) → B(x)).
For “some A is B” use conjunction: ∃x (A(x) ∧ B(x)).

🎓 Fun with Translation (Fruits & Plants recap)

  • “Everything is a fruit” → ∀x F(x)
  • “Nothing is a Vegetable” → ¬∃x S(x) ≡ ∀x ¬S(x)
  • “All fruits are plants” → ∀x (F(x) → T(x))
  • “Some fruits are plants” → ∃x (F(x) ∧ T(x))
  • “No vegetable is a plant” → ∀x (S(x) → ¬T(x)) or ¬∃x (S(x) ∧ T(x))
  • “If any fruit is a vegetable then it is also a plant” → ∀x ((F(x) ∧ S(x)) → T(x))

🔁 Translating from Logic to English

∀x (C(x) ∨ ∃y (C(y) ∧ F(x,y))) → “Everyone has a computer or knows someone who has a computer.”
∃x ∀y (x ≥ y) over integers → “There exists a greatest integer” (false).
∀x ∃y (x < y) → “For every integer there is a larger integer.”

📌 Quick Summary: Nested Quantifiers

  • ∀x ∀y P(x,y) : true only if P holds for every ordered pair.
  • ∃x ∃y P(x,y) : true if at least one pair satisfies P.
  • ∀x ∃y P(x,y) : for each x, some y works (y may depend on x).
  • ∃x ∀y P(x,y) : there is a single x that works with every y — much stronger.
★ The meaning changes if quantifiers are swapped: ∀x∃y ≠ ∃y∀x in general. Always read from left to right.
