📌 Intro & Propositional Logic
Lecture 01 · What is Discrete Math? · Propositions · Logical Operators · Truth Tables
What is Discrete Mathematics? Mathematics that deals with distinct, separate objects — not continuous. It is the mathematical foundation for computer science.

What is a Proposition?

A proposition is a statement that is clearly TRUE or FALSE — nothing in between.

✅ These ARE propositions
  • "Elephants are bigger than mice." → TRUE
  • "520 < 111" → FALSE
  • "2 + 2 = 3" → FALSE
❌ These are NOT propositions
  • "y > 5" → depends on y (open sentence)
  • "What time is it?" → a question
  • "Please don't sleep." → a request
💡 We use letters like p, q, r, s to name propositions. Truth values are T (true) and F (false).

The Conditional p → q and its Relatives

Starting with: p → q ("If it is raining, then the home team wins")

NameFormulaExampleEquivalent to…
Originalp → qIf raining → home team wins—
Contrapositive¬q → ¬pIf home team loses → not raining✅ Same as original
Converseq → pIf home team wins → raining❌ Different meaning
Inverse¬p → ¬qIf not raining → home team loses❌ Different meaning
✅ p → q ≡ ¬q → ¬p (Conditional = Contrapositive — always the same!)
✅ q → p ≡ ¬p → ¬q (Converse = Inverse — always the same!)
❌ The original is NOT the same as its converse or inverse.

Biconditional p ↔ q ("if and only if")

True when p and q have the same truth value (both T or both F).

Ways to say it: "p iff q" · "p is necessary AND sufficient for q" · "if p then q, and conversely"

Example: "You can take the flight if and only if you buy a ticket." — True when both hold or both fail.

2. System Specifications (Consistency)

A set of specs is consistent if there exists at least one truth assignment that makes ALL specs TRUE at the same time.

Example: Are these specs consistent?

  • "Diagnostic message stored in buffer OR retransmitted." → p ∨ q
  • "Message NOT stored in buffer." → ¬p
  • "If stored in buffer, then retransmitted." → p → q
✅ YES — consistent! When p = F, q = T, all three are TRUE.

❌ Add "Message NOT retransmitted" (¬q) → NOW inconsistent. No assignment works for all four.

4. Logic Puzzles — Knights & Knaves

Knights always tell the truth. Knaves always lie.

A says: "B is a knight." | B says: "We are of opposite types."

Answer: BOTH are Knaves. If A were a knight, B would be a knight too — but then B's statement "we are opposites" would be false (contradiction). So A is a knave → B is also a knave → B's lie is consistent. ✓

5. Logic Circuits (Gates)

NOT Gate (Inverter)
Input: p → Output: ¬p
Flips 0→1, 1→0
AND Gate
Inputs: p, q → Output: p ∧ q
Output is 1 only if BOTH inputs are 1
OR Gate
Inputs: p, q → Output: p ∨ q
Output is 1 if AT LEAST ONE input is 1
Example circuit outputs:
• (p ∧ ¬q) ∨ ¬r — p AND (not q), then OR with (not r)
• ¬(p ∧ (q ∨ ¬r)) — build inner expression, then negate

What is Logical Equivalence?

Two propositions are logically equivalent (written ≡ or ⇔) if they always have the exact same truth value — i.e., their biconditional is a tautology.

All Key Equivalence Laws (Reference Table)

Law NameEquivalence
Identityp ∧ T ≡ p    p ∨ F ≡ p
Dominationp ∨ T ≡ T    p ∧ F ≡ F
Idempotentp ∨ p ≡ p    p ∧ p ≡ p
Double Negation¬(¬p) ≡ p
Commutativep ∨ q ≡ q ∨ p    p ∧ q ≡ q ∧ p
Associative(p ∨ q) ∨ r ≡ p ∨ (q ∨ r)
Distributivep ∨ (q ∧ r) ≡ (p ∨ q) ∧ (p ∨ r)
De Morgan's¬(p ∧ q) ≡ ¬p ∨ ¬q    ¬(p ∨ q) ≡ ¬p ∧ ¬q
Absorptionp ∨ (p ∧ q) ≡ p    p ∧ (p ∨ q) ≡ p
Negationp ∨ ¬p ≡ T    p ∧ ¬p ≡ F

Equivalences Involving → and ↔

Conditional → key ones:
  • p → q ≡ ¬p ∨ q
  • p → q ≡ ¬q → ¬p
  • ¬(p → q) ≡ p ∧ ¬q
  • p ∨ q ≡ ¬p → q
Biconditional ↔ key ones:
  • p ↔ q ≡ (p → q) ∧ (q → p)
  • p ↔ q ≡ (p ∧ q) ∨ (¬p ∧ ¬q)
  • ¬(p ↔ q) ≡ p ↔ ¬q

Equivalence Proof (Step-by-Step)

Prove: ¬(p ∨ (¬p ∧ q)) ≡ ¬p ∧ ¬q

StepExpressionRule Used
1¬p ∧ ¬(¬p ∧ q)De Morgan's (2nd)
2¬p ∧ (¬(¬p) ∨ ¬q)De Morgan's (1st)
3¬p ∧ (p ∨ ¬q)Double Negation
4(¬p ∧ p) ∨ (¬p ∧ ¬q)Distributive
5F ∨ (¬p ∧ ¬q)Negation law
6¬p ∧ ¬q ✓Identity law

Normal Forms (DNF & CNF)

DNF — Disjunctive Normal Form
Structure: (conjunctions) joined by OR
(p ∧ ¬q) ∨ (¬p ∧ q)

Used in circuit design
CNF — Conjunctive Normal Form
Structure: (disjunctions) joined by AND
(p ∨ ¬r) ∧ (¬q ∨ p)

Used in AI theorem proving

Propositional Satisfiability

  • Satisfiable: At least one assignment of T/F makes it TRUE
  • Unsatisfiable: No assignment can make it TRUE (its negation is a tautology)
Sudoku as satisfiability: Each cell assignment becomes a propositional variable p(i,j,n) — "number n is in row i, column j." 9×9×9 = 729 variables. Solving Sudoku = finding a satisfying assignment.
🧩 Predicate Logic
Lecture 05 · Propositional Functions · Quantifiers · Translation · De Morgan for Quantifiers

Why Predicate Logic?

Propositional logic is not powerful enough. It cannot handle statements like "Every student in class is studying." — we need variables, predicates, and quantifiers.

Propositional Functions (Predicates)

A predicate is a statement with a variable. It becomes a proposition when the variable gets a value.

PredicateExampleResult
P(x) = "x > 3"P(4)TRUE
P(x) = "x > 3"P(2)FALSE
Q(x,y) = "x = y + 3"Q(3, 0)TRUE
Q(x,y) = "x = y + 3"Q(1, 2)FALSE

The Two Quantifiers

∀ — Universal Quantifier
Means "for all", "every", "each"

∀x P(x) = "P(x) is true for EVERY x in the domain"

FALSE if even ONE counterexample exists.
∃ — Existential Quantifier
Means "there exists", "for some", "at least one"

∃x P(x) = "P(x) is true for AT LEAST ONE x"

FALSE only if NO element satisfies it.

∃! — Uniqueness Quantifier

∃!x P(x) means exactly ONE element satisfies P(x).
Example: ∃!x (x − 1 = 0) over reals → TRUE (only x = 1)

Translation Rules — English ↔ Logic

⚠️ Most common exam mistake:
"Every student took Java" → ∀x (S(x) → J(x)) ✅
NOT ∀x (S(x) ∧ J(x)) ❌ (This means everything is a student AND took Java!)
English PatternUseExample
"All A are B"∀x (A(x) → B(x))"All humans are mortal"
"Some A is B"∃x (A(x) ∧ B(x))"Some student visited Mexico"
"No A is B"¬∃x (A(x) ∧ B(x))"No vegetable is a plant"
"Not all A are B"∃x (A(x) ∧ ¬B(x))"Not all students passed"

De Morgan's Laws for Quantifiers

NegationEquivalent toPlain English
¬∀x P(x)∃x ¬P(x)"Not all" = "Some are not"
¬∃x P(x)∀x ¬P(x)"None exist" = "All are not"
Example: Negate ∀x (x² > x) → ∃x (x² ≤ x)
Advanced: ¬∀x (P(x) → Q(x)) ≡ ∃x (P(x) ∧ ¬Q(x))

🔁 Nested Quantifiers
Lecture 06 · Quantifiers inside quantifiers · Order matters · Negating nested expressions

What are Nested Quantifiers?

When one quantifier appears inside the scope of another. Order matters!

Order Matters — Key Examples

StatementMeaningTrue over ℝ?
∀x ∃y (x + y = 0)Every number has an additive inverse✅ TRUE (y = −x)
∃y ∀x (x + y = 0)One single y works as inverse for all x❌ FALSE
∀x∀y P(x,y)P holds for every pair (x,y)Usually strong/false
∃x∃y P(x,y)At least one pair satisfies PUsually true
⚠️ ∀x∃y P(x,y) ≠ ∃y∀x P(x,y)
The first says "for each x, some y works (y can depend on x)".
The second says "one fixed y works for ALL x" — much stronger claim!

Swapping Rules

✅ OK to swap same quantifiers:
∀x ∀y P(x,y) ≡ ∀y ∀x P(x,y)
∃x ∃y P(x,y) ≡ ∃y ∃x P(x,y)
❌ CANNOT swap different quantifiers:
∀x ∃y P(x,y) ≢ ∃y ∀x P(x,y)
These have completely different meanings!

Negating Nested Quantifiers (Move ¬ inward)

Apply De Morgan's laws for quantifiers one step at a time, moving ¬ through each quantifier (flip ∀ ↔ ∃).

StepExpressionRule
Original∀x ∃y (xy = 1)—
Negate¬∀x ∃y (xy = 1)—
Step 1∃x ¬∃y (xy = 1)De Morgan: ¬∀ → ∃¬
Step 2∃x ∀y ¬(xy = 1)De Morgan: ¬∃ → ∀¬
Final∃x ∀y (xy ≠ 1)Simplify

What is a Valid Argument?

An argument is valid if: whenever ALL premises are TRUE, the conclusion MUST also be TRUE.
In other words: (premise₁ ∧ premise₂ ∧ ... ∧ premiseₙ) → conclusion is a tautology.

The 8 Propositional Rules of Inference

Worked Example — Full Proof

  • "We swim only if it is sunny." (r → p)

  • "If we canoe, we'll be home by sunset." (s → t)

What is a Set?

A set is an unordered collection of distinct objects. Order doesn't matter. Repetitions are ignored.
If a belongs to set A → write a ∈ A. If not → a ∉ A.

How to Describe Sets

Roster (List) Method
S = {a, b, c, d}
V = {a, e, i, o, u} (vowels)
ℕ = {0, 1, 2, 3, ...}
Set-Builder Notation
S = { x | P(x) }
Example: O = { x ∈ ℤ⁺ | x is odd, x < 10 }
= {1, 3, 5, 7, 9}
Special sets: ℕ = naturals {0,1,2,…} · ℤ = integers · ℤ⁺ = positive integers · ℚ = rationals · ℝ = reals · ℂ = complex

Cartesian Product

A × B = {(a,b) | a ∈ A and b ∈ B}

A = {a, b}, B = {1, 2, 3}
A × B = {(a,1), (a,2), (a,3), (b,1), (b,2), (b,3)}
Note: A × B ≠ B × A (unless both are equal or one is empty)
A subset of A × B is called a relation from A to B — fundamental in databases and graphs.

Cardinality & Countability

  • A set is countable if it is finite OR can be listed as a sequence (bijection with ℤ⁺)
  • Countably infinite sets have cardinality ℵ₀ (aleph-null)
  • ℤ (integers) is countable: list 0, 1, −1, 2, −2, 3, −3, …
  • ℚ (rationals) is countable (Cantor diagonal argument)
  • ℝ (real numbers) is uncountable (proved by Cantor)

Proof: Odd positive integers are countable

Define f: ℤ⁺ → O by f(n) = 2n − 1
• Injective: f(n) = f(m) → 2n−1 = 2m−1 → n = m ✓
• Surjective: any odd t = 2k−1 → t = f(k) ✓
So f is a bijection → |O| = |ℤ⁺| → countable ✓
