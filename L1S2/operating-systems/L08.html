📖 Main Memory Management

Address binding · Contiguous allocation · Paging · Dynamic loading
🎯 Background & Memory Hierarchy

Programs (instructions + data) must be in main memory (at least partially) to execute. CPU fetches instructions/data from memory according to the program counter. Memory hierarchy:

  • Registers: access in ≤ 1 CPU clock cycle
  • Cache: few cycles; sits between main memory and registers
  • Main memory: many cycles; large array of bytes each with its own address
📌 Multiprogramming requirement: keep several processes in memory to improve CPU utilization and response. Each process has a separate memory space to protect processes from each other.
🛡️ Hardware Protection: Base & Limit Registers

Legal logical address space defined by a pair of base and limit registers:

  • Base register: holds the smallest legal physical memory address.
  • Limit register: specifies the size of the range.
✅ CPU checks every memory access in user mode: if address ≥ base + limit → trap to OS (fatal error). Prevents user programs from accessing OS or other processes' memory.
🖥️ Hardware address protection: any illegal access → trap → OS handles as fatal error.
🔗 Address Binding

Process may move between disk and memory → mapping from one address space to another. Binding can happen at three stages:

⌛ Compile time
Absolute code generated. If memory location changes, recompile.
⏳ Load time
Relocatable code; binding at load time by loader.
⚙️ Execution time
Binding delayed until run-time; requires MMU. Most modern OS use this.
Multistep program processing: source code → compiler/assembler → object module → linker → load module → loader → memory image.
🧠 Logical vs. Physical Address & MMU
Logical address (virtual): generated by CPU; relative address.
Physical address: address seen by memory unit; absolute address.
Memory-Management Unit (MMU): hardware that maps logical → physical at run-time. User program deals only with logical addresses.
💡 Dynamic relocation using relocation register (base register): logical address + relocation register = physical address.
Example: logical address 5, base = 45 → physical address 50. 
CPU ----> logical address ----> MMU (add relocation register) ----> physical address ----> memory
📦 Dynamic Loading

Motivation: entire program does not need to be in memory at once. Routine loaded only when called.

  • Main program loaded first; when a routine calls another, calling routine checks if target is loaded.
  • If not loaded, relocatable linking loader loads the routine, updates address tables.
  • Advantage: unused routines never loaded → saves memory, better memory utilization.
  • No special OS support required; users can implement with library routines.
📘 Dynamic loading does not require OS support, but OS may provide helper libraries.
📐 Contiguous Memory Allocation

Each process contained in a single contiguous section of memory. Two common partitioning schemes:

Fixed-size partitions
Memory divided into static partitions. Degree of multiprogramming bounded by number of partitions. Simple but suffers from internal fragmentation.
Variable partitions (dynamic)
Hole table: initially one large hole. Allocate exactly needed size. When process terminates, merge adjacent holes. More flexible but external fragmentation appears.
Hole management: OS keeps list of free blocks. When process arrives, search for hole large enough.
⚙️ Dynamic Storage-Allocation Strategies

Given a request of size n, three popular algorithms to select a hole:

🔹 First-fit
Allocate the first hole big enough. Fast.
🔸 Best-fit
Allocate the smallest hole that is big enough. Minimizes leftover, but must search entire list.
🔹 Worst-fit
Allocate the largest hole. Leaves largest leftover.

📊 Performance: First-fit and best-fit are better than worst-fit in speed and storage utilization.

🧩 Fragmentation & Compaction
External fragmentation: total free memory exists but not contiguous; holes scattered.
Internal fragmentation: allocated memory slightly larger than requested; unused space inside allocated region.
Example: 18,464 B hole, process requests 18,462 B → 2 B hole (external). If partition-based, internal fragmentation occurs.
Compaction (to reduce external fragmentation): shuffle memory contents to combine free blocks into one large hole.
⚠️ Possible only if relocation is dynamic (execution-time binding). Requires updating base registers.
🔁 Alternative: allow non‑contiguous logical address space → Paging and Segmentation.
📄 Paging – Non‑contiguous Memory Management

Basic idea: break physical memory into fixed-size blocks called frames; break logical memory into same-size blocks called pages.

  • Frame size = power of 2 (512 B – 16 MB).
  • To run a program of size N pages, find N free frames and load program pages into them.
  • Page table (per process) maps logical page number to physical frame number. OS sets up page table; address translation done by hardware (MMU).
✅ Advantages of paging: No external fragmentation; efficient memory use; solves contiguous allocation problems. Most modern OS use paging.
⚠️ Drawbacks: Longer memory access time due to page table lookups; internal fragmentation possible on last page (not fully filled).
📌 Example: logical address space divided into pages 0,1,2,... ; page 0 mapped to frame 1, page 1 → frame 4, etc. 
Logical address = [page number | page offset]
Physical address = frame number × frame size + offset
🧮 Paging Address Translation – Example

Given logical address LA = 5 (4-bit address). Suppose page size = 4 bytes, page table: page 0 → frame 6, page 1 → frame 2, ...

Translation exercise: Logical address 11 (binary 1011). If page size 4 → page number = 11/4 = 2, offset = 3. Use page table mapping (example: page 2 → frame 7). Physical address = frame×4 + offset = 7×4+3=31.
Practice builds understanding of paging arithmetic.
💡 MMU splits logical address into page number (p) and offset (d). Page number indexes into page table → frame number; physical address = (frame << shift) + offset.
📌 Free Frames & Page Table Management

OS maintains a free-frame list – all frames currently unused. When a process requests memory:

  • Allocate required number of frames from free list.
  • Page table entries are set to point to the allocated frames.
  • When process terminates, frames are returned to free-frame list.
Page table size depends on logical address space. Each process has own page table; context switch requires reloading page table base register.
📊 Comparison & Important Concepts
⚡ Contiguous allocation
• Fixed/variable partitions
• External fragmentation
• Compaction possible (requires dynamic relocation)
📇 Paging
• No external fragmentation
• Internal fragmentation (last page)
• Hardware support (MMU + page table)
• Supports virtual memory foundation
🧠 Key takeaway: Execution-time address binding (via relocation register / MMU) is essential for modern OS: enables swapping, dynamic loading, and paging. Protection via base/limit or page-level protection bits.

📖 Core Summary – Main Memory
  • Base & limit registers provide simple hardware protection for logical address spaces.
  • Address binding can happen at compile-time, load-time, or execution-time. Execution-time binding requires MMU and relocation registers.
  • Dynamic loading loads routines only when called → reduces memory footprint.
  • Contiguous allocation (fixed/variable partitions) suffers from external fragmentation; compaction resolves it but needs dynamic relocation.
  • First-fit, best-fit, worst-fit are hole-selection strategies. First-fit and best-fit generally outperform worst-fit.
  • Paging eliminates external fragmentation, uses frames and pages, introduces internal fragmentation and page table overhead. Fundamental for virtual memory.
  • Fragmentation types: external (scattered holes) and internal (unused space inside allocated block).
