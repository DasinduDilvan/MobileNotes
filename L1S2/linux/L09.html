🧵 POSIX Threads (Pthreads)

System Programming Fundamentals & Linux – ICT1261  |  Lab Session

  • Understand shared memory

📖 Introduction

A thread is a lightweight unit of execution inside a process. A process can contain multiple threads that execute concurrently, sharing the same memory space while running independently — improving performance and responsiveness.

🔵 In this lab, we create four arithmetic threads:
➕ AdditionAdds two values
➖ SubtractionSubtracts two values
✖️ MultiplicationMultiplies two values
➗ DivisionDivides two values

⚙️ Important Pthread Functions

Function / TypeDescription
pthread_create()Creates a new thread
pthread_join()Waits for a thread to finish (blocks caller until thread exits)
pthread_exit()Terminates a thread from within itself
pthread_tData type used to store a thread ID
⚠️ Compilation flag required: Always compile with -pthread to link the pthread library.
gcc filename.c -o output_name -pthread

Activity 1

Creating a Simple Thread

Objective: Create one thread that performs an addition operation.

1
Create & open the file: gedit thread1.c
2
Type the program:
// thread1.c — Single addition thread
#include <stdio.h>
#include <pthread.h>

void* addition(void* arg)
{
    int a = 10;
    int b = 20;
    printf("Addition Result = %d\n", a + b);
    pthread_exit(NULL);
}

int main()
{
    pthread_t t1;
    pthread_create(&t1, NULL, addition, NULL);
    pthread_join(t1, NULL);
    printf("Main Thread Finished\n");
    return 0;
}
3
Compile: gcc thread1.c -o thread1 -pthread
4
Run: ./thread1
Expected Output Addition Result = 30
Main Thread Finished
🔵 What happened?
  • pthread_create() launched a new thread that ran the addition function.
  • pthread_join() paused the main thread until the child thread finished.
  • The main thread printed its message only after the child completed.

// thread2.c — Two concurrent threads
#include <stdio.h>

void* addition(void* arg)

    pthread_t t1, t2;

    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    printf("Main Thread Finished\n");
    return 0;
}

gcc thread2.c -o thread2 -pthread
4
./thread2
Possible Output A Addition = 70
Subtraction = 30
Main Thread Finished
Possible Output B Subtraction = 30
Addition = 70
Main Thread Finished
⚡ Observation — Concurrent Execution
  • The output order changes between runs because threads execute concurrently.
  • The CPU scheduler decides which thread runs first.
  • This unpredictable ordering is called non-deterministic / concurrent execution.

Four Arithmetic Threads

Objective: Create four separate threads — one each for addition, subtraction, multiplication, and division.

1
gedit thread3.c
2
Type the program:
// thread3.c — Four arithmetic threads
#include <stdio.h>
#include <pthread.h>

void* addition(void* arg)      { printf("Addition = %d\n",       40 + 10); pthread_exit(NULL); }
void* subtraction(void* arg)   { printf("Subtraction = %d\n",    40 - 10); pthread_exit(NULL); }
void* multiplication(void* arg){ printf("Multiplication = %d\n", 40 * 10); pthread_exit(NULL); }
void* division(void* arg)       { printf("Division = %d\n",       40 / 10); pthread_exit(NULL); }

int main()
{
    pthread_t t1, t2, t3, t4;
    pthread_create(&t1, NULL, addition, NULL);
    pthread_create(&t2, NULL, subtraction, NULL);
    pthread_create(&t3, NULL, multiplication, NULL);
    pthread_create(&t4, NULL, division, NULL);

    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    pthread_join(t3, NULL);
    pthread_join(t4, NULL);

    printf("All Threads Finished\n");
    return 0;
}

Looping Threads

Objective: Observe concurrent execution more clearly by using loops inside each thread.

1
gedit thread4.c
2
Type the program:
// thread4.c — Looping threads to observe concurrency
#include <stdio.h>
#include <pthread.h>

void* thread1(void* arg)
{
    int i;
    for(i = 1; i <= 5; i++)
        printf("Addition Thread Running %d\n", i);
    pthread_exit(NULL);
}

void* thread2(void* arg)
{
    int i;
    for(i = 1; i <= 5; i++)
        printf("Subtraction Thread Running %d\n", i);
    pthread_exit(NULL);
}

int main()
{
    pthread_t t1, t2;
    pthread_create(&t1, NULL, thread1, NULL);
    pthread_create(&t2, NULL, thread2, NULL);
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    printf("Threads Completed\n");
    return 0;
}
3
gcc thread4.c -o thread4 -pthread
4
./thread4
🔵 Observation
  • Thread outputs are mixed / interleaved — they execute simultaneously.
  • The exact interleaving changes every run.
  • This clearly demonstrates true concurrent execution.

Activity 5

Shared Variable & Race Condition

Objective: Understand how multiple threads share the same memory, and what can go wrong.

1
gedit thread5.c
2
Type the program:
// thread5.c — Shared variable / race condition demo
#include <stdio.h>
#include <pthread.h>

int counter = 0; // shared variable

void* increment(void* arg)
{
    int i;
    for(i = 0; i < 100000; i++)
    {
        counter++; // ⚠ race condition here!
    }
    pthread_exit(NULL);
}

int main()
{
    pthread_t t1, t2;
    pthread_create(&t1, NULL, increment, NULL);
    pthread_create(&t2, NULL, increment, NULL);
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    printf("Final Counter Value = %d\n", counter);
    return 0;
}
3
gcc thread5.c -o thread5 -pthread
4
Run 3–5 times and observe: ./thread5
Sample Outputs (inconsistent!) Final Counter Value = 200000   ← lucky run
Final Counter Value = 147382   ← race condition loss
Final Counter Value = 163910   ← different each time
🚨 Race Condition Detected!
  • Two threads each increment counter 100,000 times → expected result: 200,000.
  • Actual result is sometimes lower because both threads access the same memory simultaneously.
  • When Thread A reads counter before Thread B has written its update, an increment is lost.
  • This is a race condition — a classic concurrency bug.
  • Fix requires synchronization mechanisms (e.g., mutexes — covered in future labs).

📚 Key Concepts Summary

ConceptDescription
ProcessA running program with its own memory space
ThreadA lightweight execution unit within a process
Concurrent ExecutionMultiple threads executing at the same time
Shared MemoryAll threads within a process share the same memory
Race ConditionUnpredictable result when multiple threads modify shared data without synchronization
Non-deterministicExecution order cannot be predicted in advance
