🕐 Cron Job Scheduling

ICT1261 – Operating System Concepts & Applications  |  Lab 2 – Crontab Exercise

🎯 Objective

The purpose of this lab session is to familiarise with cron scheduling. A cron job is a task that is scheduled to run periodically on Unix-like systems. This is useful if you have a script that needs to run every so often — for example, a script to clean up a directory at the end of each day.

📋 Crontab Syntax

A crontab entry has 5 time fields followed by the command to run:

# <minute>  <hour>  <day_of_month>  <month>  <day_of_week>  <command>

# Field     Meaning          Allowed Values
# -------   ------------     -----------------------------------
# 1         minute           0–59
# 2         hour             0–23
# 3         day of month     1–31
# 4         month            1–12  (or names: jan, feb, …)
# 5         day of week      0–7   (0 or 7 = Sun, or use names)
Field #Field NameAllowed ValuesNotes
1minute0 – 59
2hour0 – 23
3day of month1 – 31
4month1 – 12or names (jan, feb, …)
5day of week0 – 70 or 7 = Sunday; or use names (mon, tue, …)
🔑 Special Symbols
* = any/every  |  , = list (e.g. 4,10,16)  |  / = step (e.g. */2 = every 2)

💡 Crontab Examples

Ex 1 — Run every day at 8:30 AM 
30 8 * * * /home/ict_lab/cron1.sh

Minute 30, Hour 8, every day of month, every month, every day of week.

Ex 2 — Execute 3 times (4 AM, 10 AM, 4 PM) every Wednesday and Saturday 
0 4,10,16 * * wed,sat /home/ict_lab/cron2.sh

Minute 0, Hours 4, 10, and 16 (comma-separated), every Wednesday and Saturday.

Ex 3 — Every 2nd minute 
*/2 * * * * cron3.sh

*/2 means "every 2 minutes" using the step operator.

💡 Use https://crontab.guru to easily convert schedule times into valid crontab expressions.

🛠️ Lab Steps — Setting Up Cron

1
Check if the cron package is installed 
dpkg -l cron
2
If not installed, install cron on Ubuntu 
apt-get install cron
3
Verify if the cron service is running 
systemctl status cron
4
Edit your own user's crontab file 
crontab -e
5
Select editor

When prompted, select option 2 to open with the nano editor.

6
Create a directory every 2 minutes

Option A — Direct command in crontab:

*/2 * * * * mkdir /home/ict_lab/test1

Option B — Using a script file:

*/2 * * * * /home/ict_lab/cronscript1.sh
Place the command mkdir test2 inside cronscript1.sh, then set the execution permission:
chmod +x /home/ict_lab/cronscript1.sh
7
Restart cron service after saving the crontab 
systemctl restart cron
8
Check your home directory after 1 minute

Wait approximately 1–2 minutes, then verify the directory was created.

✅ Your Task (In-Class)

📌 Main Task

Create a cron job that displays and saves the current system date every minute into a text file called date.txt inside your ict_lab directory.

# Hint: use the date command and redirect output
* * * * * date >> /home/ict_lab/date.txt

📝 Lab 2 Exercise Tasks

Task 1 — Display your name every 5 minutes

Create a cron job that displays your name in a new terminal every 5 minutes.

*/5 * * * * echo "Your Name"
Task 2 — Script file with multiple commands (ex3.sh)

Create a script file called ex3.sh and add the following tasks to it, then schedule it to run at a specific time:

  • Display your current path to the terminal — pwd
  • List the contents of the current directory — ls
#!/bin/bash
pwd
ls
# Schedule ex3.sh at a specific time, e.g., every day at 9:00 AM
0 9 * * * /home/ict_lab/ex3.sh
Remember to set execute permission: chmod +x /home/ict_lab/ex3.sh
Task 3 — Save running process details at minute 1 past every hour

Create a crontab task to find and save details of currently running processes with their users using ps -au, appended into a file called proDetails.txt. Update at minute 1 past every hour.

1 * * * * ps -au >> /home/ict_lab/proDetails.txt
⚠️ Use >> (append) not > (overwrite) — this adds new process snapshots to the file each hour without losing previous data.
