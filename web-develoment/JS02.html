🗂️ JavaScript Arrays

What Is an Array?

An array is a collection of individual data values that can be counted in order — first, second, third, and so on. Arrays allow multiple values to be stored under a single variable name, accessed by their position.

Key Terminology

ElementEach individual value stored inside an array is called an element.
LengthThe total number of elements in an array. Accessed via the .length property.
IndexThe position number of an element within the array. Always starts at 0.
Zero-Based IndexingIndex numbers always begin at 0 and go up to length − 1.

Two Ways to Create an Array

Square Bracket Literal (Preferred) const arr = [1, 2, 3];
Direct, concise syntax. Most common approach.
Array Constructor const arr = new Array(1, 2, 3);
Less common. Used for special cases like pre-allocating size.

Non-Numeric Arrays

  • Arrays may contain values of any JavaScript type — not just numbers
  • Strings, booleans, objects, other arrays, and mixed types are all valid
  • Example: const COIN_NAMES = ["penny", "nickel", "dime", "quarter"]

🔧 Array Methods — Add, Remove & Transform

Adding & Removing Elements

push( element, ... )Add to END
Adds one or more elements to the end of the array. The array length increases. Returns the new length.
unshift( element, ... )Add to FRONT
Adds one or more elements to the front (beginning) of the array. All existing elements shift to higher indices. Returns the new length.
pop()Remove from END
Removes and returns the last element of the array. The array length decreases by 1.
shift()Remove from FRONT
Removes and returns the first element of the array. All remaining elements shift to lower indices. The array length decreases by 1.
slice( start, finish )Extract subarray
Returns a new subarray containing elements from index start up to (but not including) index finish. The original array is not modified.

Transforming Elements

map( callback )Transform — returns new array
Definition: Returns a new array where each element is the result of applying a given function to each element of the original array. The original array is not changed.
  • Good for transforming or converting data
  • The new array has the same length as the original
  • Does not modify the original array
filter( callback )Select — returns new array
Definition: Returns a new array containing only the elements that pass a given condition (the callback returns true for them). The original array is not changed.
  • Good for selecting a subset of data based on a condition
  • The new array may be shorter than the original
  • Elements that fail the condition are excluded

Testing Methods

some( callback )At least one?
Returns true if at least one element in the array passes the test defined by the callback function. Returns false if no elements pass. Stops as soon as one passes.
every( callback )All pass?
Returns true if all elements in the array pass the test defined by the callback function. Returns false as soon as any element fails. Stops as soon as one fails.

Key Differences — map() vs filter()

map()
  • ▸ Transforms every element
  • ▸ Returns array of same length
  • ▸ Each element is converted to a new value
  • ▸ Use for: doubling numbers, converting units
VS
filter()
  • ▸ Selects elements that pass a test
  • ▸ Returns array of equal or shorter length
  • ▸ Elements are kept or removed
  • ▸ Use for: finding even numbers, filtering adults

some() vs every()

MethodReturns true when...Stops early when...
some()At least one element passesFirst passing element found
every()All elements passFirst failing element found

🔁 Array Iteration Methods

Three Ways to Iterate an Array

MethodSyntax FormKey Characteristic
Classic for loop for (let i = 0; i < arr.length; i++) Full control over index. Can break/continue. Access by index.
for...of loop for (let item of array) Cleaner syntax. Access to value directly. Can break/continue.
forEach() method array.forEach(item => { }) Functional style. Cannot use break/continue. Always runs to end.

forEach() — Definition & Key Points

Definition: forEach() is an array method that executes a provided callback function once for each element in the array, in order. It is the functional alternative to a for loop for simple iteration.
  • The callback receives the element and optionally the index
  • Cannot use break or continue to exit early
  • Does not return a new array — use map() if you need a return value
  • Best for side effects like printing, logging, or updating external variables
Syntax: array.forEach((element, index) => { // code });

🔧 Functions — Definition & Declaration

What Is a Function?

A function is an action or process that receives an input and returns an output. Functions can perform various tasks and calculations. They are named, reusable blocks of code defined once and called (executed) as many times as needed.

Why Use Functions?

  • Reusability — write once, call many times
  • Abstraction — hide complex logic behind a simple name
  • Organisation — break large programs into manageable pieces
  • Maintainability — fix or update logic in one place

Ways to Define a Function

Function DeclarationStandard named function. Hoisted. Most common for beginners.
Function ExpressionFunction assigned to a variable. NOT hoisted. Anonymous or named.
Arrow FunctionCompact syntax using =>. Not hoisted. Often used with array methods.
IIFEImmediately Invoked Function Expression — runs as soon as it is defined.
Generator FunctionAdvanced — can pause and resume execution.
MethodA function defined inside an object.

Function Declaration — Key Points

Function Declaration (function definition / function statement)
  • Created using the function keyword followed by a required name
  • This is the most standard, basic way of writing a function
  • Hoisted — can be called before the function definition appears in the code
  • Defines a named function that exists in the current scope
Syntax:
function functionName(param1, param2) {
  // statements
}

Call: functionName(arg1, arg2);

Hoisting — Declaration vs Expression

Function Declaration — Hoisted ✅
  • ▸ Definition is moved to the top of scope before execution
  • ▸ Can be called before it appears in the file
  • ▸ hello(); works even if defined below
VS
Function Expression — NOT Hoisted ❌
  • ▸ The variable is hoisted but not initialized
  • ▸ Calling it before its definition causes an error
  • ▸ Must be defined before it is called

📨 Parameters vs Arguments & Return Statement

Function Parameter — Definition

A function parameter is a name included in the function definition. It acts as a placeholder for a value the function will receive when called.

Parameter Rules

  • Parameter names must be unique — no two parameters can share the same name
  • Parameters are not required — a function can have zero parameters
  • Order matters — arguments are matched to parameters by position (left to right)
  • You do not declare the data type of a parameter — JavaScript is dynamically typed
  • Parameters are not declared with var, let, or const

Function Argument — Definition

A function argument is the actual value that is passed to the function when it is called. The argument value is received by the corresponding parameter inside the function.

Parameter vs Argument — Key Difference

Parameter
  • ▸ In the function definition
  • ▸ Acts as a placeholder / variable name
  • ▸ Example: function greet(name)
VS
Argument
  • ▸ In the function call
  • ▸ The actual value being passed
  • ▸ Example: greet("Alice")

Return Statement — Key Rules

A return statement returns one specific value and immediately stops function execution. Any code written after the return statement will never execute.
  • The return statement should always be the last statement in a function
  • The returned value can be of any data type — number, string, boolean, array, object, etc.
  • A function with no return statement implicitly returns undefined
  • A function can only return one value — but that value can be an array or object containing multiple values
Key Rule: Code placed after a return statement is dead code — it will never run. Always put return last.

📝 Function Expressions

What Is a Function Expression?

A function expression is a function that is assigned to a variable. Unlike a function declaration, function expressions are NOT hoisted — they cannot be called before the line where they are defined.

Two Types of Function Expressions

1. Anonymous Function Expression
  • A function expression without a name
  • This is the most common type of function expression
  • The function is identified only by the variable it is stored in
  • Cannot call itself by name (no recursion with its own name)
Syntax:
const greet = function () {
  console.log("Hello!");
};
greet(); // → Hello!
2. Named Function Expression
  • A function expression that has its own internal name, even though it is stored in a variable
  • The function's internal name is only accessible inside the function itself — not outside
  • Calling the internal name from outside causes a ReferenceError
Syntax:
const greet = function sayHello() {
  console.log("Hello!");
};
greet(); // → Hello! ✅
sayHello(); // → ReferenceError ❌

Why Use a Named Function Expression?

🐛 Better Debugging If an error occurs, the function's name appears in the stack trace, making it easier to identify which function caused the issue.
🔁 Self-Reference (Recursion) The internal name allows the function to call itself from within its own body — useful for recursive algorithms without relying on the external variable name.

Key Definitions Glossary

Array
A collection of individual data values stored in order, each identified by a numeric index starting from 0. Arrays can hold any JavaScript type and are created using square bracket notation.
Element
An individual value stored inside an array. Each element has a position (index) that identifies it within the array.
Index
The positional number of an element in an array. Always begins at 0 for the first element and extends to length − 1 for the last.
forEach()
An array method that executes a callback function once for each element in order. Returns undefined. Cannot be broken out of early. Used for side effects.
map()
An array method that returns a new array by applying a transformation function to every element. Does not modify the original. The new array has the same length.
filter()
An array method that returns a new array containing only elements for which the callback returns true. Does not modify the original. The result may be shorter than the original.
Function Declaration
A function defined with the function keyword and a required name. It is hoisted — meaning it can be called anywhere in its scope, even before the line where it is defined.
Function Expression
A function assigned to a variable. It is NOT hoisted — it must be defined before being called. Can be anonymous (no name) or named.
Anonymous Function
A function with no name, typically used in function expressions. The function can only be referenced through the variable it is assigned to. Most common type of function expression.
Named Function Expression
A function expression that has an internal name. The name is only accessible inside the function body — useful for debugging (stack traces) and recursion. Calling the name from outside causes a ReferenceError.
Hoisting
JavaScript's behaviour of moving function declarations to the top of their scope before execution. Function expressions are NOT hoisted. Avoid relying on hoisting.
Return Statement
A statement that sends a value back to the caller and immediately stops function execution. Any code after return is never executed. A function without return implicitly returns undefined.
