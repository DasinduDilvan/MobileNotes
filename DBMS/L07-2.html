1. Binary Relational Operations: JOIN (⋈)

Definition: JOIN generates a relation that contains all tuples that are a combination of two tuples (one from each input relation) with a common value(s) for one or more attributes.

The sequence of CARTESIAN PRODUCT followed by SELECT is used quite commonly to identify and select related tuples from two relations. The JOIN operation combines this sequence into a single operation.

JOIN is very important for any relational database with more than a single relation, because it allows combining related tuples from various relations.

General Form

For two relations R(A1, A2, …, An) and S(B1, B2, …, Bm):

R ⋈ <join condition> S

where R and S can be any relations that result from general relational algebra expressions.

Example: Retrieving the Manager of Each Department
  • We need to combine each DEPARTMENT tuple with the EMPLOYEE tuple whose SSN value matches the MGRSSN value in the department tuple.
  • Join expression: DEPT_MGR ← DEPARTMENT ⋈ MGRSSN=SSN EMPLOYEE
  • The join condition is MGRSSN = SSN — combines each department record with the employee who manages it.
  • Can also be written as: DEPARTMENT.MGRSSN = EMPLOYEE.SSN

2. Properties of JOIN

Result Structure

For R(A1, A2, …, An) ⋈R.Ai=S.Bj S(B1, B2, …, Bm):

  • Result is a relation Q with degree n + m attributes: Q(A1, A2, …, An, B1, B2, …, Bm) in that order.
  • The resulting relation has one tuple for each combination of tuples r from R and s from S, but only if they satisfy the join condition r[Ai] = s[Bj].
  • If R has nR tuples and S has nS tuples, the join result will generally have less than nR × nS tuples.
  • Only related tuples (based on the join condition) will appear in the result.
Theta-Join (General Case)
The general case of JOIN is called a Theta-join: R ⋈theta S. The join condition is called theta and can be any general boolean expression on the attributes of R and S.

Example of a Theta condition:

R.Ai < S.Bj AND (R.Ak = S.Bl OR R.Ap < S.Bq)

Most join conditions involve one or more equality conditions "AND" together:

R.Ai = S.Bj AND R.Ak = S.Bl AND R.Ap = S.Bq

3. EQUIJOIN

Definition: A join where the only comparison operator used is = (equality comparisons only) is called an EQUIJOIN.
  • The most common type of join in practice.
  • In the result of an EQUIJOIN, there are always one or more pairs of attributes (whose names need not be identical) that have identical values in every tuple.
  • The DEPT_MGR example from above is an EQUIJOIN (MGRSSN = SSN).

4. NATURAL JOIN (*)

Definition: NATURAL JOIN — denoted by * — is a variation of EQUIJOIN that eliminates the second (superfluous) attribute in an EQUIJOIN condition, because one of each pair of attributes with identical values is redundant.
  • The standard definition requires that the two join attributes (or each pair of corresponding join attributes) have the same name in both relations.
  • If the names are not the same, a renaming operation is applied first.
Example 1: DEPARTMENT * DEPT_LOCATIONS
DEPT_LOCS ← DEPARTMENT * DEPT_LOCATIONS
  • The only shared attribute name is DNUMBER.
  • An implicit join condition is created: DEPARTMENT.DNUMBER = DEPT_LOCATIONS.DNUMBER
Example 2: Q ← R(A, B, C, D) * S(C, D, E)
  • Implicit join condition (each pair of same-named attributes, AND-ed together): R.C = S.C AND R.D = S.D
  • Result keeps only one attribute of each matched pair: Q(A, B, C, D, E)

5. Complete Set of Relational Operations

The set of operations SELECT (σ), PROJECT (π), UNION (∪), DIFFERENCE (−), RENAME (ρ), and CARTESIAN PRODUCT (×) is called a complete set because any other relational algebra expression can be expressed by a combination of these operations.

Examples of expressing other operations using the complete set:

R ∩ S = (R ∪ S) − ((R − S) ∪ (S − R))
R ⋈ <join condition> S = σ <join condition> (R × S)

6. Binary Relational Operations: DIVISION (÷)

Definition: The DIVISION operation is applied to two relations R(Z) ÷ S(X), where X is a subset of Z. Let Y = Z − X (so Z = X ∪ Y), meaning Y is the set of attributes of R that are not attributes of S.
  • The result of DIVISION is a relation T(Y) that includes a tuple t if tuples tR appear in R with tR[Y] = t, and with tR[X] = ts for every tuple ts in S.
  • For a tuple t to appear in the result T, the values in t must appear in R in combination with every tuple in S.
Example (a): SSN_PNOS ÷ SMITH_PNOS

SMITH_PNOS contains Pno values {1, 2}. The result SSNS contains only those SSNs that appear in SSN_PNOS for both Pno = 1 and Pno = 2:

SMITH_PNOS (divisor) Pno: 1, 2
SSNS (result) SSN: 123456789, 453453453
Example (b): T ← R ÷ S

R(A, B) divided by S(A) where S contains {a1, a2, a3}. Result T contains only B values that are paired with all values in S:

S (divisor) A: a1, a2, a3
T (result) B: b1, b4

b1 and b4 appear in R paired with a1, a2, and a3 — all values in S. b2 and b3 do not appear with all values of S, so they are excluded.

8. Additional Operations: Aggregate Functions & Grouping (ℱ)

Problem: Mathematical aggregate functions on collections of values from the database cannot be expressed in the basic relational algebra. Examples include the average or total salary of all employees, or the total number of employee tuples.

These functions are used in simple statistical queries that summarize information from the database tuples.

Common Aggregate Functions
SUMComputes the sum of numeric values in a column.
AVERAGEComputes the mean value of a numeric column.
MAXIMUMRetrieves the highest value in a column.
MINIMUMRetrieves the lowest value in a column.
COUNTCounts tuples or values. Does not remove duplicates.
Aggregate Functional Operation (ℱ) — Examples
ExpressionMeaning
ℱMAX Salary(EMPLOYEE)Retrieves the maximum salary value from EMPLOYEE.
ℱMIN Salary(EMPLOYEE)Retrieves the minimum salary value from EMPLOYEE.
ℱSUM Salary(EMPLOYEE)Retrieves the sum of all salaries in EMPLOYEE.
ℱCOUNT SSN, AVERAGE Salary(EMPLOYEE)Counts the number of employees and computes their average salary.
Note: COUNT counts the number of rows without removing duplicates.

9. Using Grouping with Aggregation

Grouping can be combined with Aggregate Functions to summarize data per group, rather than over the entire relation.

Syntax
<grouping attribute(s)> ℱ <aggregate function(s)> (relation)
  • Grouping attribute is placed to the left of the ℱ symbol.
  • Aggregate functions are placed to the right of the ℱ symbol.
Example: Group employees by department and compute count & average salary
DNO ℱCOUNT SSN, AVERAGE Salary(EMPLOYEE)

This groups all EMPLOYEE tuples by DNO (department number) and computes the count of employees and average salary per department.

10. Additional Operations: Recursive Closure

Limitation: Recursive closure operations cannot be specified in the basic original relational algebra.

Recursive closure is applied to a recursive relationship. An example is to retrieve all SUPERVISEES of an EMPLOYEE e at all levels:

  • All employees e' directly supervised by e
  • All employees e'' directly supervised by each e'
  • All employees e''' directly supervised by each e''
  • … and so on

Although it is possible to retrieve employees at each level and then take their union, we cannot, in general, specify a query such as "retrieve the supervisees of 'James Borg' at all levels" without utilizing a looping mechanism.

The SQL3 standard includes syntax for recursive closure.
Two-Level Recursive Query Example (Borg's SSN = 888665555)
RESULT1 (Supervised directly by Borg)
SSN: 333445555, 987654321
RESULT2 (Supervised by Borg's subordinates)
SSN: 123456789, 999887777, 666884444, 453453453, 987987987
RESULT = RESULT1 ∪ RESULT2
SSN: 123456789, 999887777, 666884444, 453453453, 987987987, 333445555, 987654321

11. Additional Operations: OUTER JOIN

Problem with NATURAL JOIN / EQUIJOIN: Tuples without a matching tuple are eliminated from the join result. Tuples with null in the join attributes are also eliminated. This results in loss of information.

OUTER JOIN operations keep all tuples in R, or all those in S, or all those in both relations, regardless of whether they have matching tuples in the other relation.

Three Types of OUTER JOIN
LEFT OUTER JOIN (R ⟕ S) Keeps every tuple in the left relation R. If no matching tuple is found in S, the attributes of S are padded with NULL values.
RIGHT OUTER JOIN (R ⟖ S) Keeps every tuple in the right relation S. If no matching tuple is found in R, the attributes of R are padded with NULL values.
FULL OUTER JOIN (R ⟗ S) Keeps all tuples in both left and right relations. When no matching tuples are found, missing attributes are padded with NULL values.

12. Additional Operations: OUTER UNION

Definition: The OUTER UNION operation takes the union of tuples from two relations even if they are not type compatible (i.e., do not have exactly the same attributes).
  • Applies to two relations R(X, Y) and S(X, Z) that are partially compatible — only some of their attributes (say X) are type compatible.
  • Attributes that are type compatible are represented only once in the result.
  • Attributes that are not type compatible from either relation are kept in the result.
  • Result relation: T(X, Y, Z)
Example: STUDENT and INSTRUCTOR
STUDENT (Name, SSN, Department, Advisor)
INSTRUCTOR (Name, SSN, Department, Rank)
  • Tuples are matched on the shared attributes: Name, SSN, Department.
  • If a person is both a student and instructor, both Advisor and Rank will have values.
  • Otherwise, one of Advisor or Rank will be NULL.
Result Relation: STUDENT_OR_INSTRUCTOR (Name, SSN, Department, Advisor, Rank)
