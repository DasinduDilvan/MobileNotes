💽 Disk Storage & File Structures

ICT1212 — Database Management Systems  |  Lecture 10

1. Classification of Physical Storage Media

Storage media are evaluated along three key dimensions:

SpeedHow quickly data can be accessed.
CostPrice per unit of stored data.
ReliabilityRisk of data loss from power failure or physical device failure.
⚡ Volatile Storage
Loses its contents when power is switched off.
Examples: Cache, Main Memory
🔒 Non-Volatile Storage
Contents persist even when power is off.
Includes secondary & tertiary storage, and battery-backed main memory.

2. Storage Hierarchy

Storage is arranged in a hierarchy — faster and more expensive at the top, slower and cheaper at the bottom.

Cache
▲ Primary (Volatile, Fastest)
Main Memory
▲ Secondary (Non-Volatile, On-line)
Flash Memory
Magnetic Disk
▲ Tertiary (Non-Volatile, Off-line)
Optical Disk
Magnetic Tapes
Primary Storage Fastest but volatile. Includes cache and main memory.
Secondary Storage Non-volatile, moderately fast, on-line. Flash memory and magnetic disks.
Tertiary Storage Non-volatile, slow access, off-line. Magnetic tape and optical storage.

3. Physical Storage Media — Types & Characteristics

🗄 Cache Fastest and most costly form of storage. Volatile. Managed entirely by computer hardware.
🧠 Main Memory Fast access. Volatile. Generally too small or too expensive to store an entire database.
⚡ Flash Memory Non-volatile. Reads ~as fast as main memory; writes are slow (erase is even slower). Used widely in embedded devices (cameras, phones). NAND flash is cheaper and used for SSDs; NOR flash is more expensive.
💿 Magnetic Disk Primary medium for long-term database storage. Non-volatile. Direct-access (any order). Much slower than main memory. Data moved to RAM for processing, then written back.
📀 Optical Storage Non-volatile. Data read optically via laser. Includes CD-ROM, DVD, Blu-ray. Read/write speeds slower than magnetic disk. CD-R, DVD-R used for archival; CD-RW, DVD-RW are rewritable.
📼 Tape Storage Non-volatile. Sequential access only — much slower than disk. Very high capacity. Storage cost is cheaper than disk, but tape drives are expensive. Tape jukeboxes used for massive archival data.

4. Magnetic Disks

4.1 Physical Structure

Read-Write HeadPositioned very close to the platter surface. Reads or writes magnetically encoded information.
Platter SurfaceDivided into circular tracks. Each track is further divided into sectors.
SectorThe smallest unit of data that can be read or written on a disk.
Head-Disk AssemblyMultiple platters on a single spindle. One read-write head per platter, mounted on a common arm.
Cylinder iConsists of the i-th track of all platters stacked vertically — same position across all platters.

4.2 Disk Controller

The disk controller acts as the interface between the computer system and the disk drive hardware. It:
  • Computes and attaches checksums to each sector to verify correct data retrieval.
  • Ensures successful writes by reading back each sector after writing.
  • Performs remapping of bad sectors to spare sectors.

Earlier generation disks were susceptible to head crashes (head physically touching the platter).

4.3 Disk Interface Standards

ATA / PATAAT Adaptor range; older parallel interface.
SATASerial ATA; modern successor to PATA with card-edge connector.
SCSISmall Computer System Interconnect; used in servers.
SASSerial Attached SCSI; high-performance server standard.

4.4 Disk Connectivity Types

DASDirectly Attached Storage — disk directly connected to the computer.
NASNetwork Attached Storage — disk accessible over a network.
SANStorage Area Network — dedicated high-speed network for storage.

5. Performance Measures of Disks

Access Time Time from when a read/write request is issued to when data transfer begins.
= Seek Time + Rotational Latency
Seek Time Time for the disk arm to reposition over the correct track.
Rotational Latency Time for the desired sector to rotate under the read-write head.
Data-Transfer Rate Rate at which data can be retrieved from or stored to the disk.
MTTF (Mean Time to Failure) Average time a disk runs continuously without failure. Typically 3–5 years. Failure probability is low for new disks but increases with age.

6. Optimization of Disk-Block Access

6.1 Block

A block is a contiguous sequence of sectors from a single track. Data is transferred between disk and main memory in blocks. Typical block sizes range from 4 to 16 kilobytes.
  • Smaller blocks → more transfers from disk (slower).
  • Larger blocks → more wasted space in partially filled blocks.

6.2 Disk-Arm Scheduling

Algorithms that order pending accesses to tracks so that disk arm movement is minimized, improving throughput.

6.3 File Organization for Optimization

  • Optimize block access time by organizing blocks to match data access patterns.
  • Store related data on the same or nearby cylinders.
  • Files may become fragmented over time (due to insertions, deletions, or scattered free blocks), causing increased disk arm movement.
  • Some systems provide defragmentation utilities to consolidate files and speed up access.

6.4 Nonvolatile Write Buffers

Speeds up disk writes by immediately writing blocks to a non-volatile RAM buffer (battery-backed RAM or flash memory). Writes can then be reordered to minimize disk arm movement.

6.5 Log Disk

A disk dedicated to writing a sequential log of block updates. Used like nonvolatile RAM. Write to log disk is very fast (no seeks required). No special hardware needed (unlike NV-RAM). File systems typically reorder disk writes to improve overall performance.

7. Flash Storage

NAND Flash Used widely for storage — much cheaper than NOR flash. Erase is very slow. Solid State Disks (SSDs) use multiple NAND flash devices in parallel to achieve transfer rates of 100–200 MB/sec.
NOR Flash More expensive than NAND. Better for code execution (random read access). Less common for bulk storage.

8. RAID Technology

RAID = Redundant Arrays of Independent Disks
Disk organization techniques that manage large numbers of disks, providing a view of a single large, fast, and reliable disk. The "I" originally stood for inexpensive but today is interpreted as independent.
  • High capacity & speed — multiple disks used in parallel.
  • High reliability — data stored redundantly; can be recovered even if a disk fails.

8.1 RAID Levels

Level Name Description Usage
RAID 0 Striped Disk Array (No Fault Tolerance) Data striped across disks. No redundancy. Only when data safety is not important.
RAID 1 Mirroring & Duplexing Each disk has an exact copy (mirror). High reliability. Widely used; good for high-availability.
RAID 2 Error-Correcting Coding Memory-style ECC applied across disks. Never used — subsumed by RAID 3.
RAID 3 Bit-Interleaved Parity Parity stored on a dedicated disk. Not used anymore.
RAID 4 Dedicated Parity Drive Block-interleaved parity on one dedicated disk. Never used — subsumed by RAID 5.
RAID 5 Block-Interleaved Distributed Parity Parity distributed across all disks. No single bottleneck. Most commonly used — offers good balance.
RAID 6 P + Q Redundancy (Double Parity) Two independent parity blocks; survives two disk failures. Rarely used; RAID 1 & 5 sufficient for most.

8.2 Choosing a RAID Level

Factors to consider: monetary cost, normal performance, performance during failure, and performance during rebuild of a failed disk.

8.3 RAID Hardware Issues

Software RAID RAID implemented entirely in software with no special hardware support.
Hardware RAID Uses special hardware. Employs non-volatile RAM to record ongoing writes. Power failure during write can corrupt the disk.
Latent Failures Previously written data gets silently damaged. Can cause data loss even if only one disk fails.
Data Scrubbing Continuously scans disks for latent failures and recovers from parity/copy.
Hot Swapping Replacing a failed disk while the system is running, without powering down.
Spare Disks Many systems keep spare disks online, ready to replace failed disks immediately upon detection.

9. Storage Access

A database file is partitioned into fixed-length storage units called blocks. Blocks are the units of both storage allocation and data transfer.

Buffer A portion of main memory available to store copies of disk blocks. Reduces disk accesses by keeping frequently used blocks in memory.
Buffer Manager The subsystem responsible for allocating buffer space in main memory. The database system aims to minimize the number of block transfers between disk and memory.
Blocking Concepts:
  • Blocking — storing multiple records in one disk block.
  • Blocking Factor (bfr) — number of records per block.
  • If records do not evenly fill a block, empty space may remain.
  • Spanned Records — records that exceed a block size and span multiple blocks.

10. File Organization

A database is stored as a collection of files. Each file is a sequence of records. A record is a sequence of fields.

Common approach: assume fixed record size; each file holds records of one relation only; different files for different relations.

10.1 Fixed-Length Records

All records have the same fixed size. Stored sequentially. Simple to implement. When a record is deleted, two approaches are used:
  • Compact (shift up) — move all subsequent records up to fill the gap. Expensive.
  • Move last record — copy the last record into the deleted slot. Faster.
  • Free List — store a linked list of free (deleted) slots in the file header. New records are inserted into the first free slot. Most efficient approach.

10.2 Variable-Length Records

Variable-length records arise when:
  • Multiple record types are stored in one file.
  • Fields have variable lengths (e.g., VARCHAR).
  • Record types allow repeating fields.

Representation: Attributes stored in order. Variable-length attributes represented by a fixed-size (offset, length) pair, with actual data appended after all fixed-length attributes. Null values are indicated using a null-value bitmap.

10.3 Slotted Page Structure (for Variable-Length Records)

Slotted page header contains:
  • Number of record entries.
  • End of free space in the block.
  • Location and size of each record.

Records can be moved within a page to keep them contiguous (no gaps); the header entry is updated accordingly. Pointers should point to the header entry, not directly to the record, allowing records to be relocated without breaking references.

11. Organization of Records in Files

Heap Organization A record can be placed anywhere in the file where there is free space. No ordering is maintained.
Sequential Organization Records stored in sequential order based on the value of a search key. Efficient for range queries.
Hashing Organization A hash function is applied to an attribute of each record. The result specifies which block of the file the record should be placed in.

12. Data Dictionary Storage

The data dictionary (also called the system catalog) stores metadata — data about data.
Relation Metadata
  • Names of relations
  • Names, types, and lengths of attributes
  • Names and definitions of views
  • Integrity constraints
User & Security Metadata
  • User account information
  • Encrypted passwords
  • Groups and permissions
Statistical Metadata
  • Number of tuples in each relation
  • Descriptive data about the database
Physical Organization Metadata
  • How each relation is stored (sequential/hash/…)
  • Physical location of each relation
  • Information about indices

12.1 Relational Representation of System Metadata

The data dictionary is itself stored as a set of relations (tables), enabling consistent access through the same query mechanisms used for user data. Key metadata relations include:
Relation_metadata relation_name, number_of_attributes, storage_organization, location
Attribute_metadata relation_name, attribute_name, domain_type, position, length
Index_metadata index_name, relation_name, index_type, index_attributes
View_metadata view_name, definition
User_metadata user_name, encrypted_password, group
