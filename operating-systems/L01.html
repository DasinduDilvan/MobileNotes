What is an Operating System?

  • Intermediary between user and hardware
  • Executes user programs and makes solving user problems easier
  • Makes the computer system convenient to use
  • Uses computer hardware in an efficient manner
  • Improves system reliability
  • Provides security and isolation between processes

Computer System Components

Hardware CPU, Memory, I/O devices - provides basic computing resources
Operating System Controls and coordinates hardware use among applications
Application Programs Word processors, compilers, web browsers, games
Users People, machines, other computers

What Operating System Does

  • Resource Allocator: Manages all resources (CPU time, memory space, file storage, I/O devices)
  • Control Program: Controls execution of programs to prevent errors and improper use
  • Manages CPU, memory, storage, and I/O devices
  • Optimized differently for:
    • Personal computers (PCs)
    • Servers
    • Mobile devices
    • Embedded systems

Operating System Definition

No universally accepted definition
  • Kernel: The one program running at all times on the computer
  • System Programs: Ship with the operating system but are not part of the kernel
  • Application Programs: All programs not associated with the operating system
  • Without an operating system, a computer is just useless hardware

Computer Startup Process

  • Bootstrap Program is loaded at power-up or reboot
  • Stored in ROM or EPROM, generally known as firmware
  • Initializes all aspects of the system
  • Loads the operating system kernel into memory
  • Kernel loads and starts the initial process
  • System is then running and ready to accept user input

Interrupts

  • Interrupts are signals from hardware or software
  • Hardware interrupt: Triggered by hardware devices via system bus
  • Software interrupt (trap): Triggered by software via system call
  • CPU stops current execution and transfers to interrupt service routine
  • CPU saves its current state before handling interrupt
  • Interrupt vector: Contains addresses of interrupt service routines
  • Operating system is interrupt driven - waits for interrupts to occur

Direct Memory Access (DMA)

  • Used for high-speed I/O devices
  • Device controller transfers data directly to/from memory
  • No CPU intervention required for data transfer
  • CPU is free to perform other tasks during transfer
  • Only one interrupt is generated per block of data
  • Improves system performance significantly

  • Non-volatile Memory: Retains contents without power (ROM, hard disk, SSD)

Caching

  • Copying information into faster storage system
  • Improves access speed for frequently used data
  • Cache memory is located inside or very close to CPU
  • CPU first checks cache before accessing main memory
  • Significantly reduces average access time
  • Based on principle of locality of reference

Computer System Architecture

  • Single Processor Systems: One main CPU executing instructions
  • Multiprocessor Systems: Two or more processors sharing memory
  • SMP (Symmetric Multiprocessing): All processors are peers, no master-slave relationship
  • AMP (Asymmetric Multiprocessing): Master-slave relationship among processors
  • Advantages of multiprocessing:
    • Increased throughput
    • Economy of scale
    • Increased reliability and fault tolerance

‚Ä¢ CPU switches between jobs
‚Ä¢ Increases CPU utilization

‚Ä¢ Response time < 1 second

  • Organizes jobs so CPU always has one to execute
  • Job scheduling: Selects which job to run next
  • Virtual memory: Allows execution of processes not completely in memory
  • Enables multiple users to share computer simultaneously

Dual Mode Operation

  • User Mode: Mode in which user applications execute
  • Kernel Mode (Supervisor/System Mode): Mode for OS kernel execution
  • Mode bit: Hardware feature that indicates current mode (0 = kernel, 1 = user)
  • Privileged instructions: Can only be executed in kernel mode
  • System calls cause mode switch from user to kernel mode
  • Protects OS and critical system resources from user programs
  • After system call completion, returns to user mode

Process Management

  • Creating and deleting processes
  • Suspending and resuming processes
  • Process synchronization mechanisms
  • Process communication mechanisms
  • Deadlock handling - prevention, avoidance, detection, recovery
  • CPU scheduling to maximize efficiency

Memory Management

  • Tracking which parts of memory are being used and by whom
  • Allocating and deallocating memory space as needed
  • Moving data between main memory and secondary storage
  • Deciding which processes to load into memory
  • Managing virtual memory systems
  • Optimizing memory usage for performance

File Management

  • Creating and deleting files
  • Creating and deleting directories
  • Supporting primitives for manipulating files and directories
  • Mapping files onto secondary storage
  • Backup of files on stable (non-volatile) storage media
  • Access control and permissions

Mass Storage Management

  • Disk scheduling: Ordering disk I/O requests for efficiency
  • Partitioning: Dividing storage into logical units
  • Protection: Ensuring data integrity and security
  • Free-space management: Tracking available storage space
  • Storage allocation and deallocation
  • Disk formatting and bad-block management

I/O Subsystem

  • Device drivers: Software interface for hardware devices
  • Buffering: Temporary storage while transferring data
  • Caching: Storing frequently accessed data
  • Spooling: Overlapping output of one job with input of other jobs
  • Managing I/O queues and scheduling
  • Handling device interrupts

Protection & Security

  • User IDs: Unique identifier for each user
  • Group IDs: Set of users sharing access rights
  • Access control: Managing permissions for files and resources
  • Defense against attacks: External (unauthorized access) and internal (malicious users)
  • Privilege escalation: Gaining higher access rights when needed
  • Authentication and authorization mechanisms
  • Encryption and secure communication
