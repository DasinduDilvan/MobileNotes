üêö Introduction to the Shell

A shell is a program that starts after user authorization and handles the user's interaction with the system.

Shell Execution Cycle

1
Accept the Input (from user)
2
Interpret the Input
3
Execute the Input (via Kernel)
4
Interpret the Output
5
Wait for Next Input
Architecture: Each user has their own shell. The shell sits between the user and the Kernel. The Kernel communicates directly with the Computer Hardware.

Shell Types

Shell NamePromptOriginRemarks

CSH ‚Äî C Shell % University of California (BSD) Syntax similar to the C language
BASH ‚Äî Bourne-Again Shell $ Free Software Foundation Most common shell in Linux. Focus of this course.
Each shell has its own programming language. This course focuses on BASH.

üìú Shell Scripts

What is a Shell Script? A program/file composed of shell commands ‚Äî a sequence of commands that can be read/written by a text editor and developed using programming techniques.
Inputs to Shell
  • A Command ‚Äî utility coming with the OS to do a particular task
  • A Program ‚Äî a sequence of commands (shell script)

Why Write Shell Scripts? (Use of Shells)

  • Automate administrative tasks
  • Execute complex operations efficiently
  • Perform repetitive tasks
  • Maximize the power of the command line
  • Manage systems processed during startup and shutdown
  • Execute portable shell scripts across different UNIX systems
  • Enable users to modify their environment

‚ñ∂Ô∏è Command Execution

Utilities Programs such as who and date that we can run or execute. UNIX commands invoke utility programs stored in the directory hierarchy (e.g., ls ‚Üí executable in /bin).
Commands The name of a program and any arguments specified to change its behavior. Shell also contains built-in commands it recognizes and executes internally (e.g., echo, cd).

Command Syntax

# Command with arguments
$ find / -name bash ‚Äìprint

# Multiple commands on one line separated by ;
$ whoami;date;pwd

Command Grouping

Commands can be grouped by placing them between parentheses. A group of commands shares the same stdin, stdout, and stderr.

# Without grouping ‚Äî only pwd output goes to file.txt
$ date; ls; pwd > file.txt

# With grouping ‚Äî ALL outputs go to file.txt
$ (date; ls; pwd) > file.txt

Back Quote / Grave Operator ( ` ` )

Anything enclosed in back quotes is executed as a command. The output replaces the back-quoted expression. 
# Embed command output in echo
$ echo Today is `date`
Today is Tue Jan 16 09:45:36 LKT 2017

# Assign command output to a variable
$ myvariable=`ls -l`
$ echo $myvariable

‚ÜîÔ∏è I/O Redirection

stdin (Standard Input) Input from keyboard by default. File Descriptor: 0
stdout (Standard Output) Output sent to monitor by default. File Descriptor: 1
stderr (Standard Error) Error messages sent to screen by default. File Descriptor: 2
In Unix, everything is a file. Every file has a number called a file descriptor. These three files (stdin, stdout, stderr) are always open when a program runs.

Redirection Operators

OperatorMeaningExample
> or 1>Redirect stdout to a file (overwrites)ls -al > allfiles.txt
<Redirect stdin from a file instead of keyboardsort < aa.txt
>> or 1>>Append stdout to a filedate >> usrdata
2>Redirect stderr to a filels /root/ 2> error.log
2>>Append stderr to a file
&>Redirect both stdout and stderr to a file

Redirection Examples

# Redirect stdout to a file
$ who > loggedusers

# Redirect stdin from a file
$ cat < loggedusers

# Combine: redirect input AND output
$ sort  sorted_file_list.txt

# Append multiple commands to one file
$ whoami >> usrdata
$ date >> usrdata
$ ls >> usrdata

# Separate stdout and stderr to different files
$ cat x y 1> file1 2> file2
# file1 ‚Üí contents of y (stdout)
# file2 ‚Üí error for missing x (stderr)

# Redirect permission denied error to log
$ ls -l /root/ 2> error.log

üîß Pipe / Pipeline Mechanism

A pipe makes the output of one program the input of another program. Operator: |
Redirection Connects programs to files
Pipeline Connects programs to programs

Pipeline Examples

# Sort logged users and save to file
$ who | sort > sortedloggedusrs

# Page through a long listing
$ ls -l | less

# Sort disk usage numerically in reverse
$ du | sort -nr

# cut ‚Äî copies only certain parts of input to output
$ date
Thu Jan 15 11:28:47 SLST 2017
$ date | cut -f1 -d " "
Thu
$ date | cut -f2 -d " "
Jan

# tr ‚Äî translate characters
$ echo "Hello World" | tr eo 34
H3ll4 W4rld

Background Processes

Commands can be executed in the background by suffixing with an ampersand &.

$ date &
Thu Jan 15 11:28:47 SLST 2016

üîÑ Subshell

To perform tasks, the current (parent) shell creates a new (child) shell. This child shell is called a subshell.

When is a Subshell Created?

  • To execute a group command ‚Äî e.g., $(ls; pwd; date)
  • To execute a script
  • To execute commands in the background ‚Äî e.g., find /home -name myFile -print &
If { } curly braces are used instead of ( ), no subshell is created ‚Äî commands run in the current shell.

üì¶ Variables

Environment Variables (Global)
  • Predefined when users log in
  • Named in UPPERCASE by convention
  • Transmitted to subshells (child gets a copy)
  • View with: env
Shell Variables (Local)
  • User-defined; used in programming
  • Named in lowercase by convention
  • NOT passed to subshells
  • View with: set (shows both local & global)

Setting & Using Variables

# Set a variable (no spaces around =)
$ name=sunil
$ echo $name
sunil

# Multi-word values must be quoted
$ fullname="Sunil Sarath"
$ echo My name is $fullname
My name is Sunil Sarath

# Variable referencing another variable
$ SAMPLE="Hello World"
$ SAMPLE1="Hello $SAMPLE"
$ echo $SAMPLE1
Hello Hello World

# Variable with grave operator ‚Äî assign command output
$ NAME=`echo $USER`
$ echo $NAME
ict1

# Use variable as a command
$ DATE="cat datefile"
$ $DATE
Thu Jan 15 08:16:48 SLST 2004

Exporting Variables (Making Them Available to Subshells)

# Export a variable to subshells
$ OS=Linux
$ export OS

# Create subshell and access the exported variable
$ bash
$ echo $OS
Linux

# Return to parent shell
$ Ctrl+D
Use env to see environment variables. Use set to see all variables (local + global).

‚ûï Arithmetic Expressions

Bash has limited direct support for arithmetic. Direct operations don't compute automatically. 
# Direct ‚Äî does NOT compute
$ num=3
$ echo $num + 2
3 + 2

# Using expr utility ‚Äî DOES compute
$ echo `expr $num + 2`
5

# Parentheses to control evaluation order (escape with \)
$ echo `expr \( 2 + 3 \) \* 5`
25

üöÄ Getting Started with Shell Programming

Steps to Write and Run a Shell Script

1
Use any editor (e.g., vi, gedit) to write the shell script
2
Set execute permission: chmod permission scriptname or chmod u+x scriptname
3
Execute the script: bash scriptname / sh scriptname / ./scriptname / . scriptname (runs in current shell)

Shell Script Structure

The first line of every script must specify the shell using #!/bin/bash (called a shebang). 
#!/bin/bash
#
# My first shell script
#
echo "Hello World"
# Save as "first", then:
$ chmod 755 first
$ ./first
Hello World

Comments

Comments improve understandability. The # character ignores all characters up to the end of the line (except on line 1 where #!/bin/bash is the shebang).

#
# This program adds two numbers
#

üîê System Administration in Linux

The root account is the most privileged account in Linux. When using this account, it is crucial to be as careful as possible.

Root account capabilities include:

  • Adding user accounts
  • Changing user passwords
  • Removing user accounts
  • Examining log files
  • Installing software
  • Carrying out all facets of system administration
