üóÇÔ∏è File System Organization

Virtual Filesystem
  • Hierarchical representation of files and directories
  • Unix interface to the physical filesystem
Physical Filesystem
  • Collection of data represented by the virtual filesystem
  • Data arranged on one or more actual storage devices (e.g. Hard disks)

üìÅ Files and Directories (Virtual Filesystem)

A File
  • A collection of information
  • Can be referred to by a filename
A Directory
  • A special kind of file
  • A way of grouping files together
  • Consists of entries ‚Äî files or other directories
Unix File System Structure: The Unix File System is a hierarchical file system with a tree structure. The topmost directory is the root directory ( / ), from which all other directories branch.

Pathnames

Absolute Pathname Starts from the root directory /

$ cat /home/subash/aa.txt
Relative Pathname Starts from the current working directory

$ pwd ‚Üí /home
$ cat subash/aa.txt
Directory components are separated by forward slashes /

üîó Links ‚Äî Hard Links vs. Soft Links

Hard Links

Create: ln existing_file new_link
ln file2 /home/nimal/file3
  • Hard links have the same inode number
  • ls -l shows link count in second column
  • Links have actual file contents
  • Removing any link just reduces the link count; others remain unaffected
  • Cannot create a hard link for a directory
  • Even if original file is removed, the link still shows file contents
  • One file can have many different names (hard links)

Soft (Symbolic) Links

Create: ln -s existing_file new_softlink
ln -s ~/tmp/file2 sfile2
  • Soft links have different inode numbers
  • ls -l shows second column value 1 and link points to original
  • Contains the path of the original file, not the contents
  • Removing soft link doesn't affect the original
  • If original file is removed, link becomes a "dangling" link
  • Can link to a directory
  • Similar to MS Windows shortcuts
Remove links: Use rm. The file is removed only when the last hard link is removed.

Hard Link Exercise

# Create a Test directory and a new file inside it
$ mkdir Test
$ cd Test
$ touch sample1

# Create a hard link to sample1 named sample2
$ ln sample1 sample2

# Display inodes ‚Äî both will share the same inode number
$ ls -il sample1 sample2

# Remove original and check content through link
$ rm sample1
$ cat sample2   # Content still accessible!

Soft Link Exercise

# Create a soft link for sample2
$ ln -s sample2 sample3

# Display inodes ‚Äî different inode numbers, 'l' before permissions
$ ls -il sample2 sample3

# List contents
$ cat sample2
$ cat sample3

# Remove the original and check
$ rm sample2
$ ls                   # sample3 still appears
$ cat sample3          # Error: no such file or directory (dangling link)

üåê Network File Systems

Network File System (NFS)

NFS is a client/server application that lets a computer user view and optionally store and update files on a remote computer as though they were on the user's own computer.
  • NFS is a protocol ‚Äî a way of mounting Linux discs/directories over a network
  • An NFS server can export one or more directories that can be mounted on a remote Linux machine
  • Main home use: share data on a central server (e.g., music collection) to all PCs ‚Äî single copy, central location

Mounting Filesystems in Linux

  • A filesystem is a hierarchy of directories used to organize files on a computer or storage media
  • On Linux/Unix-like systems, directories start from the root directory
  • To access files on a storage device, the user must inform the OS where in the directory tree to mount the device
  • Mounting a filesystem means making a particular filesystem accessible at a certain point in the Linux directory tree

ü§ù Samba

"Samba is an Open Source/Free Software suite that provides seamless file and print services to SMB/CIFS clients."
Source: www.samba.org
  • Installed on any platform other than Microsoft Windows
  • Makes use of the TCP/IP protocol installed on the host server
  • Allows the host to interact with Windows clients/servers
  • Runs on Unix platforms and "speaks" to Windows clients
  • Allows a Unix system to move into a Windows "Network Neighborhood"
  • Windows users can access file and print services

SMBD ‚Äî SMB Daemon

  • File and print services are provided by smbd (the SMB Daemon)
  • Handles "share mode" and "user mode" authentication and authorization
  • You can protect shared file and print services by requiring passwords
  • Each user has their own username and password
  • System Administrator can grant or deny access

Known Problems & Solutions

Problem: Windows machine recognizes the Samba server but does not allow access.
Solution: Different encryption/decryption systems for Windows and Unix. Find enableplaintextpassword and disable Windows encryption.
