📜 AWK & sed — Text Processing Power Tools

Pattern scanning · Stream editing · Data extraction · Automation
🔍 AWK — Overview & Philosophy

AWK (Aho, Weinberger, Kernighan) is a programming language for structured data manipulation and formatted reports. It processes text line by line, matches patterns, and executes associated actions.

📁 Records & Fields
Views text as records (lines) and fields (columns). Splits by whitespace by default.
🔧 Programming features
Variables, conditionals, loops, arithmetic & string operators.
📄 Formatted reports
Generate structured output with BEGIN & END blocks.
⚙️ AWK reads from file(s) or stdin, outputs to stdout. Does not work with binary files.
⚙️ AWK Syntax & Workflow
awk '/search pattern1/ {Actions} /pattern2/ {Actions}' inputfile
  • Pattern: regular expression; Actions: statements inside { }.
  • Multiple pattern-action pairs allowed, separated by newline.
  • Default action: { print } → prints matching lines if no action given.
  • If pattern missing → action applied to every line.
  • Statements delimited by semicolon (;).
📌 Working methodology: AWK reads input one line at a time. For each line, checks patterns in order; if pattern matches, executes corresponding action. If no pattern matches and no action defined → default print.
📂 AWK Examples — employee.txt demonstration

Sample data (employee.txt):
100 Nihal Manager Sales $5,000
200 Jagath Developer Technology $5,500
300 Sanjaya Sysadmin Technology $7,000
400 Nishadi Manager Marketing $9,500
500 Ranga DBA Technology $6,000

Print all lines
awk '{print;}' employee.txt
Pattern match (Nihal or Nishadi)
awk '/Nihal/ > /Nishadi/' employee.txt
Print specific fields
awk '{print $2, $5;}' employee.txt
awk '{print $2, $NF;}' employee.txt # $NF = last field
💡 Built-in variables: $0 (whole line), $1,$2,... (fields), NF (number of fields), NR (record number), FS (field separator).
🏁 BEGIN and END — initialization & finalization

BEGIN block executes before reading input. END block runs after all lines processed.

awk 'BEGIN { print "Header: Name\tSalary" } { print $2, $5 } END { print "Report complete" }' employee.txt
📊 Useful one-liners:
• Employees with ID > 200: awk '$1 > 200' employee.txt
• Technology department: awk '$4 ~ /Technology/' employee.txt
• Count employees in Technology:
awk 'BEGIN {c=0} $4 ~ /Technology/ {c++} END {print "Tech count =", c}' employee.txt
✂️ sed — Stream Editor Introduction

sed (Stream EDitor) performs text transformations: substitution, deletion, insertion, search & replace, without opening files interactively. Supports regular expressions.

sed OPTIONS... [SCRIPT] [INPUTFILE...]
🧠 Most common command: s (substitute). Changes occurrences of a regex into a replacement string.

Example file: Unixos.txt containing lines with "unix" string.

🔄 sed Substitution — flags & advanced usage
Basic replace
sed 's/unix/linux/' Unixos.txt
Replaces first occurrence per line.
Replace Nth occurrence
sed 's/unix/linux/2'
Changes second match only.
Global replacement
sed 's/unix/linux/g'
All occurrences on each line.
Specific line number
sed '3 s/unix/linux/'
Affects only line 3.
🖨️ /p flag (print): sed 's/unix/linux/p' prints replaced line twice (use -n to suppress default output).
📍 Range of lines: sed '1,3 s/unix/linux/' applies substitution to lines 1 through 3.
🗑️ sed deletion & line manipulation
Delete a specific line
sed '5d' Unixos.txt
Delete last line
sed '$d' Unixos.txt
Delete from nth to end
sed '12,$d' file
Delete pattern-matching lines
sed '/abc/d' Unixos.txt
Add blank line after each line
sed G Unixos.txt
Add double space (sed G inserts newline)
🧹 Practical deletion: sed '/^$/d' removes empty lines. sed '/Technology/d' removes lines containing "Technology".
🏛️ sed with text files — Universities & system files

Universities.txt (Sri Lankan Universities):
Ruhuna University
Colombo University
Moratuwa University
Kelaniya University
Use sed to rename, edit, or delete entries.

sed 's/University/Univ./' Universities.txt # replace first occurrence per line
sed '2,4 s/University/U/g' Universities.txt # global on lines 2-4
📁 sed works directly on streams; use -i flag for in‑place editing (e.g., sed -i 's/old/new/g' file.txt).
📊 AWK vs sed – When to use which?
🎯 AWK
  • Field-oriented data processing
  • Complex report formatting
  • Arithmetic / conditionals / loops
  • Numeric and string computations
✂️ sed
  • Stream editing, text substitution
  • Simple find & replace
  • Line deletion, insertion, search
  • Faster for pattern-based replacements
🧠 Both support regular expressions. AWK is more like a programming language; sed excels at scripted edits.
🔧 AWK advanced patterns & built-in variables
  • NR (Number of Records) → awk 'NR==3 {print $0}' print only line 3
  • NF (Number of Fields) → awk '{print $NF}' last field each line
  • FS (Field Separator) → awk -F':' '{print $1}' /etc/passwd
  • Multiple patterns: awk '/Manager/ {print $2} /DBA/ {print $2}' emp.txt
  • Arithmetic: awk '$5 > 6000 {print $1, $2}' employee.txt
📌 BEGIN/END with counters: awk 'BEGIN {sum=0} {sum+=$5} END {print "Total salary:", sum}' employee.txt
✍️ sed extended commands (insert / append / change)
Insert line before
sed '2i\New line before line 2' file
Append after line
sed '3a\Appended after line 3' file
Change line
sed '4c\Replace entire line 4' file
Multiple commands with -e
sed -e 's/unix/linux/' -e '/os/d' file
💡 Use sed -n 's/pattern/replacement/p' to print only replaced lines, avoiding duplicate output.
📝 Hands-on exercises & quick reference
  • From employee.txt, print names and salaries of employees from Technology department using AWK.
  • Use sed to replace all occurrences of "Manager" with "Lead" in the file.
  • Delete lines 2 to 4 from Universities.txt with sed.
  • Find the average salary from employee.txt using AWK (fields: $5).
  • Use sed global replace to change "unix" to "GNU/Linux" globally, then save changes to new file.
# AWK average salary example
awk '{sum+=$5} END {print "Avg =", sum/NR}' employee.txt
# sed in-place backup
sed -i.bak 's/unix/linux/g' Unixos.txt
📖 Summary – AWK & sed at a glance
  • AWK ideal for structured data, column‑based reports, field arithmetic, conditionals. BEGIN/END add headers/footers.
  • sed perfect for quick substitutions, deletions, text transformations in scripts without opening files.
  • Both use regular expressions, can be combined in shell pipelines: cat file | awk '{...}' | sed 's/.../.../'.
  • Common AWK variables: NR, NF, $0, $1...$NF, FS, OFS.
  • Common sed flags: s///g (global), p (print), d (delete), -i (in‑place).
  • Master these tools for efficient log analysis, configuration management, and report generation on Unix/Linux.
🧪 Pro tip: Learn to combine AWK & sed with grep for powerful one-liners — indispensable for sysadmins and developers.
