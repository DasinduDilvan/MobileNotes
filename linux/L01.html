1. Logging In to a Linux System

When you first log in on a Linux system, you are always associated with a directory called the home directory (also called the working directory or current directory). Your home directory has the same name as your username and is where your personal files and sub-directories are saved.

Example prompt: 
/home/rashmi]~$

The $ symbol is called a prompt ‚Äî it signals that the system is waiting for you to enter a command.

Key Facts about the Linux Command Line

  • Every Unix command is a sequence of letters, numbers, and characters ‚Äî no spaces within the command name.
  • Linux is case-sensitive: cat and Cat are different commands.
  • The prompt is displayed by a special program called the shell. Shells accept and run commands, and can also be programmed using shell scripts.

Types of Shells

Bourne Shells The original shell sh was written by Steven Bourne. Most shells since then end in sh to indicate they extend the original idea.
C Shells A different family of shells with C-like syntax. Less common in modern Linux environments.
bash ‚Äî Bourne Again Shell
Linux comes with bash, written by the Free Software Foundation. It is the default shell in Linux. As long as you are logged in, bash is constantly running and displaying the $ prompt.

Linux Command Syntax

command [options] [arguments]

All Linux commands start with the name of the command and can be followed by options (modify behavior, usually prefixed with -) and arguments (targets like filenames or paths).

2. Getting Help

man ‚Äî Manual Pages

The man command displays reference pages for any command. Linux man pages cover every available command.

man ls        # shows the manual for 'ls'

Press q to exit the man page viewer.

4. The File System

Unix provides files and directories. A directory is like a folder ‚Äî it contains files and can contain other directories. The collection of all directories and files is called the file system.

Root Directory
Initially, the file system has one directory called root (/). Inside root are more directories, and inside those are files and more directories ‚Äî creating a hierarchical tree structure.
Paths
A path refers to either a directory or a filename. All items before the final name in a path must be directories.
Example: /home/ruhuna/ICT/File1

6. Access Permissions

Each file and directory has permissions that control access. Permissions are defined for three sets of users:

Owner (u)The user who created the file.
Group (g)Users belonging to the file's group.
Others (o)All other users.

Permission Types

read (r)View contents of file or directory.
write (w)Change contents or create/delete files in a directory.
execute (x)Run the file, or cd into the directory. Directories must be executable to be accessed.
‚ö†Ô∏è Important: Users with write access to a directory can delete files in that directory even if they don't have write permission on the file itself!

Permission Examples ‚Äî Files

Permission StringMeaning
-rw-------Only owner can read and write.
-rwxrwxrwxEveryone can read, write, and execute (and delete). Not common!
-rw-r--r--Everyone can read, but only owner can write.

Permission Examples ‚Äî Directories

Permission StringMeaning
drwx------Only owner can view, cd into, and delete.
drwxrwx---Owner and group can view, write, etc.
drwxr-xr-xEveryone can cd into, but only owner can modify.

Octal Representation of Permissions

Permissions can be expressed as octal numbers: read = 4, write = 2, execute = 1. Add values together for each set.

chmod ‚Äî Change File Permissions

chmod (change mode) changes the permissions on a file or directory.

Symbolic Mode 
chmod [ugoa][+/-][rwx] <file>
u=user, g=group, o=others, a=all
+ = add, - = remove

$ chmod go+rx dir ‚Äî readable by group & others
$ chmod u+x myscript ‚Äî make executable for owner
Octal Mode 
chmod ### <file>
Each # is the sum for owner, group, others.

$ chmod 755 dir ‚Äî rwxr-xr-x
$ chmod 600 file ‚Äî rw-------
$ chmod 754 file1
owner: rwx (4+2+1=7)
group: r-x (4+0+1=5)
others: r-- (4+0+0=4)

chown & chgrp ‚Äî Change Owner / Group

chown ‚Äî Change the owner of a file or directory.

$ chown ict2 file1 file2
Changes owner of file1 and file2 to user 'ict2'.
chgrp ‚Äî Change the group of a file or directory.

$ chgrp ent1 file1 file2
Changes group of file1 and file2 to group 'ent1'.

7. Text Processing Commands

grep ‚Äî Search for Patterns

grep searches its input for a pattern (simple substring or regular expression). It is case-sensitive by default.

$ grep Science science.txt      # case-sensitive search
$ grep -i science science.txt   # case-insensitive
$ grep -ivnc science science.txt
OptionMeaning
-iIgnore upper/lower case distinctions
-vDisplay lines that do NOT match
-nPrecede each matching line with its line number
-cPrint only the total count of matched lines

wc ‚Äî Word Count

wc counts the number of words, lines, and characters in files.

wc [-clw] [file1 file2 ... fileN]
OptionMeaning
wc -lPrint number of lines
wc -wPrint number of words
wc -cDisplay count of bytes
wc -mPrint count of characters
wc -LPrint length of the longest line only
Output format of wc file:   [lines] [words] [bytes] filename

cmp ‚Äî Compare Two Files

cmp compares two files byte by byte. If a difference is found, it reports the byte and line number where the first difference occurs. If no differences are found, no output is produced.

$ cmp file1 file2

diff ‚Äî Show Differences Between Files

diff analyzes two files and prints the lines that differ. It outputs instructions for how to change one file to make it identical to the second.

$ diff file1 file2
Output notation: 2c2 means "line 2 in the first file needs to be changed to match line 2 in the second file." Lines prefixed with < are from file1; lines with > are from file2.

tr ‚Äî Translate Characters

tr operates on standard input (does not accept a filename). It replaces all occurrences of characters in string1 with corresponding characters in string2.

$ echo "Hello World" | tr eo 34
H3ll4 W4rld

8. Pipes

A pipe (|) is a method of interprocess communication (IPC). It means the output of one program on one side of the pipe serves as input for the program on the other end. A set of piped commands is called a pipeline.

Why use pipes? By combining simple OS utilities, you can easily solve complex tasks without writing a program. 
$ ls -l              # list files
$ ls -l | sort       # list and sort
$ ls -l | grep "D"   # list and filter for "D"

10. Linux Processes

A process is any running program. Under Linux, a process has:

PIDProcess Identity (unique ID)
PPIDParent Process ID
UIDOwner (User ID)

Process Commands

CommandDescription
psView active processes.
$ ps -A ‚Äî all processes   $ ps -l ‚Äî long list
Ctrl+CStop execution of a program launched from the terminal.
kill pidStop the execution of any process by its PID.
$ kill 1258
$ kill 9 1258 ‚Äî signal 9 is the most powerful (force kill)

Launching Jobs on Linux

A job is any executable run from the terminal.

  • Jobs are launched as foreground tasks by default ‚Äî they block the terminal until done.
  • To launch a job in the background, append & at the end of the command:
$ sleep 50 &    # run in background
$ fg            # bring background job to foreground

11. Compression & Archiving

gzip ‚Äî Compress Files Compresses files and saves them in .gz format. The compressed file occupies less space than the original.

$ gzip example.txt
tar ‚Äî Archive a File System Used to archive directories/files. Although tar and gzip are different, they are often used together.

tar + gzip Together

Create a compressed archive:

$ tar -czvf archive.tar.gz /path/to/directory
FlagMeaning
-cCreate an archive.
-zCompress the archive with gzip.
-vVerbose mode ‚Äî display progress in terminal (optional but helpful).
-fAllows you to specify the filename of the archive.
-xExtract an archive (replaces -c).

Extract a compressed archive:

$ tar -xzvf archive.tar.gz
