📜 AWK & sed
Lecture 07 — Pattern scanning, stream editing, text processing
AWK — Overview
AWK (Aho, Weinberger, Kernighan) processes text line-by-line. Sees data as records (lines) and fields (columns, split by whitespace by default).
awk '/pattern/ {action}' inputfile
awk '{print}' file        # print all lines
awk 'NR==3 {print}' file  # print line 3
Built-in Variables
$0 whole line · $1,$2… fields
NF number of fields · NR record (line) number
FS field separator · $NF last field
BEGIN / END
BEGIN { } runs before any input
END { } runs after all input
Used for headers, totals, counters.
AWK Examples (employee.txt)
100 Nihal Manager Sales $5,000
200 Jagath Developer Technology $5,500
300 Sanjaya Sysadmin Technology $7,000
400 Nishadi Manager Marketing $9,500
# Print specific fields
awk '{print $2, $5}' employee.txt        # Name, Salary
awk '{print $2, $NF}' employee.txt       # Name, last field

# Pattern match
awk '/Nihal/' employee.txt
awk '$4 ~ /Technology/' employee.txt     # Dept = Technology

# Condition
awk '$1 > 200' employee.txt              # ID > 200
awk '$5 > 6000 {print $1,$2}' employee.txt

# Counter
awk 'BEGIN {c=0} $4~/Technology/ {c++} END {print "Count=",c}' employee.txt

# Sum salary
awk '{sum+=$5} END {print "Total:", sum}' employee.txt

# With field separator
awk -F':' '{print $1}' /etc/passwd       # usernames

# Header and footer
awk 'BEGIN {print "Name\tSalary"} {print $2,$5} END {print "Done"}' employee.txt
sed — Stream Editor
sed transforms text without opening files interactively. Supports regex. Most common command: s (substitute).
sed 's/old/new/' file          # replace 1st occurrence per line
sed 's/old/new/g' file         # replace ALL occurrences (global)
sed 's/old/new/2' file         # replace 2nd occurrence per line
sed 's/old/new/p' file         # print replaced lines (shows twice)

sed '3 s/old/new/' file        # only on line 3

sed -i.bak 's/old/new/g' file  # in-place with .bak backup
Deletion

🧵 POSIX Threads (Pthreads)
Lab — Creating threads, concurrency, shared memory, race conditions
Key Concepts
ProcessRunning program with its own memory space
ThreadLightweight execution unit inside a process; shares memory with siblings
Concurrent ExecutionMultiple threads running simultaneously; order non-deterministic
Race ConditionUnpredictable result when threads modify shared data without synchronization
Compilation: Always use -pthread flag:
gcc filename.c -o output -pthread
Core Pthread Functions
Function / TypePurpose
pthread_tData type to store thread ID
pthread_create(&tid, NULL, func, arg)Create a new thread; runs func
pthread_join(tid, NULL)Wait (block) until thread finishes
pthread_exit(NULL)Terminate thread from within itself
Activity 1 — Single Thread
// thread1.c
#include <stdio.h>
#include <pthread.h>

void* addition(void* arg)
{
    printf("Addition Result = %d\n", 10 + 20);
    pthread_exit(NULL);
}

int main()
{
    pthread_t t1;
    pthread_create(&t1, NULL, addition, NULL);
    pthread_join(t1, NULL);
    printf("Main Thread Finished\n");
    return 0;
}
pthread_create() launches thread. pthread_join() blocks main until child finishes. Output: Addition Result = 30 then Main Thread Finished
Activity 3 — Four Arithmetic Threads
// thread3.c
#include <stdio.h>
#include <pthread.h>

void* addition(void* arg)       { printf("Addition = %d\n",       40+10); pthread_exit(NULL); }
void* subtraction(void* arg)    { printf("Subtraction = %d\n",    40-10); pthread_exit(NULL); }
void* multiplication(void* arg) { printf("Multiplication = %d\n", 40*10); pthread_exit(NULL); }
void* division(void* arg)       { printf("Division = %d\n",       40/10); pthread_exit(NULL); }

int main()
{
    pthread_t t1, t2, t3, t4;
    pthread_create(&t1, NULL, addition,       NULL);
    pthread_create(&t2, NULL, subtraction,    NULL);
    pthread_create(&t3, NULL, multiplication, NULL);
    pthread_create(&t4, NULL, division,       NULL);
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    pthread_join(t3, NULL);
    pthread_join(t4, NULL);
    printf("All Threads Finished\n");
    return 0;
}
Output order may vary — threads execute concurrently and CPU scheduler decides order.
Activity 5 — Race Condition
// thread5.c — shared variable
#include <stdio.h>
#include <pthread.h>

int counter = 0;  // shared variable

void* increment(void* arg)
{
    int i;
    for(i = 0; i < 100000; i++)
        counter++;  // ⚠ race condition!
    pthread_exit(NULL);
}

int main()
{
    pthread_t t1, t2;
    pthread_create(&t1, NULL, increment, NULL);
    pthread_create(&t2, NULL, increment, NULL);
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    printf("Final Counter = %d\n", counter);
    return 0;
}
🚨 Race Condition: Expected 200,000 but actual result varies (e.g. 147,382). Both threads read counter before the other writes — increments get lost. Fix requires mutexes (synchronization).
Lab Questions Quick Answers
QuestionAnswer
What is a thread?Lightweight execution unit inside a process; shares memory with other threads of same process.
Process vs Thread?Process has own memory; threads share process memory and are lighter/faster to create.
Why does output order change?Threads run concurrently; CPU scheduler decides order — non-deterministic.
Why share memory?Threads are part of same process, so they naturally access the same address space.
What is a race condition?When multiple threads modify shared data without sync → unpredictable results.
Purpose of pthread_join()?Blocks calling thread until target thread finishes; prevents zombie threads.
Why -pthread flag?Links the pthread library required for Pthread functions.
