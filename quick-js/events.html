1What is an Event?

An event is something that happens in the browser ‚Äî a click, a keypress, a page load, a form submission. JavaScript lets you "listen" for these moments and run code in response.

üí° Think of events like notifications. The browser says "hey, something happened!" and your JS code decides what to do about it.
User ActionsClicking, typing, scrolling, hovering
Browser ActionsPage loaded, window resized, timer fired
Your ResponseRun a function, update the DOM, fetch data

2The Basic Pattern

Everything in JS events builds on one line. Learn this by heart:

element.addEventListener("eventType", callbackFunction);

Your first working event:

// HTML: <button id="btn">Click me</button>

const btn = document.getElementById("btn");

btn.addEventListener("click", function () {
  alert("Button clicked!");
});
elementThe DOM node to watch
"eventType"Which event to listen for
callbackFunction to run when fired

3The Event Object

Every callback automatically receives an event object packed with information about what just happened.

btn.addEventListener("click", function (event) {
  console.log(event.type);       // "click"
  console.log(event.target);     // the element clicked
  console.log(event.timeStamp);  // when it happened (ms)
  console.log(event.clientX);   // mouse X coordinate
  console.log(event.clientY);   // mouse Y coordinate
});
‚úÖ e.target ‚Äî the element the user actually interacted with
‚ö†Ô∏è e.currentTarget ‚Äî the element the listener is attached to

5Mouse Events

const box = document.getElementById("box");

// Highlight on hover
box.addEventListener("mouseenter", () => box.style.background = "#2c6e6b");
box.addEventListener("mouseleave", () => box.style.background = "");

// Double click to expand
box.addEventListener("dblclick", () => alert("Double clicked!"));

// Track cursor position
document.addEventListener("mousemove", (e) => {
  console.log("X:", e.clientX, "Y:", e.clientY);
});

6Keyboard Events

document.addEventListener("keydown", function (e) {
  console.log(e.key);      // "a", "Enter", "ArrowUp"
  console.log(e.code);     // "KeyA", "Enter", "ArrowUp"
  console.log(e.ctrlKey);  // true if Ctrl held
  console.log(e.shiftKey); // true if Shift held
  console.log(e.altKey);   // true if Alt held
});

// Practical: detect Enter key in input
input.addEventListener("keydown", function (e) {
  if (e.key === "Enter") {
    console.log("User submitted:", e.target.value);
  }
});
‚ö†Ô∏è e.key gives the human-readable value ("Enter", "a"). e.code gives the physical key position ("KeyA") ‚Äî useful for games where WASD should work regardless of keyboard language.

7Form Events

üö® Always call e.preventDefault() on form submit! Without it the page reloads, wiping all your JS state and data.
const form = document.getElementById("myForm");

// Submit ‚Äî fires when the form is submitted
form.addEventListener("submit", function (e) {
  e.preventDefault(); // üõë stop the page from reloading!
  console.log("Form submitted");
});

// Input ‚Äî fires on every single keystroke (live)
input.addEventListener("input", (e) => console.log(e.target.value));

// Focus / Blur ‚Äî user enters / leaves the field
input.addEventListener("focus", () => console.log("Focused!"));
input.addEventListener("blur",  () => console.log("Left the field"));

// Change ‚Äî fires when value changes AND user leaves the field
select.addEventListener("change", (e) => console.log(e.target.value));
inputEvery keystroke ‚Äî instant feedback
changeAfter edit + user leaves field
submitWhen form is sent (use preventDefault!)

8Event Propagation (Bubbling)

When you click a child element, the event bubbles upward through all parent elements. This catches many beginners off guard.

<!-- HTML -->
<div id="parent">
  <button id="child">Click me</button>
</div>

document.getElementById("parent").addEventListener("click", () =>
  console.log("Parent fired!")
);
document.getElementById("child").addEventListener("click", () =>
  console.log("Child fired!")
);
// Click button ‚Üí logs "Child fired!" then "Parent fired!"

// Stop it from bubbling up:
child.addEventListener("click", (e) => {
  e.stopPropagation(); // parent never hears this
});
Event Bubbles UP the DOM Tree

document ‚Üê html ‚Üê body ‚Üê div#parent ‚Üê button#child ‚Üê üñ±Ô∏è Click

After firing on the target, the event travels up through each ancestor

9Event Delegation ‚≠ê

Instead of adding a listener to every child, add one listener on the parent. It captures events from all children ‚Äî including ones added to the DOM in the future.

<!-- HTML -->
<ul id="list">
  <li>Item 1</li>
  <li>Item 2</li>
  <li>Item 3</li>  <!-- future items work too! -->
</ul>

// ‚úÖ ONE listener handles ALL list items, forever
document.getElementById("list").addEventListener("click", function (e) {
  if (e.target.tagName === "LI") {
    console.log("Clicked:", e.target.textContent);
    e.target.style.color = "#2c6e6b";
  }
});
üöÄ Why delegation? Better performance (1 listener vs 100), works on dynamic content, and keeps your code clean. Essential pattern for lists, tables, and any dynamically-rendered UI.

10Removing Event Listeners

// ‚úÖ Named function ‚Äî can be removed later
function handleClick() {
  console.log("Clicked!");
}
btn.addEventListener("click", handleClick);
btn.removeEventListener("click", handleClick); // ‚úÖ works

// ‚ùå Anonymous function ‚Äî impossible to remove
btn.addEventListener("click", () => console.log("stuck forever!"));

// ‚úÖ { once: true } ‚Äî auto-removes after first fire
btn.addEventListener("click", function () {
  console.log("Only fires once!");
}, { once: true });
üö® You must pass the exact same function reference to removeEventListener. Always name handlers if you plan to remove them later.

11Custom Events

Create and fire your own events ‚Äî great for component-to-component communication.

// 1. Create with a custom data payload
const myEvent = new CustomEvent("orderPlaced", {
  detail: { orderId: 42, total: 99.99 }
});

// 2. Listen anywhere
document.addEventListener("orderPlaced", function (e) {
  console.log("Order:", e.detail.orderId); // 42
  console.log("Total:", e.detail.total);   // 99.99
});

// 3. Fire it
document.dispatchEvent(myEvent);

12Mini Project ‚Äî Live Character Counter

A real-world feature combining input events with live DOM updates:

<!-- HTML -->
<textarea id="txt" maxlength="200" placeholder="Type here..."></textarea>
<p>Characters left: <span id="counter">200</span></p>

// JS
const txt     = document.getElementById("txt");
const counter = document.getElementById("counter");

txt.addEventListener("input", function () {
  const remaining = 200 - txt.value.length;
  counter.textContent = remaining;
  counter.style.color = remaining < 20 ? "#e91e63" : "#2c6e6b";
});

üéØ What this demonstrates

  • Listening to input event on a textarea
  • Reading e.target.value in real time
  • Updating the DOM on every keystroke
  • Conditional styling based on remaining count
