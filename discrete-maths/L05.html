📖 Predicate Logic

Propositional functions · Quantifiers · Translation · Negation · Equivalences
❓ Why Predicate Logic?

Propositional logic is not expressive enough for many mathematical statements. Example:

  • “Every computer in ICT Lab 11 is functioning properly.”
  • From this, we cannot deduce “First computer is functioning properly” using propositional rules alone.
  • Need a language that talks about objects, properties, and relations.
💡 Predicate logic introduces variables (x, y, z), predicates (P(x), M(x)), and quantifiers (∀, ∃) to express generality.
🧩 Propositional Functions (Predicates)

A propositional function is a statement containing variables. It becomes a proposition (true/false) when variables are assigned values from a domain (U).

Example
P(x) = “x > 3”
P(4) is true, P(2) is false.
Multiple variables
Q(x, y) = “x = y + 3”
Q(1,2) false, Q(3,0) true.
📌 A(c, n): “Computer c is connected to network n”.
If MATH1 is connected to CAMPUS2 but not CAMPUS1 → A(MATH1, CAMPUS1) false, A(MATH1, CAMPUS2) true.
⚡ Propositional functions can be combined with logical connectives (¬, ∧, ∨, →, ↔).
Example: P(3) ∨ P(-1) is a proposition, but P(x) ∧ P(y) is not until variables are bound.
🎯 Quantifiers – Expressing scope

Quantification creates a proposition from a propositional function. Two fundamental quantifiers:

∀ Universal Quantifier
“For all”, “for every”, “all”
∀x P(x) : P(x) holds for every x in domain.
∃ Existential Quantifier
“There exists”, “for some”, “at least one”
∃x P(x) : P(x) true for some x in domain.
🔁 Domain must always be specified; without it, quantified statements have no meaning.
🌍 Universal Quantifier ∀

∀x P(x) is true iff P(x) is true for every element in the domain. False if a counterexample exists.

  • P(x): “x > 0”, domain = integers → ∀x P(x) false (negative numbers).
  • P(x): “x > 0”, domain = positive integers → ∀x P(x) true.
  • P(x): “x + 1 > x”, domain = real numbers → ∀x P(x) true.
Example: Q(x): “x < 2”, domain = real numbers → ∀x Q(x) is false (e.g., x = 3 is counterexample).
📐 If domain is finite {x₁, x₂, …, xₙ}, then ∀x P(x) ≡ P(x₁) ∧ P(x₂) ∧ … ∧ P(xₙ).
🔍 Existential Quantifier ∃

∃x P(x) is true iff there exists at least one element in the domain for which P(x) is true.

  • P(x): “x > 3”, domain = real numbers → ∃x P(x) true (e.g., x = 4).
  • Q(x): “x = x + 1”, domain = reals → ∃x Q(x) false (no real satisfies).
  • P(x): “x² > 10”, domain = {1,2,3,4} → ∃x P(x) true (since 4²=16>10).
For finite domain: ∃x P(x) ≡ P(x₁) ∨ P(x₂) ∨ … ∨ P(xₙ).
✨ Uniqueness Quantifier (∃!)

∃!x P(x) means “there exists a unique x such that P(x)” (one and only one).

  • Example: ∃!x (x - 1 = 0), domain real numbers → true (only x=1).
  • P(x): “x > 0”, domain integers → ∃!x P(x) is false (many positive integers).
📖 Alternative expression without ∃!: ∃x ( P(x) ∧ ∀y (P(y) → y = x) ).
⚡ Precedence of Quantifiers

Quantifiers ∀ and ∃ have higher precedence than all logical operators.

∀x P(x) ∨ Q(x) means (∀x P(x)) ∨ Q(x) .
To bind Q(x) inside the quantifier, use parentheses: ∀x (P(x) ∨ Q(x)).
📝 Translating English ↔ Predicate Logic

Key principle: For “all students” use ∀ with implication; for “some student” use ∃ with conjunction.

❌ Common Mistake:
“Every student has taken Java”
∀x (S(x) ∧ J(x)) is wrong (means everyone is a student AND took Java).
✅ Correct: ∀x (S(x) → J(x))
❌ ∃x (S(x) → J(x)) is wrong for “some student took Java”.
✅ Correct: ∃x (S(x) ∧ J(x))
Example 1: “Every student in this class has studied calculus.”
Domain = all students in class → ∀x C(x)
Domain = all people → ∀x (S(x) → C(x)) where S(x) = “x is a student in this class”.
Example 2: “Some student has visited Mexico.”
Domain = students → ∃x M(x)
Domain = all people → ∃x (S(x) ∧ M(x))
📍 “Every student has visited Canada or Mexico”: ∀x (S(x) → (C(x) ∨ M(x)))
📍 “Some student has visited both”: ∃x (S(x) ∧ C(x) ∧ M(x))
🌱 Translation with Specific Domains

Universe: {Fruit, Vegetable, Plant}
Predicates: F(x): fruit, S(x): vegetable, T(x): plant.

  • “Nothing is a vegetable.” → ¬∃x S(x) ≡ ∀x ¬S(x)
  • “All fruits are plants.” → ∀x (F(x) → T(x))
  • “Some fruits are plants.” → ∃x (F(x) ∧ T(x))
  • “No vegetable is a plant.” → ¬∃x (S(x) ∧ T(x)) ≡ ∀x (¬S(x) ∨ ¬T(x))
  • “If any fruit is a vegetable then it is also a plant.” → ∀x ((F(x) ∧ S(x)) → T(x))
⚖️ Logical Equivalences (Predicate)

Two quantified statements are logically equivalent iff they have same truth value for all predicates and domains.

✅ ∀x (P(x) ∧ Q(x)) ≡ ∀x P(x) ∧ ∀x Q(x)
Proof: If every element makes both true, then all make P true and all make Q true, and vice versa.
❌ ∃x P(x) ∧ ∃x Q(x) is NOT equivalent to ∃x (P(x) ∧ Q(x)).
Counterexample: domain integers, P(x)=“x even”, Q(x)=“x odd”. LHS true, RHS false (nothing is both even & odd).
⚠️ Also ∀x P(x) ∨ ∀x Q(x) is not equivalent to ∀x (P(x) ∨ Q(x)).
Example: P(x)=“x even”, Q(x)=“x odd”, domain integers. RHS true, LHS false.
🔁 De Morgan’s Laws for Quantifiers

Negation moves the quantifier and flips it:

¬∀x P(x) ≡ ∃x ¬P(x)
“Not all are P” = “Some are not P”
¬∃x P(x) ≡ ∀x ¬P(x)
“No x is P” = “All x are not P”
NegationEquivalent StatementWhen True
¬∃x P(x)∀x ¬P(x)For every x, P(x) is false
¬∀x P(x)∃x ¬P(x)There exists x for which P(x) is false
Example: Negation of ∀x (x² > x) ≡ ∃x (x² ≤ x).
Negation of ∃x (x² = 2) ≡ ∀x (x² ≠ 2).
📌 Show ¬∀x(P(x)→Q(x)) ≡ ∃x(P(x) ∧ ¬Q(x))
Using De Morgan: ¬∀x(P→Q) ≡ ∃x ¬(P→Q) ≡ ∃x (P ∧ ¬Q).
🧠 Quantifiers as Conjunctions / Disjunctions

If domain is finite, quantifiers reduce to logical connectives:

∀x P(x) ≡ P(x₁) ∧ P(x₂) ∧ … ∧ P(xₙ)
∃x P(x) ≡ P(x₁) ∨ P(x₂) ∨ … ∨ P(xₙ)
Even if domain is infinite, this intuition helps understand quantifier behavior.
✍️ More Translation Practice
  • “Every student in this class has taken a course in Java.”
    Domain all people: ∀x (Student(x) → HasTakenJava(x))
  • “Some student in this class has taken a course in Java.”
    ∃x (Student(x) ∧ HasTakenJava(x))
  • “All Americans eat cheeseburgers.”
    ∀x (American(x) → EatsCheeseburger(x))
  • “There is an American who does not eat cheeseburgers.”
    ∃x (American(x) ∧ ¬EatsCheeseburger(x))
  • “For every person, if person is a student in this class then they have visited Mexico or Canada.”
    ∀x (S(x) → (M(x) ∨ C(x)))
💡 Key reminder: ∀ is usually paired with → (implication), ∃ with ∧ (conjunction) in restricted domains.
📚 Summary – Key Takeaways
  • Propositional functions generalize propositions using variables & predicates.
  • Universal ∀: true for every element; Existential ∃: true for at least one element.
  • To translate “all A are B”: ∀x (A(x) → B(x)). To translate “some A is B”: ∃x (A(x) ∧ B(x)).
  • De Morgan for quantifiers: ¬∀ ≡ ∃¬ , ¬∃ ≡ ∀¬ .
  • Logical equivalences: ∀ distributes over ∧, ∃ does NOT distribute over ∧ (but ∃ distributes over ∨).
  • Quantifiers bind tightly; always use parentheses to clarify scope.
