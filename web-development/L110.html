📖 Introduction to OOP

  • Object Oriented Programming (OOP) is a programming concept that assumes any system as a set of interacting & interrelated objects.
  • PHP supports OOP to build modular, reusable, and maintainable code.
  • OOP helps organize large applications efficiently.
Core Advantage: OOP promotes DRY (Don't Repeat Yourself) code and makes large systems easier to manage.

🧱 Class & Object

Class
  • A programmer-defined data type
  • Includes both functions and data
  • Acts as a template used to define objects
  • Defined using the class keyword followed by the class name
Object
  • An individual instance of the data structure defined by a class
  • Objects are also known as instances
  • Define a class once and create many objects from it
  • Created using the new keyword
Analogy: A class is like a blueprint for a car. The actual cars — polo, mini, beetle — are the objects (instances).

🔧 Member Variables & Member Functions

Member Variable
  • Variables defined inside a class
  • Declared with var, private, or protected keywords
  • Accessed via member functions
  • Called attributes of the object once an object is created
Member Function
  • Functions defined inside a class
  • Used to access and manipulate object data
  • Also called methods

🔗 The $this Variable

  • $this is a built-in variable available inside all objects which points to the current object.
  • Represents an instance of the class or object.
  • Used to access properties and call other methods of the current class.
  • Always begins with the $ sign.
  • Is followed by the -> (arrow) operator.
  • The property name after -> does not take a $ sign — e.g., $this->property
How it works: PHP uses $this as a placeholder until a real object is created. When a member function is running, the value of $this is substituted with the actual object the function is called on.

🆕 The new Keyword

  • To create an instance of a class, the new keyword must be used.
  • Classes should be defined before instantiation.
  • If a string containing the name of a class is used with new, a new instance of that class will be created.
  • When creating a class, you can optionally add brackets to the class name.

🏗️ Constructors

  • All objects can have a special built-in method called a constructor.
  • Used to pass parameters at the time of object creation.
  • Called automatically whenever an object is created from a class.
  • PHP provides a special function __construct() to define a constructor.
  • You can pass as many arguments as you like into the constructor.
Syntax: public function __construct($param1, $param2 = defaultValue) { ... } 
class Player {
    public $name;
    public $health;

    // The Constructor — called automatically on object creation
    public function __construct($name, $startingHealth = 100) {
        $this->name   = $name;
        $this->health = $startingHealth;
        echo "New player '$name' has entered the game!\n";
    }

    public function status() {
        return "{$this->name} has {$this->health} HP.";
    }
}

// Creating objects triggers __construct() automatically
$hero = new Player("Aragorn", 150);
$npc  = new Player("Villager"); // Uses default health of 100
echo $hero->status(); // Output: Aragorn has 150 HP.

🏛️ Four Pillars of OOP

1. Encapsulation

Hiding internal states using public / private access modifiers. Bundles data and methods into a single unit (class) and restricts direct access.

2. Inheritance

Reusing code from parent classes using the extends keyword. A "child" class derives properties and methods from a "parent" class.

3. Polymorphism

Different classes sharing the same interface. The same method name behaves differently depending on the object calling it ("many shapes").

4. Abstraction

Focusing on what an object does, not how. Hides complex implementation details and exposes only the necessary features.

🔒 Encapsulation & Visibility

Encapsulation is the practice of bundling data (properties) and methods into a single unit (a class) and restricting direct access to some components. This prevents outside code from accidentally corrupting internal state.

Access Control Modifiers determine the scope in which a class member can be accessed:

public

No access restrictions. Can be accessed via instances anywhere.

protected

Accessible within the class definition and by extending (child) classes.

private

Can be accessed and modified only within the defining class.

class BankAccount {
    private $balance = 0; // Cannot be accessed directly outside the class

    public function deposit($amount) {
        if ($amount > 0) {
            $this->balance += $amount;
        }
    }

    public function getBalance() {
        return "Your balance is: $" . $this->balance;
    }
}

$myAccount = new BankAccount();
$myAccount->deposit(100);
echo $myAccount->getBalance(); // Output: Your balance is: $100

🧬 Inheritance

Inheritance allows a "child" class to derive properties and methods from a "parent" class. This promotes code reusability — you don't have to rewrite common logic for similar objects. Uses the extends keyword. 

class Vehicle {
    public $brand;
    public function honk() {
        return "Beep beep!";
    }
}

class Car extends Vehicle { // Car inherits from Vehicle
    public $model = "Mustang";
}

$myCar        = new Car();
$myCar->brand = "Ford";
echo $myCar->honk(); // Inherited from Vehicle → "Beep beep!"

🔀 Polymorphism

Polymorphism (meaning "many shapes") allows different classes to be treated as instances of the same parent class or interface. It ensures that the same method name can behave differently depending on the object calling it.

interface Shape {
    public function calculateArea();
}

class Circle implements Shape {
    public function calculateArea() {
        return "Calculating circle area (πr²).";
    }
}

class Square implements Shape {
    public function calculateArea() {
        return "Calculating square area (side²).";
    }
}

// Both are treated as 'Shape' but behave differently
$shapes = [new Circle(), new Square()];
foreach ($shapes as $shape) {
    echo $shape->calculateArea() . "\n";
}

🎭 Abstraction

Abstraction involves hiding complex implementation details and showing only the necessary features of an object.

Real-world analogy: A coffee machine — you press a button (the interface), and you don't need to know how the internal boiler or grinder works. 
abstract class Mailer {
    // The user just calls this — simple interface
    public function sendEmail($address) {
        $this->connectToServer();
        echo "Email sent to $address";
    }

    // Complex internal logic hidden from the user
    abstract protected function connectToServer();
}

class Gmail extends Mailer {
    protected function connectToServer() {
        echo "Connecting to Google SMTP...\n";
    }
}

$mail = new Gmail();
$mail->sendEmail("hello@example.com");

📌 Static Methods and Properties

  • Declaring class properties or methods as static makes them accessible without needing an instantiation of the class.
  • Static variables belong to the class itself, not to any object of that class.
  • To access static methods and static properties from within the class, use self instead of $this.

The self Keyword

  • Represents the class itself (not an instance).
  • Never begins with a $ dollar sign.
  • Is followed by the :: operator (scope resolution operator).
  • The property name after :: always takes the $ dollar sign — e.g., self::$property

Comparison: $this vs self

Feature $this self
Refers to The current object instance The current class
Accesses Non-static properties/methods Static properties, methods, and constants
Operator -> (Arrow operator) :: (Scope Resolution operator)
Context Only inside non-static methods Inside both static and non-static methods

✅ Advantages of Object Oriented PHP

Code Reusability Write once, use many times through inheritance and object reuse.
Easy Maintenance Changes in one class have limited effects on the rest of the codebase.
Better Organization Code is grouped logically by class and responsibility.
Scalability Applications can grow without becoming unmanageable.
Security Encapsulation hides internal state and prevents unwanted access.

🏗️ Introduction to MVC

Model-View-Controller (MVC) is a structural pattern that separates an application into three main logical components:

Model Handles Data & Business Logic. Queries the database and processes data.
View Handles Presentation & HTML. Displays data to the user.
Controller Handles Request Handling & Glue. Coordinates Model and View.

🔄 The MVC Workflow

How a request flows through the system:

  1. User requests a page.
  2. Controller intercepts the request.
  3. Controller asks the Model for data.
  4. Model queries the Database.
  5. Controller gives data to the View.
  6. View sends HTML back to the User.
USER → CONTROLLER → MODEL → DATABASE → CONTROLLER → VIEW → USER

Recommended Folder Structure

/project ├── /app │ ├── /Controllers │ ├── /Models │ └── /Views ├── /public │ └── index.php ← Entry Point └── /vendor ← Libraries

🌟 Benefits of MVC

Parallel Development Frontend and Backend teams can work simultaneously without conflict.
Maintainability Easy to locate and fix bugs in specific layers of the application.
Reusability Models can be reused across multiple controllers without duplication.
SEO Friendly Easy to implement clean, readable URLs for search engine optimization.
