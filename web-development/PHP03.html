🔧 PART 1 — PHP Functions

💡 What Is a Function?

A function is a named block of code that performs a specific task. It can accept input values (called parameters), process them, and optionally return a single value. Functions are defined once and called (reused) as many times as needed.

📦 Types & Advantages of Functions

Two Types of PHP Functions

🔧 Built-in Functions Provided by PHP out-of-the-box. No need to define — just call by name and pass values. Called with all lowercase by convention. Examples: strlen(), date(), sort()
🛠️ User-Defined Functions Written by the developer to handle specific application requirements. Defined using the function keyword.

Advantages of Using Functions

⏱️ Saves Time & SpaceWrite code once, call it as many times as needed — no repetition.
🐛 Improved ReliabilityFix a bug in one place (inside the function) rather than hunting for it everywhere it is used.
📖 Better ReadabilityCode is isolated into named blocks, making the overall structure easier to understand at a glance.

🛠️ User-Defined Functions

User-defined functions are functions created by the programmer according to their own requirements. Declaration always starts with the keyword function.

Syntax

function functionName() {
  // code to be executed
}

Naming Guidelines

  • Give the function a name that reflects what it does (e.g., calculateTotal())
  • A valid name starts with a letter or underscore — never a digit
  • Can contain letters, numbers, and underscores after the first character
  • Function names are case-sensitive — myFunc() ≠ MyFunc()

📨 Arguments & Passing Methods

What Are Arguments?

Arguments are values sent into a function when it is called. Inside the function, they are received as parameters. Multiple arguments are separated by commas and evaluated left to right.
Syntax: function MyFunction($arg1, $arg2) { }

Pass by Value vs Pass by Reference

Pass by Value (Default)copy of variable
  • A copy of the variable's value is passed to the function
  • Changes made inside the function do NOT affect the original variable
  • The original variable remains unchanged after the function completes
  • This is the default behaviour in PHP
  • Syntax: normal parameter — function f($x) { }
Example: $orignum = 10; → after addFive($orignum) → $orignum is still 10
Pass by Reference (&)reference to original
  • A reference (pointer) to the original variable is passed
  • Changes made inside the function DO affect the original variable
  • The argument must be a variable — not a literal value
  • Add an ampersand & before the parameter name in the definition
  • Syntax: function f(&$x) { }
Example: $orignum = 10; → after addSix($orignum) → $orignum becomes 16

Value vs Reference — Quick Comparison

Pass by Value
  • ▸ A copy is used
  • ▸ Original NOT affected
  • ▸ Syntax: function f($x)
  • ▸ Default behaviour
VS
Pass by Reference (&)
  • ▸ A reference is used
  • ▸ Original IS changed
  • ▸ Syntax: function f(&$x)
  • ▸ Must explicitly use &

⚙️ Default Argument Values

A parameter can have a default (fallback) value assigned in the function definition. If the caller does not pass a value for that parameter, the default is used automatically.

Key Rules

  • Default parameters must come after non-default parameters in the definition
  • Only constant values (not variables) can be used as default values
  • A function can have a mix of required and optional (default) parameters
Syntax: function myFunc($required, $optional = "defaultValue") { }

📐 Variable-Length Argument Functions

A function may need to accept a flexible or unknown number of arguments. Declare the function with no parameters at all, then use PHP's three special functions inside to access whatever was passed.

Rules

  • The function call can pass more arguments than defined in the function header
  • You can pass arguments to a function that has no parameters defined at all

Three Special Functions

Recursion

A function that calls itself is called a recursive function. It must have a base case — a condition that stops the recursion. Without a base case, the function recurses infinitely and crashes.
Factorial example: Base case = factorial(0) = 1. Recursive case = factorial(n) = n × factorial(n-1)

📂 Programming in Multiple Files

When programs grow large, they are split across multiple files. Common code (headers, footers, database connections) is written once and included in other files. Including a file produces the same result as copy-pasting its code at that location.

Four Include Statements

StatementFile Not Found?Includes Multiple Times?Best Used For
includeWarning — script continuesYesOptional files (e.g., widgets, sidebars)
requireFatal error — script stopsYesEssential files (e.g., DB config)
include_onceWarning — script continuesNo — only onceOptional, prevent re-declaration
require_onceFatal error — script stopsNo — only onceEssential, prevent re-declaration
_once variants: If the file has already been included, it will not be included again — preventing duplicate function or class declarations that would cause errors.

include vs require — Key Rule

include / include_once
  • ▸ File not found → Warning
  • ▸ Script continues executing
  • ▸ File is optional
VS
require / require_once
  • ▸ File not found → Fatal error
  • ▸ Script stops immediately
  • ▸ File is essential

feof() — Key Points

  • Useful for looping through files of unknown length
  • Returns TRUE when the pointer has reached the end of the file
  • Cannot read from files opened in write-only modes: w, a, x
  • Used as the loop condition: while (!feof($file))

write (mode 'w') vs append (mode 'a')

Mode 'w' — Write
  • ▸ Erases all existing content
  • ▸ Pointer starts at beginning
  • ▸ Creates file if it doesn't exist
  • ▸ Use for: creating fresh files
VS
Mode 'a' — Append
  • ▸ Preserves existing content
  • ▸ Pointer starts at end
  • ▸ Creates file if it doesn't exist
  • ▸ Use for: log files, audit trails

📌 Glossary & Final Comparisons

Key Definitions

Function
A named, reusable block of code that performs a specific task. May accept parameters and optionally return a single value. Two types: built-in (provided by PHP) and user-defined (written by the developer).
Parameter vs Argument
Parameter — variable in the function definition (placeholder). Argument — actual value passed at the call site. Same concept, different perspective.
Pass by Value
Default passing method. A copy of the variable is used inside the function. Original variable is NOT changed. Changes made inside the function are local only.
Pass by Reference (&)
The function receives a reference (pointer) to the original variable. Changes inside the function DO affect the original. Use & before the parameter: function f(&$x).
Default Argument Value
A fallback value assigned to a parameter in the function definition. Used when the caller does not pass a value. Must come after non-default parameters.
Recursion
A function that calls itself. Must have a base case to stop recursion. Without a base case, the function recurses infinitely and crashes. Classic example: factorial calculation.
include vs require
Both include a PHP file at the point of the statement. include shows a Warning and continues if file not found. require shows a Fatal Error and stops. The _once variants prevent re-inclusion.
File Handle / Pointer
A resource variable returned by fopen() representing the open file and the current read/write position within it. All file functions (fread, fwrite, fclose, feof etc.) operate on this handle.
EOF (End of File)
A marker indicating there is no more data to read in a file. Detected with feof(). Returns TRUE when the file pointer reaches the end. Used in while loops: while(!feof($file)).
feof() Restriction
Cannot be used with files opened in write-only modes (w, a, x). Must open with a readable mode (r, r+, a+) before calling feof() or any reading function.
unlink()
Permanently deletes a file from the server. Unlike moving to a trash folder, this deletion is irreversible. Always double-check the filename before calling.
or die() Pattern
A PHP error-handling idiom: if the left side returns false/null, execute die() to stop the script and display an error message. Used with fopen() to safely handle file-open failures.
