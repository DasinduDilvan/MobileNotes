📦 Arrays in JavaScript

What is an Array?

An array is a collection of individual data values in which it is possible to count the values off in order: here is the first, here is the second, and so on.

Element An individual value stored inside an array.
Length The total number of elements in the array.
Index The position of each element. Always starts at 0 and goes up to length - 1.
⚠️ Important: In JavaScript, index numbers always begin with 0 and extend up to one less than the length of the array.

🛠️ Creating an Array

Method 1: Using Square Brackets Preferred

let arr     = [];                    // empty array
let numbers = [1, 2, 3];            // array of numbers
let mixed   = [1, "two", true];     // array with mixed types

Method 2: Using the Array Constructor

let arr     = new Array();           // empty array
let numbers = new Array(5);          // array with length 5, all empty slots
let colors  = new Array("red", "green", "blue"); // array with values

Non-numeric Arrays

Arrays may contain values of any JavaScript type, including strings:

const COIN_NAMES = [
  "penny", "nickel", "dime",
  "quarter", "half-dollar", "dollar"
];

🔁 Array Iteration Methods

Cycling through each element of an array in turn is one of the most useful array idioms. There are three common ways:

1. Classic for Loop

for (let i = 0; i < array.length; i++) {
  console.log(array[i]);
}

2. for...of Loop

for (let item of array) {
  console.log(item);
}

3. forEach() Method

Executes a provided function once for each array element.

// Syntax
array.forEach((element, index) => {
  // code to execute
});

// Example
const numbers = [1, 2, 3, 4, 5];
numbers.forEach((num, idx) => {
  console.log(`Index ${idx}: ${num}`);
});
// Index 0: 1 | Index 1: 2 | Index 2: 3 | ...
💡 Note: for...of and forEach() both iterate over every element. forEach also provides access to the current index and the original array as optional parameters.

➕➖ Adding and Removing Elements

push(element, ...) Adds one or more elements to the end of the array.
unshift(element, ...) Adds one or more elements to the front of the array.
pop() Removes and returns the last element of the array.
shift() Removes and returns the first element of the array.
slice(start, finish) Returns a subarray from start up to (not including) finish. Does NOT modify the original. 
let arr = ["A", "B", "C"];

arr.push("D", "E");   // ["A","B","C","D","E"]
arr.unshift("X","Y"); // ["X","Y","A","B","C","D","E"]

let last  = arr.pop();    // last  = "E"
let first = arr.shift(); // first = "X"

let part  = arr.slice(1, 4); // ["A","B","C"] — original unchanged

🗺️ map() Method

Returns a new array after applying a function to each element. Good for transforming data. The original array is not modified.

// Syntax
array.map((element, index, array) => {
  // return transformed value
});

// Example — squaring numbers
const numbers = [1, 2, 3, 4, 5];
const squared = numbers.map(num => num * num);
console.log(squared); // [1, 4, 9, 16, 25]

🔍 filter() Method

Returns a new array containing only the elements that pass a condition (i.e., the callback returns true).

// Syntax
array.filter((element, index, array) => {
  // return true to keep element, false to discard
});

// Example — keeping even numbers
const numbers = [1, 2, 3, 4, 5];
const even = numbers.filter(num => num % 2 === 0);
console.log(even); // [2, 4]

✅ some() and every() Methods

some() Returns true if at least one element passes the test. Returns false if none pass.
every() Returns true only if all elements pass the test. Returns false if any fail. 
// Syntax
array.some((element, index, array) => { /* return true if condition met */ });
array.every((element, index, array) => { /* return true if condition met */ });

// Example
const ages = [16, 20, 25];
console.log(ages.some(age => age >= 18));   // true  (20 and 25 pass)
console.log(ages.every(age => age >= 18));  // false (16 fails)

⚙️ Functions in JavaScript

What is a Function?

Functions are actions or processes that receive an input and return an output. They can perform various tasks and calculations.

Ways to Define a Function

Function DeclarationNamed function, most common for beginners.
Function ExpressionAnonymous or named function stored in a variable.
Arrow Function ExpressionCompact shorthand syntax using =>.
IIFEImmediately Invoked Function Expression — runs as soon as defined.
Function ConstructorUsing new Function().
Generator FunctionFunctions that can pause and resume using function*.
MethodsFunctions defined as properties of an object.

📝 Function Declaration

When you create a function with a name, it's called a function declaration (also called function definition or function statement). This is the most standard, basic way of writing a function, used very often as a beginner.

Key feature: Function declarations are hoisted — meaning they can be called before they appear in the code.

Anatomy of a Function Declaration

function sendMessage(msg) {
  // ↑ keyword  ↑ function name  ↑ parameter
  console.log(msg);  // ← function statement (body)
}
function keywordTells JS you are creating a function.
Function nameThe identifier used to call the function later.
Parameter(s)Named placeholders for values the function will receive.
Function bodyThe block of code { } that executes when called.

How to Call a Function

function hello() {
  console.log("How can I help you?");
}

hello(); // ← calling the function

🔤 Function Parameter vs Function Argument

Parameter

A parameter is a name defined when creating the function. It acts as a placeholder for a future value.

Rules for parameters:
  • Names cannot be repetitive within the same function.
  • Parameters are optional — not always required.
  • Order matters — values passed must match the order defined.
  • You do not define the data type.

Argument

An argument is the actual value passed to a function when calling it. It is represented by the corresponding parameter inside the function.

function sendMessage(msg) {
  // 'msg' is the PARAMETER
  console.log(msg);
}

sendMessage("Hello! :)");
// "Hello! :)" is the ARGUMENT

↩️ Return Statement in Functions

  • A return statement returns one specific value and stops the function execution.
  • Any code after return will not execute — the return should always be the last statement.
  • The returned value can be any data type.
  • If no value is specified after return, the function returns undefined.

Return with a value

function add(start) {
  return start + 1;
}
console.log(add(0)); // 1

Return without a value

function noValues() {
  return;
}
console.log(noValues()); // undefined
⚠️ Common pitfall: If return and its value are on separate lines, JavaScript may insert a semicolon after return, causing it to return undefined. Always wrap multi-line return values in parentheses: 
function add(start) {
  return (      // ← parentheses prevent auto-semicolon insertion
    start + 1
  );
}

📌 Function Expressions

A function expression assigns a function to a variable. Unlike function declarations, function expressions are not hoisted — they can only be called after the line where they are defined.

Declaration vs Expression: A function declaration defines a named function that is hoisted; a function expression assigns a function to a variable and is not hoisted.

Anonymous Function Expression

A function expression without a name. This is the most common type.

const greet = function () {
  console.log("Hello!");
};

greet(); // Hello!
// With parameters:
const sendMessage = function (name, lastName) {
  console.log(`Hello, ${name} ${lastName}`);
};

sendMessage("Ana", "Denson"); // Ana Denson

Named Function Expression

A function expression with its own internal name, even though it is stored in a variable. The name is only accessible inside the function itself.

const greet = function sayHello() {
  console.log("Hello!");
};

greet();      // Hello!
sayHello();   // Error: not defined

Why Use a Named Function Expression?

Better Debugging If there's an error, the function's internal name appears in stack traces, making it easier to identify.
Self-reference (Recursion) The function can call itself internally using its own name. 
// Recursion example
const countdown = function count(n) {
  if (n === 0) return;
  console.log(n);
  count(n - 1);
};

countdown(3); // 3, 2, 1
