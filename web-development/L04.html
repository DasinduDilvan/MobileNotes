⚡ Event-Driven Programming

Most JavaScript written in the browser is event-driven: the code doesn't run right away — it executes only after some event fires.

Key Idea: Code is not executed immediately. Instead, it waits and responds to user or browser actions.

How it works — Step by Step

1
A UI element (e.g., a button) is displayed on the page for the user to interact with.
2
The user interacts with it (e.g., clicks the button).
3
The element emits an event — an announcement that something interesting has occurred.
4
Any function listening to that event now executes. This function is called an event handler.
function onClick() {
  // This runs when the event fires
  console.log('Clicked!');
}

🌳 The DOM — Document Object Model

Every element on a page is accessible in JavaScript through the DOM (Document Object Model).

Definition: The DOM is a tree of nodes corresponding to HTML elements on a page.
  • You can modify, add, and remove nodes on the DOM.
  • These changes are reflected on the actual web page.

🔍 Getting DOM Objects — querySelector

To access an HTML element's corresponding DOM object in JavaScript, use:

document.querySelector('css selector');
  • Returns the first element that matches the given CSS selector.
  • Uses standard CSS selector syntax (tag name, #id, .class, etc.)
// Returns the element with id="button"
let element = document.querySelector('#button');

👂 Adding Event Listeners — addEventListener

Each DOM object has the following method:

element.addEventListener(eventName, functionName);
eventName

The string name of the JavaScript event to listen for.
Examples: "click", "focus", "blur"

functionName

The name of the JavaScript function to execute when the event fires (the event handler).

const button = document.querySelector('button');
button.addEventListener('click', onClick);

🏷️ HTML Event Attributes (Inline Event Handling)

An event handler is something written into a web page that responds whenever a specific type of event happens.

  • HTML allows event handler attributes, with JavaScript code, to be added directly to HTML elements.
  • Events can be handled directly in HTML using inline event attributes.
<!-- General syntax -->
<input type="button" name="click1" value="Click me!"
  onclick="event_handler_code">

<!-- Example -->
<button onclick="alert('Button Clicked!')">Click Me</button>

⚠️ The Script Loading Problem

When <script src="script.js"></script> is placed in <head>, the JavaScript runs before the page body is parsed.

Problem: When the script reaches document.querySelector('button'), the <button> element hasn't been parsed yet — so querySelector returns null. Calling addEventListener on null throws an error:
Uncaught TypeError: Cannot read property 'addEventListener' of null

✅ The Fix — Use defer

Add the defer attribute to the <script> tag. This tells the browser to delay executing the script until after the DOM is fully loaded.

<script src="script.js" defer></script>
Old-school alternatives (avoid these):
  • Placing the <script> tag at the bottom of the page
  • Listening for the "load" event on the window object
These are outdated. defer is widely supported and the preferred modern approach.

💻 Full Working Example

Printing "clicked" to the Web Console when the user clicks a button:

index.html

<html>
<head>
  <meta charset="utf-8">
  <title>First JS Example</title>
  <script src="script.js" defer></script>
</head>
<body>
  <button>Click Me!</button>
</body>
</html>

script.js

function onClick() {
  console.log('clicked');
}

const button =
  document.querySelector('button');
button.addEventListener(
  'click', onClick
);
Result: When the button is clicked, "clicked" is printed to the browser's Web Console. The defer attribute ensures the DOM is ready before the script accesses it.

🧪 Practice Question

Task: Create a web page with a select box (options: Audi, BMW). When the user clicks a button after selecting an option, display "You selected: [option]" as a paragraph of text.

index.html

<select id="carSelect">
  <option value="">Select a car</option>
  <option value="BMW">BMW</option>
  <option value="Audi">Audi</option>
</select>

<button id="btn">Select a Car</button>

<p id="result"></p>

script.js

let button =
  document.querySelector('#btn');
let selectBox =
  document.querySelector('#carSelect');
let display =
  document.querySelector('#result');

button.addEventListener("click",
  function() {
    let car = selectBox.value;
    display.textContent =
      "You selected: " + car;
  }
);
Key concepts used: querySelector with ID selectors (#id), addEventListener, anonymous function as handler, reading .value from a select element, and writing to .textContent.
