➡️ Arrow Functions (ES6 Feature)

What Is an Arrow Function?

An arrow function is an ES6 (2015) feature that provides a shorter, more concise syntax for writing functions. Also called "fat arrow functions" because of the => symbol.
  • Introduced in ECMAScript 2015 (ES6)
  • Often used for callbacks and concise one-liner functions
  • The variable holding the function should use const (recommended)
  • Single-line expressions are automatically returned — no return keyword needed
  • Multi-line bodies require curly braces { } and an explicit return statement

Basic Syntax

const functionName = (parameters) => expression;

parameters — zero, one, or multiple inputs
expression — automatically returned if single-line
const — recommended keyword for the variable

Four Forms of Arrow Functions

No Parameters() => ...
  • Use empty parentheses () when there are no parameters
  • Parentheses are required when there are no parameters
const greet = () => "Hello!";
Single Parameterx => ...
  • Parentheses around the parameter are optional (but can be included)
  • Most concise form for single-input functions
const square = x => x * x;
Multiple Parameters(a, b) => ...
  • Parentheses are required when there are two or more parameters
const add = (a, b) => a + b;
Multi-line Body{ } + return
  • When the function body has more than one statement, use curly braces { }
  • An explicit return statement is required — it is NOT automatic with { }
const multiply = (a, b) => {
  const result = a * b;
  return result;
};

Arrow Function vs Regular Function

Regular Function
  • ▸ Uses function keyword
  • ▸ Longer syntax
  • ▸ Has its own this binding
  • ▸ Can be hoisted (if declaration)
  • ▸ Suitable for object methods
VS
Arrow Function
  • ▸ Uses => symbol
  • ▸ Shorter, concise syntax
  • ▸ No own this — inherits from surrounding scope
  • ▸ Not hoisted
  • ▸ Best for callbacks and one-liners

⏱️ Synchronous vs Asynchronous JavaScript

Direct (Synchronous) Execution

Synchronous execution means code runs line by line, in order. Each statement must complete before the next one begins. Functions are called one after another and follow the same sequence.
  • Code executes from top to bottom in strict order
  • Each function waits for the previous one to finish completely before starting
  • Simple and predictable — but can block the page if a task takes a long time

Indirect (Asynchronous) Execution

Asynchronous execution means certain functions can run outside the normal sequence. They do not block other code — they are started and the program continues, with the result handled later when the task completes.
  • Functions can go outside the execution order and not depend on it
  • Allows time-consuming tasks (API calls, timers, file reads) to run in the background
  • The rest of the program continues without waiting
  • Result is handled later via a callback, promise, or async/await

Real-World Analogy — Café Order

Café ScenarioJavaScript Equivalent

Coffee is being preparedBackground task running (non-blocking)
You talk to a friend while waitingOther JS code continues executing

Ways to Implement Asynchronous Flow

CallbacksA function passed as an argument to be executed later. Covered in this module.

Async / AwaitSyntactic sugar over promises for cleaner async code.
TimersBuilt-in functions like setTimeout() and setInterval().

Non-blocking I/OReading files, making network requests without pausing execution.

Sync vs Async — Comparison

Synchronous (Direct)
  • ▸ Executes in order, line by line
  • ▸ Each task blocks the next
  • ▸ Simple and predictable
  • ▸ Can cause UI to freeze on long tasks
VS
Asynchronous (Indirect)
  • ▸ Executes out of order when needed
  • ▸ Non-blocking — program continues
  • ▸ More complex to reason about
  • ▸ Keeps UI responsive

🔔 Callback Functions

What Is a Callback Function?

A callback function is a function that is passed as an argument to another function, to be executed later — usually after a task is completed.

Key Concepts

  • The callback is passed by reference (without calling it) — no parentheses when passing
  • The receiving function decides when to call the callback
  • Callbacks are the foundation of asynchronous JavaScript
  • Used with timers, event listeners, array methods (forEach, map, filter), and more
Key distinction:
performTask(displayMessage) — ✅ passes the function as a callback (no parentheses)
performTask(displayMessage()) — ❌ calls the function immediately and passes its return value

How Callbacks Enable Asynchronous Behaviour

⏰ setTimeout()

What Is setTimeout()?

setTimeout() is a built-in JavaScript function that executes a callback function once after a specified delay (in milliseconds). It is asynchronous — the rest of the code continues running while the timer counts down.

Syntax

setTimeout(function, delay);

function — the callback to execute after the delay
delay — time in milliseconds (1000 ms = 1 second)

Key Behaviour — Execution Order

  • setTimeout() does not pause the program — it schedules the callback and moves on immediately
  • Code after the setTimeout() call runs before the callback executes
  • The callback runs after the delay, once the call stack is empty
Classic Example: If you call one(), then setTimeout(three, 2000), then two() — the output order is: one → two → three. three is delayed by 2 seconds while two runs immediately.
FeaturesetTimeout()
Runs how many times?Once (after the delay)
Delay unitMilliseconds (1000 ms = 1 second)
Blocks execution?No — asynchronous, non-blocking
What it acceptsA callback function + a delay value
Use caseDelayed execution, animations, simulating async tasks

What Is an Object?

An object is a collection of key-value pairs. The keys are called properties. Values can be any JavaScript type — primitives, arrays, functions (called methods), or other objects.

Other ObjectsObjects can be nested inside other objects

Three Ways to Create an Object

MethodSyntaxKey Point
Object Literal const obj = { key: value }; Most common and concise. Define properties directly inside { }.
new Object() Constructor const obj = new Object(); Less common. Properties added after creation using dot notation.
ES6 Shorthand Literal const obj = { name, age }; When variable names match property names, you can omit the : value part.
ES6 Shorthand: If the variable name and property name are the same, you write it once — { name } instead of { name: name }. This is called property shorthand.

🔑 Accessing, Adding, Updating & Deleting Properties

Two Ways to Access Properties

Dot Notation — obj.property
  • Most common and readable
  • Property name must be a valid identifier (no spaces, no special chars)
  • Used when the property name is known in advance
  • Example: student.name
Bracket Notation — obj["property"]
  • Property name is written as a string
  • Required when property name contains spaces or special characters
  • Used when property name is dynamic (stored in a variable)
  • Example: student["first name"]

When Dot Notation Fails — Use Bracket Notation

Dot notation fails when the property name contains spaces, starts with a digit, or when the property name is stored in a variable (dynamic access). In all these cases, use bracket notation.

Accessing Properties Dynamically

  • Store the property name in a variable, then use bracket notation with that variable
  • JavaScript evaluates the variable first, then uses its value to access the property
const key = "name";
console.log(student[key]);  → same as student.name

⚙️ Methods & the this Keyword

What Is a Method?

When a function is stored as a value (property) inside an object, it is called a method. Methods allow objects to have behaviour as well as data.

Method Shorthand Syntax (ES6)

  • In ES6+, you can write methods without the function keyword
  • Instead of greet: function() { } you write greet() { }
  • Both forms are functionally equivalent — shorthand is cleaner and preferred
Traditional Method Syntaxgreet: function() {
  return "Hi!";
}
ES6 Shorthand Methodgreet() {
  return "Hi!";
}

The this Keyword

this is a special keyword in JavaScript that refers to the object that is currently calling the function. The value of this depends entirely on how the function is called, not where it is defined.
  • Inside an object method, this refers to the object that owns the method
  • It allows methods to access the object's own properties using this.propertyName
  • Arrow functions do NOT have their own this — they inherit it from the surrounding scope
Example: If student.greet() is called, then inside greet(), this refers to student. So this.name gives student.name.

this — Where It Points

ContextWhat this refers to
Inside an object method (regular function)The object that called the method
Inside an arrow functionThe outer/enclosing scope's this (not the object)
In global scope (non-strict mode)The global object (window in browsers)
⚠️ Arrow functions and this: Because arrow functions do not have their own this, they should not be used as object methods when you need to reference the object via this.

🏭 Factory Functions

What Is a Factory Function?

A factory function is a regular function that creates and returns an object. It acts as a "factory" for producing multiple similar objects with the same structure and behaviour, without using the class or new keyword.

Key Characteristics

  • The object is created at runtime, specifically at the return { ... } statement
  • Each call to the factory function creates a new, independent object
  • Can use ES6 shorthand when parameter names match property names
  • this inside a method of a factory-created object refers to that specific object instance

Why Use Factory Functions?

🔁 ReusabilityCreate many objects with the same shape using one function call per object.
📦 EncapsulationGroup related data and behaviour (methods) together in one object.
✅ No new keyword neededSimpler than class-based creation — no confusion about new.

this Inside a Factory Function Method

When a method is defined inside a factory function's returned object, this refers to the returned object itself — not the factory function's parameters. This ensures the method always accesses the object's own properties, regardless of the outer scope.

Key Definitions Glossary

Arrow Function
An ES6 shorthand function syntax using =>. No own this binding. Single-line expressions are auto-returned. Multi-line bodies need { } and explicit return. Not hoisted.
Synchronous Execution
Code runs line by line in strict order. Each statement completes before the next begins. Can block the page if a task takes long.
Asynchronous Execution
Code can run outside the normal sequence. Long tasks run in the background without blocking the rest of the program. Results are handled later via callbacks, promises, or async/await.
Callback Function
A function passed as an argument to another function, to be executed later — usually after a task completes. Passed by reference (without parentheses). The foundation of async JavaScript.
setTimeout()
A built-in JS function that schedules a callback to run once after a specified delay in milliseconds. Asynchronous — does not block code execution while waiting.
Object
A collection of key-value pairs. Keys are called properties. Values can be primitives, arrays, functions (methods), or other objects. Objects are dynamic — properties can be added, updated, or deleted at runtime.
Property
A key-value pair stored inside an object. The key is a string name; the value can be any type. Accessed using dot notation or bracket notation.
Method
A function stored as a property of an object. Gives objects behaviour. Defined using function expressions or ES6 shorthand syntax. Can use this to refer to the owning object.
this Keyword
A special keyword that refers to the object currently calling the function. Its value depends on how the function is called, not where it is defined. Arrow functions do not have their own this.
Factory Function
A regular function that creates and returns an object. Used to produce multiple similar objects with the same structure. Does not use the new or class keywords.
ES6 Property Shorthand
When a variable name matches the desired property name in an object literal, you can write it once instead of twice. { name } is equivalent to { name: name }.
Dynamic Object
JavaScript objects are dynamic — properties can be added, updated, or deleted at any time after the object is created. No fixed schema is required.
