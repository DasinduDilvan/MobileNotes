📜 Scripting Language

A scripting language is a programming language designed to automate tasks or control other software.

Interpreted, not compiled Scripts run line-by-line through an interpreter — they are not compiled before execution.
Dynamic behavior Used to add dynamic behavior to applications such as web pages, servers, and system tools.
Common examples JavaScript, Python, PHP, Ruby, Shell scripts.

🖥️ Client-Side Scripting

Client-side scripting refers to code that runs inside the user's web browser, not on the server. It allows web pages to become interactive, dynamic, and responsive without reloading the page.

Common tasks of client-side scripting:
  • Updating the DOM (changing text, styles, layout)
  • Validating form inputs before submitting to the server
  • Handling user events (clicks, typing, scrolling)
  • Fetching data from servers using APIs (AJAX / fetch)
The most commonly used client-side scripting language is JavaScript.

Key Characteristics

Improved User ExperiencePages respond without full reloads.
Browser JS EnginesThe browser's built-in engine executes the code.
Limited AccessRestricted for security reasons (sandboxed).
Essential for SPAsEnables Single Page Applications and modern web apps.

🌐 How Web Pages Work

Understanding the request-response cycle is fundamental to web development.

  1. You type a URL in the browser's address bar and press Enter.
  2. The browser sends an HTTP GET request to the server at that URL.
  3. The server responds by sending back an HTML file.
  4. The HTML contains references to other resources like <img src="pup.png"> and <link src="style.css"> — these trigger additional requests.
  5. The server replies with those resources (images, CSS files, JS files).
  6. Once all resources are loaded, the browser renders the complete web page.
HTML + CSS alone produces a web page that doesn't do anything.
HTML describes structure and content. CSS describes appearance and style. To add behavior, we need JavaScript.

What HTML/CSS cannot do alone:

HTML + CSS can:
  • Look the way we want
  • Link to other pages
  • Display differently on different screen sizes
HTML + CSS cannot:
  • Get user input
  • Save user input
  • Show/hide elements dynamically
  • React to events like clicks or typing
👉 Enter JavaScript!

⚡ JavaScript

JavaScript is a programming language. It is currently the only programming language that your browser can execute natively. If you want to make a webpage interactive (animations, events, DOM updates, API calls), you must use JavaScript or a language that compiles to JavaScript (e.g., TypeScript).

Created 1995 by Brendan Eich at Netscape. The first version was written in just 10 days.
Name origin Has nothing to do with Java — it was named that way purely for marketing reasons.
Standardized as ECMAScript — widely used across browsers, servers, and apps.

🔧 Features of JavaScript

Dynamic Typing Variable types are defined based on the stored value. A variable x can store a string, number, array, or object at any time.
Case Sensitive count and Count are treated as completely different identifiers.
Event-Driven Programming Handles user interactions (clicks, keyboard events, mouse movements) efficiently.
Platform Independent Write the script once and run it on any platform or browser without affecting the output.
Interpreted Language Scripts are processed line by line by the JavaScript interpreter — a built-in component of the web browser.
First-Class Functions Functions are treated like variables. You can pass them as arguments, return them, or store them in variables.
Asynchronous & Non-Blocking Uses event loop, callbacks, promises, and async/await. API calls run in the background — no UI freezing.

📦 How JavaScript Gets Loaded in the Browser

  1. The browser parses the HTML file and encounters a <script src="script.js"> tag.
  2. The browser makes an HTTP request to the server for the script.js file (just like for CSS or images).
  3. The server responds with the JavaScript file.
  4. The JavaScript file executes client-side — in the browser on the user's computer.
JavaScript Execution Rules:
There is no "main method". The script file is executed from top to bottom. There is no compilation by the developer — JavaScript is compiled and executed on the fly by the browser (just-in-time / JIT compilation).

🔗 How to Use JavaScript with HTML

JavaScript code can be placed in the <head> or <body> sections of an HTML page, always wrapped inside <script> and </script> tags. The type attribute is not required — JavaScript is the default scripting language in HTML.

In <head> Script is loaded before the body renders. Functions defined here can be called from body elements.
In <body> Script runs as the page loads. Can use document.write() to insert content.
External .js file Best practice for reusable code. Reference via src attribute. Does not contain <script> tags.

JavaScript in <head>

<!DOCTYPE html>
<html>
<head>
  <script>
    function showMessage() {
      alert("Hello from the HEAD section!");
    }
  </script>
</head>
<body>
  <button onclick="showMessage()">Click Me</button>
</body>
</html>

JavaScript in <body>

<body>
  <h2>JavaScript Inside Body</h2>
  <script>
    document.write("This is JS running inside the BODY tag.");
  </script>
</body>

External JavaScript File

<!-- In <head> or <body>: -->
<script src="externalFile.js"></script>

<!-- Full URL -->
<script src="https://www.w3schools.com/js/myScript.js"></script>

<!-- File path -->
<script src="/js/myScript.js"></script>

<!-- Same directory -->
<script src="myScript.js"></script>
External scripts are practical when the same code is used in many different web pages. External script files have the .js extension and do not contain <script> tags inside them.

📤 Display Outputs Using JavaScript

JavaScript can display output in several ways:

1. Alert Box Displays a popup message. Pauses the script until the user clicks OK (blocking).
2. document.write() Writes directly to the webpage while loading. If used after loading, it erases the whole page.
3. innerHTML Most common method. Updates HTML content inside an element selected by getElementById(id).
4. console.log() Writes to the browser console. Used mainly for debugging, not for displaying to users.
5. textContent Safer alternative to innerHTML. Inserts plain text only — avoids HTML injection risks.

Alert Box

<script>
  window.alert("Welcome to JavaScript!");
</script>

innerHTML

<p id="result"></p>
<script>
  document.getElementById("result").innerHTML =
    "This text appears inside the paragraph!";
</script>

console.log()

<script>
  let x = 10;
  let y = 20;
  console.log("The sum is:", x + y); // Output: The sum is: 30
</script>

textContent (Safer Alternative)

<p id="demo"></p>
<script>
  document.getElementById("demo").textContent =
    "<b>This will not be bold</b>"; // Renders as plain text
</script>
To view console output: right-click on the page → Inspect → click the Console tab. The Console tab is also a REPL (Read-Eval-Print Loop) — an interactive shell where you can type and test JavaScript expressions directly.

🔨 Functions

One way to define a JavaScript function:

function name() {
  statement;
  statement;
  // ...
}
function hello() {
  console.log('Hello!');
  console.log('Welcome to JavaScript');
}

hello(); // Calls the function
hello(); // Calls it again
Function Hoisting: Function declarations are "hoisted" — you can call them before they appear in the code. The browser behaves as if the definition is moved to the top of the scope. However, other ways of defining functions (e.g., arrow functions assigned to variables) are not hoisted. Best practice: avoid relying on hoisting. 
// This works due to hoisting:
hello();
hello();

function hello() {
  console.log('Hello!');
}

📦 Variables

Naming Rules

Variable names can contain:
  • Letters (a–z, A–Z)
  • Digits (0–9)
  • Underscores (_)
  • Dollar signs ($)
Variable names must:
  • Begin with a letter, _, or $
  • Not start with a digit
  • Not be a reserved keyword
  • Be case sensitive (count ≠ Count)
Best Practice: Use meaningful names and follow camelCase convention for variables and functions. Example: userName, totalPrice.

var, let, const

// Function scope variable (old way)
var x = 15;

// Block scope variable (modern)
let fruit = 'banana';

// Block scope constant; cannot be reassigned
const isHungry = true;

You do not declare the datatype of a variable — JavaScript is dynamically typed.

var — Function Scope Does not go out of scope at the end of a block (like an if-statement or loop), only at the end of a function. Can cause unexpected bugs.
let — Block Scope Goes out of scope at the end of the block {} in which it is declared. Safer and more predictable than var.
const — Block Scope + No Reassignment Cannot be reassigned after declaration. However, the underlying object can still be mutated (like Java's final).
Variables Best Practices:
  • Use const whenever possible.
  • Use let if you need to reassign.
  • Do not use var.
  • You'll see a lot of old code with var, but const and let are well-supported and recommended by Google and AirBnB JavaScript Style Guides.

Scope Examples

// var — readable OUTSIDE the for-loop (function scope)
function printMessage(message, times) {
  for (var i = 0; i < times; i++) {
    console.log(message);
  }
  console.log('Value of i is ' + i); // Works! prints 3
}

// let — causes ERROR outside the for-loop (block scope)
function printMessage(message, times) {
  for (let i = 0; i < times; i++) {
    console.log(message);
  }
  console.log('Value of i is ' + i); // ERROR: i is not defined
}

// const — ERROR if reassigned
const y = 10;
y = 0;  // ERROR!
y++;    // ERROR!

const list = [1, 2, 3];
list.push(4); // OK — mutating the object is allowed

🏷️ Data Types

JS variables do not have types, but the values do. There are six primitive types:

Booleantrue and false
NumberEverything is a double (no integer type). Special values: NaN, +Infinity, -Infinity.
StringIn 'single' or "double" quotes. Immutable. No char type.
SymbolUsed for unique identifiers (advanced topic).
Nullnull — a value meaning "this has no value".
UndefinedThe value of a variable that has been declared but not assigned a value.
There are also Object types, including Array, Date, String (the object wrapper), etc.

Numbers

All numbers are floating point. Operators and precedence are like C. A Math class is available: Math.floor(), Math.ceil(), etc.

const homework = 0.45;
const midterm  = 0.2;
const final    = 0.35;
const score = homework * 87 + midterm * 90 + final * 95;
console.log(score); // 90.4

Strings

Strings can be defined with single or double quotes (no functional difference). They are immutable. Use + for concatenation. Use the .length property (not a function) to get the size.

let snack = 'coo';
snack += 'kies';
snack = snack.toUpperCase();
console.log("I want " + snack); // "I want COOKIES"

Template Literals

Template literals provide a clean way to create multi-line strings and perform string interpolation. Created using backticks (` `). Variables and expressions are embedded using ${...} syntax.

// Multi-line string
let str = `The quick brown fox
jumps over the lazy dog.`;

// String interpolation
let a = 10;
let b = 20;
let result = `The sum of ${a} and ${b} is ${a+b}.`;
console.log(result); // "The sum of 10 and 20 is 30."

Boolean

Boolean operators: && (AND), || (OR), ! (NOT).

let isHungry = true;
let isTeenager = age > 12 && age < 20;

if (isHungry && isTeenager) {
  pizza++;
}
Truthy and Falsy Values: Non-boolean values are automatically converted when used in control statements.
  • Falsy: null, undefined, 0, NaN, '', ""
  • Truthy: Everything else
if (username) {
  // Only runs if username is defined and non-empty
}

🧩 Function Parameters

Function parameters are not declared with var, let, or const.

function printMessage(message, times) {
  for (var i = 0; i < times; i++) {
    console.log(message);
  }
}
