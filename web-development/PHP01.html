LECTURE 1 🌐 The Internet & Web Applications

What Is the Internet?

The Internet is a global system of interconnected computer networks that use the standard Internet protocol suite (TCP/IP) to serve billions of users worldwide. Connected computers can exchange text, graphics, voice, video, and programs.
  • Origins: In the late 1960s, ARPA (Advanced Research Projects Agency) developed the ARPANET, which eventually evolved into today's Internet
  • Uses TCP/IP as the standard communication protocol

What Is a Web Application?

A web application is a computer program that allows website visitors to view, submit, and retrieve data to/from a database over the Internet using a web browser.
  • Uses server-side scripts (PHP, ASP) for storage and retrieval of information
  • Uses client-side scripts (JavaScript, HTML) to present information to users
  • Example: Google Apps, online banking portals, e-commerce sites

🏗️ Client-Server Architecture & 3-Tier Model

Web Client & Web Server

🖥️ Web Client The browser running on the user's computer. Makes HTTP requests to the server and displays the response. Examples: Chrome, Firefox, Edge.
🖧 Web Server Responds to client requests by providing resources. Executes server-side scripts (PHP, JSP, ASP). Consists of physical hardware, an OS, and server software.
Web server and client communicate using HTTP (Hypertext Transfer Protocol) — a platform-independent protocol.

📜 Scripting — Client-Side vs Server-Side

What Is Scripting?

Scripting is a way to automate getting information to and from computers. A script is a computer program (a series of commands in a file) that can be executed without being compiled.

Client-Side vs Server-Side Scripting

Client-Side Scripting
  • ▸ Runs on the user's browser
  • ▸ Downloaded, interpreted and executed by the browser
  • ▸ Enhances functionality and appearance
  • ▸ Makes pages more dynamic and interactive
  • ▸ Can validate user input
  • ▸ Reduces requests to the server
  • ▸ Performance depends on user's computer speed
  • ▸ Example: JavaScript
VS
Server-Side Scripting
  • ▸ Runs on the web server
  • ▸ Interpreted and executed on the server
  • ▸ Generates results sent to user as HTML
  • ▸ Interacts with databases and back-end storage
  • ▸ Script code is never seen by the user
  • ▸ Wide range of programmatic capabilities
  • ▸ Access to server-side software
  • ▸ Examples: PHP, ASP, JSP, Perl, Python

🛠️ XAMPP Components

What Is XAMPP?

XAMPP is a cross-platform web development environment that bundles Apache, MySQL, and PHP together. Works on Windows, Linux, and Mac.
X — Cross Platform Runs on Windows, Linux, and Mac operating systems.
A — Apache Web Server Provides the mechanics for delivering web pages to users. PHP sits inside Apache — they work together to create dynamic pages. Free, open-source, most popular web server application.
M — MySQL The database component. Provides data-storage for the web application — all data, products, accounts, etc. reside here. Free and open source. Supports multiple OS.
P — PHP Server-side scripting language embedded inside Apache. Processes dynamic content, accesses MySQL, generates HTML responses.

Apache — Key Facts

  • Most popular web server application in the world
  • Free and open-source
  • Platform-independent — supports different operating systems
  • PHP component sits inside Apache — they work together

MySQL — Key Facts

  • Provides the data-storage side of the web application
  • All data — products, accounts, user information — resides in MySQL
  • Free and open source
  • Supports different operating systems

LECTURE 2 🐘 PHP — Introduction & Features

What Is PHP?

PHP stands for "PHP: Hypertext Preprocessor" (a recursive acronym). It is a widely-used general-purpose scripting language especially suited for web development that can be embedded directly into HTML.

Historical Facts

  • Conceived in 1994 by Rasmus Lerdorf
  • PHP pages contain HTML with embedded PHP code that does "something"
  • PHP code is enclosed in special start and end tags: <?php ... ?>
  • These tags allow you to jump into and out of PHP mode within an HTML document

Features of PHP

💰 Free to DownloadAvailable at www.php.net — no cost
🔓 Open SourceAnyone can view, modify, and redistribute the source code. Supported freely by community.
🗄️ Database SupportSupports multiple DBMSs including MySQL, PostgreSQL, SQLite, and more.
🌐 Platform IndependentRuns on Windows, Linux, Mac — code works across platforms without changes.
🔌 Server CompatibleCompatible with almost all servers used today — Apache, IIS, and others.
📝 Embeddable in HTMLPHP code can be mixed directly with HTML in the same file.

Server-Side Scripting — Definition (Lecture 2)

Server-side scripting is a web server technology in which a user's request is fulfilled by running a script directly on the web server to generate dynamic web pages. Used to provide interactive web sites that interface to databases or other data stores on the server.

📦 PHP Variables — Rules & Scope

What Is a Variable?

A variable is a representation of a particular value. PHP variables can hold values or expressions. PHP has no command for declaring a variable — it is created the moment you first assign a value to it.

Rules for PHP Variables

  • A variable starts with the $ sign, followed by the variable name
  • A variable name must start with a letter or underscore — never a digit
  • A variable name can only contain alpha-numeric characters and underscores (A-z, 0-9, _)
  • Variable names are case-sensitive — $age and $AGE are different variables
  • PHP has no command for declaring a variable — it is created on first assignment
Examples: $txt = "Hello world!";  |  $x = 5;  |  $_name = "Alice";

Variable Scope

The scope of a variable is the part of the script where the variable can be referenced/used. PHP has three different variable scopes: local, global, and static.
Global Scopedeclared outside functions
  • A variable declared outside a function has global scope
  • Can only be accessed outside functions by default
  • To use a global variable inside a function, use the global keyword
  • Using a global variable inside a function without global keyword generates an error
global keyword syntax: global $x, $y; — placed inside the function to access global variables
Local Scopedeclared inside functions
  • A variable declared within a function has local scope
  • Can only be accessed within that function
  • Using a local variable outside its function generates an error
  • Local variables are deleted when the function finishes executing
Static Scopestatic keyword
  • Normally, local variables are deleted when a function completes
  • Using the static keyword prevents a local variable from being deleted
  • The variable retains its value between function calls
  • Still only accessible within the function (it remains local in scope)
  • Declared as: static $x = 0; — only initialized once
Use case: Counters that need to remember their value across multiple calls to the same function.

🔀 PHP Conditional Statements

What Are Conditional Statements?

Conditional statements are used to perform different actions for different conditions. They allow the program to make decisions at runtime based on whether conditions are true or false.

Four Types of Conditional Statements

StatementDefinitionUse When
if Execute code only if a specified condition is true One condition, one action if true
if...else Execute one block if condition is true, another block if false Binary choice — true or false path
if...elseif...else Select from multiple conditions in order; else handles the remaining cases Multiple conditions, evaluated top-down
switch Selects one of many blocks based on a variable's value matching a case One variable, many possible exact-match values

switch Statement — Key Rules

  • Evaluates one expression/variable against multiple case values
  • Each case ends with a break statement — without it, execution "falls through" to the next case
  • The default case runs when no other case matches — equivalent to the final else
  • More efficient than many elseif conditions when checking one variable against many values
switch syntax:
switch($var) {
  case 'value1': // code; break;
  case 'value2': // code; break;
  default: // code; break;
}

if...elseif...else — Key Rules

  • Conditions are evaluated from top to bottom
  • The first matching condition executes its block — remaining conditions are skipped
  • The final else block runs if none of the conditions are true
  • Can have multiple elseif blocks between the initial if and final else

Key Definitions Glossary

Internet
A global system of interconnected computer networks using TCP/IP. Evolved from ARPANET (developed by ARPA in the late 1960s). Enables exchange of text, graphics, voice, video, and programs between connected computers.
Web Application
A computer program that allows visitors to view, submit, and retrieve data to/from a database over the Internet using a web browser. Combines server-side scripts (PHP/ASP) and client-side scripts (JavaScript/HTML).
Web Client
The browser running on the user's computer. Makes HTTP requests to the server and displays the responses. Examples: Chrome, Firefox, Edge.
Web Server
A physical server running server software that responds to client HTTP requests by providing resources. Executes server-side scripts. Consists of hardware, an OS, and web server software (e.g., Apache).
HTTP (Hypertext Transfer Protocol)
The platform-independent communication protocol used by web servers and clients to exchange data over the web.
3-Tier Architecture
A web architecture with three layers: Tier 1 (Client/Presentation — browser), Tier 2 (Application/Logic — web server with scripts), Tier 3 (Data — database server).
Scripting Language
A computer language with a series of commands in a file that can be executed without being compiled. Used to automate tasks. Examples: JavaScript (client), PHP (server).
PHP
PHP: Hypertext Preprocessor. A widely-used open-source general-purpose scripting language especially suited for web development. Can be embedded in HTML. Created in 1994 by Rasmus Lerdorf. Platform independent, supports MySQL, compatible with Apache and IIS.
Variable (PHP)
A named representation of a value. Always starts with $. Created on first assignment — no declaration command needed. Case-sensitive. Name must start with a letter or underscore, cannot start with a digit.
Variable Scope
The part of the script where a variable can be referenced. PHP has three scopes: global (outside functions), local (inside functions — deleted when done), and static (inside functions — value persists across calls).
static keyword
Used inside a function to prevent a local variable from being deleted when the function ends. The variable retains its value between calls but remains accessible only within the function.
global keyword
Used inside a function to access a variable that was declared in the global scope. Without it, a global variable is not accessible inside a function.
