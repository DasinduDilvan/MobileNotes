⚡ Event-Driven Programming

What Is Event-Driven Programming?

Event-driven programming is a paradigm where code does not run immediately — instead, it executes only after a specific event fires. The program waits for user interactions or browser actions, then responds accordingly.
  • Most JavaScript written in the browser is event-driven
  • Code is passive — it sits and waits until triggered
  • Events can be triggered by users (click, type, hover) or by the browser (page load, resize)

How Event-Driven Flow Works

1. UI Element
A clickable button, input field, or other interactive HTML element exists on the page.
2. User Interaction
The user performs an action — clicking, hovering, typing, submitting, etc.
3. Event Fires
The element emits an event — an announcement that something interesting has occurred (e.g., a "click" event).
4. Event Handler Runs
Any function that is listening for that event is executed. This function is called an event handler.

What Is an Event?

An event is a signal (announcement) fired by an HTML element when something interesting happens — usually as a result of user interaction or a browser action. Events are the mechanism that makes web pages interactive.

What Is an Event Handler?

An event handler is a JavaScript function that is set up to execute when a specific event fires on a specific element. It "handles" (responds to) the event. Also called a listener or callback in this context.

🌳 The DOM — Document Object Model

What Is the DOM?

The DOM (Document Object Model) is a tree of objects representing every HTML element on a web page. It is the programming interface that allows JavaScript to access, modify, add, and remove HTML elements dynamically.

Key Properties of the DOM

  • Every HTML element on the page is accessible in JavaScript through the DOM
  • The DOM represents the page as a tree of nodes — each element is a node
  • JavaScript can modify nodes → changes the element on the page
  • JavaScript can add nodes → adds new elements to the page
  • JavaScript can remove nodes → removes elements from the page
  • The global document object is the entry point to the DOM

🔍 querySelector — Accessing DOM Elements

What Is querySelector?

document.querySelector() is a built-in JavaScript function that returns the first DOM element matching a given CSS selector. It is the standard way to access an HTML element from JavaScript.

Syntax & Rules

document.querySelector('css selector');

css selector — any valid CSS selector string
Returns the first matching element — or null if no match is found
  • Uses the same CSS selector syntax as your stylesheets
  • #id — select by ID  |  .class — select by class  |  button — select by tag
  • Returns only the first match — use querySelectorAll() to get all matches
  • If no element matches, it returns null — calling methods on null causes an error
document.querySelector('#button')Select by ID
Returns the element whose id attribute is "button". IDs should be unique per page.
document.querySelector('.box')Select by Class
Returns the first element with class "box".
document.querySelector('button')Select by Tag
Returns the first <button> element found in the document.

👂 addEventListener — Attaching Events

What Is addEventListener?

addEventListener() is a method available on every DOM object. It registers a function (event handler) to be called whenever a specified event fires on that element.

Syntax & Rules

element.addEventListener(eventName, functionName);

eventName — a string: the name of the JS event to listen for (e.g., "click")
functionName — the function to run when the event fires
Note: Pass the function without parentheses — onClick not onClick()

Key Concepts

  • The event handler function is passed as a callback — called only when the event fires
  • You can attach multiple listeners to the same element for the same or different events
  • The function name passed must be without parentheses — otherwise it executes immediately instead of being registered
  • This is the modern, recommended way to handle events (vs inline HTML attributes)
  • removeEventListener() can be used to detach a listener later

Two-Step Process

Step 1
Get the element
via querySelector
→
Step 2
Add listener
via addEventListener
Full pattern:
const btn = document.querySelector('#myBtn');
btn.addEventListener('click', handleClick);

function handleClick() {
  console.log('Button was clicked!');
}

addEventListener vs Inline HTML Events

addEventListener (Recommended)
  • ▸ Keeps JS separate from HTML
  • ▸ Can attach multiple handlers to one element
  • ▸ Can be removed later with removeEventListener
  • ▸ Cleaner, more maintainable code
  • ▸ Modern best practice
VS
Inline HTML Attributes (Older)
  • ▸ Mixes HTML and JavaScript together
  • ▸ Only one handler per event per element
  • ▸ Cannot be removed cleanly
  • ▸ Harder to maintain in large projects
  • ▸ Still widely seen in older code

🏷️ HTML Inline Event Attributes

What Are HTML Event Attributes?

HTML allows event handler attributes to be added directly to HTML elements. These attributes contain JavaScript code (or a function call) that runs when the specified event occurs. This is called inline event handling.

How Inline Event Handling Works

  • Event attributes are added directly inside the HTML tag
  • The attribute value is JavaScript code as a string
  • The browser executes the code when the event fires
  • Example: <button onclick="alert('Clicked!')">Click Me</button>
Syntax:
<element eventAttribute="javascriptCode">

Example:
<input type="button" value="Click me!" onclick="event_handler_code">
<button onclick="alert('Button Clicked!')">Click Me</button>

⏳ The defer Attribute & Script Loading Problem

The Problem — Script in <head>

⚠️ Classic Error: When a <script> tag is placed in the <head>, the JavaScript runs before the <body> elements exist in the DOM. Any querySelector call returns null because the elements haven't been parsed yet.

Why This Happens — Step by Step

Browser parses HTML
Reads the HTML file top to bottom.
Reaches <script> tag
In the <head> — before <body> elements exist.
Fetches and runs JS
JavaScript executes immediately. The <body> has not been parsed yet.
querySelector returns null
The element doesn't exist in the DOM yet — querySelector('button') returns null.
Error occurs
Calling addEventListener on null throws a TypeError.

The Solution — defer Attribute

Adding the defer attribute to a <script> tag tells the browser to download the script but not execute it until after the entire DOM has been fully parsed.
Syntax: <script src="script.js" defer></script>

What defer Does

  • The script file is downloaded in parallel with HTML parsing (non-blocking)
  • The script executes only after the entire HTML document has been parsed
  • All DOM elements are guaranteed to exist when the script runs
  • querySelector calls will find their elements correctly
  • Widely supported across modern browsers

Old Workarounds — Why to Avoid Them

Old MethodProblem
Put <script> at the bottom of the page (before </body>) Works but is untidy and considered outdated. Hard to maintain in large projects.
Listen for the "load" event on the window object Waits for ALL resources (images, CSS) to load — slower than necessary. Outdated pattern.
✅ Best Practice: Always use defer on external script tags placed in <head>. It is modern, well-supported, and the cleanest solution.

Key Definitions Glossary

Event
A signal fired by a browser or HTML element when something notable happens — usually a user interaction (click, keypress) or browser action (page load). Events are the trigger mechanism for interactive web pages.
Event-Driven Programming
A programming paradigm where code does not execute immediately — instead it waits and runs only when a specific event fires. Most browser JavaScript is event-driven.
Event Handler
A function registered to execute when a specific event fires on a specific element. Also called an event listener or callback. It "handles" the incoming event.
DOM (Document Object Model)
A tree-structured programming interface representing every HTML element on a page as an object (node). JavaScript accesses and manipulates the page through the DOM. The global entry point is the document object.
document.querySelector(selector)
A DOM method that returns the first HTML element matching the given CSS selector. Returns null if no match is found. Used to get a reference to a DOM element before attaching events or changing content.
addEventListener(event, fn)
A DOM method that registers a callback function to run when a specified event fires on an element. The modern, recommended approach for event handling. Multiple listeners can be added to the same element.
Inline Event Attribute
An HTML attribute (e.g., onclick, onchange) that contains JavaScript code directly inside an HTML tag. Older approach — mixes HTML and JavaScript. Only one handler per event per element.
defer (script attribute)
An attribute added to a <script> tag that tells the browser to download the script in parallel but only execute it after the full DOM has been parsed. Prevents null errors from querySelector. Current best practice.
null (from querySelector)
The value returned by querySelector when no matching element is found — often because the script ran before the DOM was ready. Calling addEventListener on null throws a TypeError. Solved by using defer.
Node (DOM Node)
An individual object in the DOM tree, corresponding to one HTML element. Every element on the page — <div>, <button>, <p> etc. — is represented as a DOM node that JavaScript can interact with.
