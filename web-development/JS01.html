🗂️ Scripting Languages & Client-Side Scripting

What Is a Scripting Language?

A scripting language is a programming language designed to automate tasks or control other software. Scripts are usually interpreted (not compiled) — they run line-by-line through an interpreter.
  • Used to add dynamic behaviour to applications such as web pages, servers, and system tools
  • Common examples: JavaScript, Python, PHP, Ruby, Shell scripts
  • Key difference from compiled languages: no separate compilation step — the interpreter processes source code directly at runtime

What Is Client-Side Scripting?

Client-side scripting refers to code that runs inside the user's web browser, not on the server. It allows web pages to become interactive, dynamic, and responsive without reloading the page.

Common Tasks of Client-Side Scripts

  • DOM Manipulation — updating text, styles, or layout on the page
  • Form Validation — checking inputs before submitting to the server
  • Event Handling — responding to clicks, typing, scrolling, and other user actions
  • API Communication — fetching data from servers using AJAX / fetch without a page reload

Key Characteristics of Client-Side Scripting

✅ Improved User ExperiencePages respond instantly to user actions without waiting for the server.
🔧 Browser Engines Execute ItEach browser has a built-in JavaScript engine (e.g., V8 in Chrome) that runs the code.
🔒 Limited Access (Security)Browser scripts cannot access the local file system or OS for security reasons.
🌐 Essential for Modern WebSingle Page Applications (SPAs) like Gmail and React apps rely entirely on client-side scripting.

Most Commonly Used Client-Side Language

JavaScript is the most commonly used — and currently the only programming language that browsers can execute natively.

💡 What Is JavaScript?

JavaScript is a programming language and currently the only programming language that browsers can execute natively. If you want to make a webpage interactive, you must use JavaScript — or a language that compiles to JavaScript (e.g., TypeScript).

Historical Facts

  • Created in 1995 by Brendan Eich at Netscape
  • The first version was written in 10 days
  • JavaScript has nothing to do with Java — it was named that way purely for marketing reasons
  • Today it is standardized as ECMAScript and used across browsers, servers (Node.js), and apps
Important distinction: JavaScript ≠ Java. They are completely separate languages with different syntax, purpose, and design. The name similarity is a historical marketing decision, not a technical relationship.

JavaScript vs PHP — Client vs Server

⚡ JavaScript
  • ▸ Runs in the browser (client-side)
  • ▸ Executed by the browser's JS engine
  • ▸ Used for UI interactivity & DOM updates
  • ▸ Can also run server-side via Node.js
VS
🐘 PHP
  • ▸ Runs on the server (server-side)
  • ▸ Executed by the web server
  • ▸ Used for database access & page generation
  • ▸ Client never sees PHP source code

🌟 Features of JavaScript

FeatureDefinition
Dynamic Typing Variable types are determined by the value stored, not declared by the programmer. The same variable can hold a string, number, or array at different times.
Case Sensitive Identifiers like variable and function names are case-sensitive. count and Count are two different variables.
Event-Driven JavaScript can handle user interactions (clicks, keyboard input, mouse movements) through an event-driven model — code runs in response to events.
Platform Independent Write the script once and run it on any platform or browser without changes. The browser's JS engine handles execution.
Interpreted Language Scripts are processed line by line by the JavaScript interpreter — a built-in component of the web browser. No separate compilation step by the developer.
First-Class Functions Functions are treated like variables — they can be passed as arguments to other functions, returned from functions, and stored in variables.
Asynchronous & Non-Blocking Uses an event loop with callbacks, promises, and async/await. Tasks like API calls run in the background — the UI does not freeze while waiting.

🔗 How to Use JavaScript with HTML

Three Ways to Include JavaScript

MethodWhereHowKey Point
Internal — in <head> Inside <head> tag Wrapped in <script> tags Functions defined here are available to the body. Good for defining reusable functions.
Internal — in <body> Inside <body> tag Wrapped in <script> tags Runs when the browser reaches that point in parsing. Can directly manipulate elements above it.
External File Separate .js file <script src="file.js"></script> Best practice. Reusable across multiple pages. File does not contain <script> tags.

The <script> Tag — Key Rules

  • JavaScript code must be placed between <script> and </script> tags
  • Can be placed in either <head> or <body>
  • The type="text/javascript" attribute is not required — JavaScript is the default scripting language in HTML
  • External scripts use the src attribute: <script src="myScript.js"></script>

External Script — Three Reference Types

Full URLLinks to a script hosted on another server
src="https://example.com/js/file.js"
File PathLinks to a script in a specific folder on the same server
src="/js/myScript.js"
No PathLinks to a script in the same folder as the HTML file
src="myScript.js"

Advantages of External Scripts

  • The same code can be reused across many different web pages
  • Separates HTML structure from JavaScript behaviour — cleaner code
  • Browsers can cache the JS file, speeding up subsequent page loads
  • External .js files do NOT contain <script> tags — just pure JavaScript

Execution Model — Key Rules

  • There is no "main method" — the script file executes from top to bottom
  • There is no compilation step by the developer — JavaScript is compiled and executed on the fly by the browser
  • The browser uses JIT (Just-In-Time) compilation — technically different from pure interpretation
  • JavaScript runs client-side — on the user's computer inside the browser, not on the server

Function Hoisting

Hoisting: Function declarations (using the function keyword) are hoisted — the browser moves them to the top of their scope before execution begins. This means you can call a function before it is defined in the code.
⚠️ Warning: There are other ways to define functions (function expressions, arrow functions) that are NOT hoisted. Avoid relying on hoisting in your code — it leads to confusing bugs.

console.log()

  • Prints log messages to the browser's Developer Console
  • JavaScript's equivalent of printf() in C — used for debugging
  • To view: Right-click on the page → Inspect → Console tab
  • The Console is also a REPL (Read-Eval-Print Loop) — you can type JS expressions directly to test them
  • Output does not appear on the visible webpage — only in the developer tools

📦 Variables — var, let, const

Variable Naming Rules

  • Variable names are case-sensitive — count and Count are different
  • Can contain: letters (a–z, A–Z), digits (0–9), underscores (_), dollar signs ($)
  • Must begin with a letter, underscore, or dollar sign — never a digit
  • JavaScript reserved keywords cannot be used as variable names
  • Convention: use camelCase for variables and functions — e.g., userName, totalPrice

Three Declaration Keywords

var
  • ▸ Function scope
  • ▸ Accessible anywhere inside the function, even outside loops/if blocks
  • ▸ Can be re-declared and reassigned
  • ▸ Gets hoisted to top of function
  • ▸ Avoid using in modern JS
let
  • ▸ Block scope
  • ▸ Only accessible within the { } block it is declared in
  • ▸ Can be reassigned, cannot be re-declared in the same scope
  • ▸ Not accessible outside its block
  • ▸ Use when reassignment is needed
const
  • ▸ Block scope
  • ▸ Cannot be reassigned after declaration
  • ▸ Must be initialized at declaration
  • ▸ Objects/arrays can still be mutated (like Java's final)
  • ▸ Use by default — preferred choice

Scope — var vs let/const

var — Function Scope
  • ▸ Scoped to the entire function
  • ▸ Accessible outside of if, for, and other blocks
  • ▸ Only goes out of scope at the end of the function
  • ▸ NOT accessible outside the function it was declared in
VS
let / const — Block Scope
  • ▸ Scoped to the nearest enclosing block { }
  • ▸ Accessing outside its block causes a ReferenceError
  • ▸ Predictable, safer scoping behaviour
  • ▸ Strongly recommended over var

const — Important Nuance

const prevents reassignment of the variable binding — but it does NOT prevent mutation of the underlying object or array. For example, you cannot do list = [] but you CAN do list.push(4). This is similar to Java's final keyword, NOT C++'s const.

Best Practices

Rule of thumb:
  • ✅ Use const whenever possible (default choice)
  • ✅ Use let only when the variable needs to be reassigned
  • ❌ Do NOT use var — it has confusing function-scope behaviour

This is recommended by both the Google and AirBnB JavaScript Style Guides. Note: older code on the internet uses var heavily — ignore that pattern.

Dynamic Typing

  • Variables in JS do not have a fixed type
  • The value has a type, not the variable
  • You do not declare the datatype before using a variable
  • The same variable can hold different types at different times
  • Function parameters are NOT declared with var, let, or const

🏷️ Data Types in JavaScript

Six Primitive Types

Boolean Number String Null Undefined Symbol

Plus Object types — which include Array, Date, and String object wrappers.

TypeDefinitionValues / Notes
Boolean Represents a logical true/false value true and false only. Used in conditions and logical operators.
Number All numeric values — there is no separate integer type Everything is a floating-point double. Special values: NaN (not-a-number), +Infinity, -Infinity. Use Math.floor(), Math.ceil() etc.
String A sequence of characters representing text Defined with single ' or double " quotes. Immutable. No separate char type — letters are strings of length 1. Concatenate with +. Check length with .length property.
Null Represents an intentional absence of any value The value is null — explicitly set to "no value".
Undefined A variable that has been declared but not assigned a value Default value of uninitialized variables.
Symbol A unique, immutable identifier Advanced use case — used for unique property keys.
Object A collection of key-value pairs; includes arrays, functions, dates Array, Date, String object wrapper, custom objects, etc.

Truthy & Falsy Values

Non-boolean values are automatically converted to boolean when used in conditions. This is called truthy/falsy evaluation.
Falsy Values (evaluate to false) null   undefined   0   NaN   ''   ""
Truthy Values (evaluate to true) Everything else — any non-zero number, any non-empty string, any object or array.

Template Literals

Definition: Template literals provide an easy way to create multi-line strings and perform string interpolation (embedding variables/expressions directly into strings).
  • Created using backtick characters: ` ` — not single or double quotes
  • Variables and expressions are embedded using ${ } syntax — e.g., ${a + b}
  • Can span multiple lines without special characters
  • Much cleaner than string concatenation with +

🔧 Functions in JavaScript

Function Declaration Syntax

Syntax:
function name(param1, param2) {
  // statements;
}
  • Functions are defined with the function keyword followed by a name and parentheses
  • Parameters are listed inside the parentheses — they are NOT declared with var, let, or const
  • The function body is enclosed in { }
  • Functions are called by writing their name followed by ()

Function Hoisting

  • Function declarations (using the function keyword) are hoisted — they can be called before they appear in the code
  • This works because the browser moves declarations to the top of their scope before execution
  • Other function forms (expressions, arrow functions) are NOT hoisted
  • Best practice: Do not rely on hoisting — define functions before calling them

First-Class Functions

In JavaScript, functions are first-class citizens — they are treated like any other value. This means functions can be:
  • Stored in a variable
  • Passed as an argument to another function
  • Returned from another function

Key Definitions Glossary

Scripting Language
A programming language designed to automate tasks or control other software. Scripts are usually interpreted, running line-by-line through an interpreter without a separate compilation step.
Client-Side Scripting
Code that runs inside the user's web browser (not on the server). Enables interactive, dynamic web pages without full page reloads. JavaScript is the primary client-side scripting language.
JavaScript
A programming language created in 1995 by Brendan Eich. Currently the only language browsers can execute natively. Standardized as ECMAScript. Has nothing to do with Java despite the name.
Dynamic Typing
The type of a variable is determined by the value it currently holds, not by an explicit declaration. The same variable can hold different types at different times.
Hoisting
A JavaScript behaviour where function declarations are moved to the top of their scope before code execution. This allows functions to be called before they appear in the source code. Not all function forms are hoisted.
Block Scope
A variable is only accessible within the block (curly braces { }) in which it was declared. Exiting the block makes the variable inaccessible. Used by let and const.
Function Scope
A variable is accessible anywhere within the function it was declared in — including inside loops, if-statements, and other blocks within that function. Used by var.
Template Literal
A string defined using backticks (` `) that supports multi-line text and string interpolation via ${expression} syntax. Cleaner alternative to string concatenation with +.
First-Class Function
Functions treated as values — they can be assigned to variables, passed as arguments, and returned from other functions. A fundamental feature of JavaScript.
DOM (Document Object Model)
A programming interface for HTML documents. Represents the page as a tree of objects. JavaScript can manipulate the DOM to dynamically change content, structure, and styles.
REPL (Read-Eval-Print Loop)
An interactive programming environment that reads input, evaluates it, prints the result, and loops. The browser's Console tab is a JavaScript REPL — useful for quickly testing code.
JIT Compilation (Just-In-Time)
A technique where code is compiled to machine code at runtime (just before execution) rather than ahead of time. Modern JS engines (like V8) use JIT — this is why JavaScript is technically compiled, not purely interpreted.
