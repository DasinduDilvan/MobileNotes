Form Encoding & Validation

🔐 Part 1 — HTML Form Encoding & Client Data Submission

Why Encoding Matters

❌ Without Encoding
  • Special characters (? & # =) can change the meaning of the URL
  • Spaces and non-ASCII characters can make the request invalid
  • Data can get corrupted or misread by the server
✅ With Encoding
  • URLs are safe to send over the web
  • The server parses values exactly as intended
  • You stay compliant with URL/HTTP rules

🏷️ What is enctype?

enctype (encoding type) specifies how form data is encoded when submitting a form to the server.

<form method="post" enctype="value">
  • enctype works with both GET and POST, but is only meaningful for POST requests.
  • If enctype is not specified, browsers automatically apply application/x-www-form-urlencoded encoding.

📦 Encoding Form Data — Default (application/x-www-form-urlencoded)

Field Structure

  • Fields are listed in the order they appear in the form
  • Data sent as name=value pairs
  • Pairs separated by &
name=Nimal Perera (Plain text)  →  name=Nimal+Perera (Encoded)

Handling Empty Fields

  • Fields with null values may be omitted.
  • Unselected radio buttons and checkboxes do not appear in the encoded data.
  • Hidden fields with a value attribute are included.

How Data is Converted

SpacesEncoded as +
Special CharactersPercent encoding: %HH (ASCII HEX value)
Line BreaksEncoded as %0D%0A in multi-line text fields

Common ASCII → HEX Reference

CharacterHEX ValueCharacterHEX Value
!21%25
"22&26
#23'27
$24@40
(28)29

Example Encoding

city=Kandy & Colombo  →  city=Kandy+%26+Colombo
Faculty=Technology & University=University of Ruhuna↵Matara
→  Faculty=Technology&University=University+of+Ruhuna%0D%0AMatara

Structure Rules

  • Only form elements with a name attribute are included.
  • <input type="submit" value="Submit"> has no name, so it is ignored in encoded data.
  • Browsers treat submit buttons as action triggers, not form data.

📋 Other enctype Values

multipart/form-data

Used when a form includes file uploads.

<form method="post"
  enctype="multipart/form-data">
  <input type="file" name="upload">
  <input type="submit" value="Upload">
</form>
text/plain

Sends data in a human-readable format without encoding special characters.

<form method="post"
  enctype="text/plain">
  <input type="text" name="username">
  <input type="submit" value="Send">
</form>

🧪 JavaScript & Encoding — encodeURIComponent()

JavaScript ensures safe transmission of values using the built-in encodeURIComponent(value) function.

let name = encodeURIComponent("Nimal Perera");
// Result: "Nimal%20Perera"
Note: Unlike the default form encoding which uses + for spaces, encodeURIComponent uses %20.

✅ Part 2 — HTML Form Validation

Definition: Form validation is the process of verifying that user input is correct, complete, and acceptable before processing.

Goals of Validation

🚫 Prevent invalid dataStop bad or harmful input from being submitted.
😊 Improve user experienceGive immediate, helpful feedback to users.
🛡 Reduce server errorsClean data before it reaches the backend.
📐 Enforce input rulesEnsure fields follow required formats.

⚙️ The Event Object & event.preventDefault()

What Happens During an Event

  • Browser detects the action
  • Browser automatically creates an event object
  • Event object is passed to the JavaScript handler
element.addEventListener("click", function(event) {
  console.log(event); // inspect the event object
});

What the Event Object Stores

Which element triggered ite.g., the button that was clicked
Type of evente.g., "click", "submit", "keydown"
Mouse position / key pressedcoordinates or key code
Default behaviorwhat the browser would normally do

Preventing Form Submission

JavaScript controls submission using event.preventDefault(). This is called when validation fails to stop invalid data from being sent.

Without preventDefault()

Form submits immediately — no validation possible.

With preventDefault()

JavaScript validates first. Submission is blocked if invalid.

form.addEventListener("submit", function(event) {
  event.preventDefault(); // Stop form submission
});
Browser Default Actions (what preventDefault stops):
  • Form submit → reload page & send request
  • Link click → navigate to URL
  • Right-click → open context menu
  • Key press → insert character
Key definitions to remember:
Event = Something that happens  |  Event Object = Information about what happened  |  preventDefault() = Stop browser's automatic response

Example

HTML

<form id="myForm">
  <input type="text"
    id="username"
    placeholder="Enter Name">
  <button type="submit">
    Submit
  </button>
</form>

JavaScript

document.getElementById("myForm")
  .addEventListener("submit",
    function(event) {
      event.preventDefault();
      // stop submission
      alert("Form submitted!");
  });

🔲 Validating Field Emptiness

There are multiple methods to check if a field is empty. Each has tradeoffs.

Method 1 — HTML required Attribute

The simplest approach. The browser enforces it automatically before submission.

<form>
  <input type="text" placeholder="Enter Name" required>
  <button type="submit">Submit</button>
</form>

Browser shows "Please fill in this field." if empty.

Method 2 — JavaScript Empty String Check

Manually checks if the value equals an empty string.

let name = document.getElementById("name");

if (name.value === "") {
    alert("Name cannot be empty");
}
⚠️ Problem: A value of "  " (only spaces) will pass this check.
Method 3 — Trimmed Empty Check ⭐ Best Practice

Uses .trim() to remove leading/trailing spaces before checking.

let name = document.getElementById("name");

if (name.value.trim() === "") {
    alert("Name is required");
}
Why better? Removes leading/trailing spaces and correctly detects logically empty input.
Method 4 — Length-Based Validation

Checks the .length property. Can be combined with .trim().

if (name.value.length === 0) {
    alert("Field empty");
}

// Combined with trim (recommended):
if (name.value.trim().length === 0) {
    alert("Field empty");
}

🔍 Pattern Validation using RegEx

What is test()?

test() is a method used with Regular Expressions (RegEx). It checks whether a value matches a pattern.

// Syntax:
regexPattern.test(value)

// Returns:
// true  → value matches the pattern
// false → value does NOT match the pattern

let pattern = /^[0-9]+$/;
pattern.test("123");   // true
pattern.test("abc");   // false
pattern.test("12ab");  // false
Numbers Only Validation

HTML Approach

<input type="text"
  pattern="[0-9]+"
  required>

JavaScript Approach

let value = input.value.trim();
let numbersOnly = /^[0-9]+$/;

if (!numbersOnly.test(value)) {
    alert("Only numbers allowed");
}
Characters Only Validation

HTML Approach

<input type="text"
  pattern="[A-Za-z]+"
  required>

JavaScript Approach

let charsOnly = /^[A-Za-z]+$/;

if (!charsOnly.test(value)) {
    alert("Only letters allowed");
}
Alphanumeric Validation (Letters + Numbers)

HTML Approach

<input type="text"
  pattern="[A-Za-z0-9]+"
  required>

JavaScript Approach

let alphaNum = /^[A-Za-z0-9]+$/;

if (!alphaNum.test(value)) {
    alert("Only letters and numbers");
}

🧪 Practice Question 2 — Student Details Form

Task: Design a Student Details Entry form with Student Name, Contact Number, Index Number, and Submit. Validate: Name → letters only, Contact → numbers only, Index → alphanumeric only.

HTML (index.html)

<form id="studentForm">
  <label>Student Name:</label><br>
  <input type="text" id="name" required><br><br>

  <label>Contact Number:</label><br>
  <input type="text" id="contact" required><br><br>

  <label>Index Number:</label><br>
  <input type="text" id="indexno" required><br><br>

  <button type="submit">Submit</button>
</form>
<script src="validate.js"></script>

JavaScript (validate.js)

document.getElementById("studentForm")
  .addEventListener("submit", function(event) {

    event.preventDefault(); // stop submission immediately

    const name    = document.getElementById("name").value.trim();
    const contact = document.getElementById("contact").value.trim();
    const indexno = document.getElementById("indexno").value.trim();

    const namePattern    = /^[A-Za-z\s]+$/;
    const contactPattern = /^[0-9]+$/;
    const indexPattern   = /^[A-Za-z0-9]+$/;

    if (!namePattern.test(name)) {
        alert("Student Name must contain characters only.");
        return;
    }
    if (!contactPattern.test(contact)) {
        alert("Contact Number must contain numbers only.");
        return;
    }
    if (!indexPattern.test(indexno)) {
        alert("Index Number must contain letters and numbers only.");
        return;
    }

    alert("All details are valid! Form will submit.");
    this.submit(); // manually submit after validation passes
});
