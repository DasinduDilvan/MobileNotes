📘 Lecture 13 — Introduction to Laravel

What is Laravel?

Definition Laravel is an open-source PHP framework designed for building modern, elegant web applications with clean, readable syntax.
Creator Created by Taylor Otwell. It follows the MVC (Model-View-Controller) architectural pattern.
Core Goal Make PHP development faster, easier, and more secure — without sacrificing developer experience.

Why Laravel?

  • Clean and simple syntax — Readable PHP code that is easy to write and maintain.
  • Built-in authentication — User login and registration systems are ready out of the box.
  • Powerful routing system — Define how URLs map to your application's logic easily.
  • Blade templating engine — A lightweight yet powerful engine for writing HTML views with dynamic data.
  • Strong security features — Built-in protection against common web vulnerabilities (e.g., CSRF, SQL injection).

MVC Architecture

Laravel follows the MVC pattern, which separates an application into three interconnected parts:

Model Handles Data
→
View User Interface
→
Controller Application Logic
🗄️ Model Responsible for interacting with the database. It retrieves, stores, updates, and deletes data.
🖥️ View The HTML/Blade templates the user sees. It only handles presentation — no business logic.
⚙️ Controller The brain. It receives user requests, uses Models to get data, and passes that data to Views.

Key Features of Laravel

Eloquent ORM A simple, expressive way to interact with your database using PHP objects instead of raw SQL queries.
Artisan CLI A built-in command-line tool. Used to generate code, run migrations, and manage the application from your terminal.
Middleware Filters that run between a request and your application. Example: checking if a user is logged in before allowing access.
Routing System Defines how HTTP requests (URLs) are directed to the correct controller or function.
RESTful Support Laravel makes it easy to build REST APIs, which follow standard HTTP verbs (GET, POST, PUT, DELETE).

Where is Laravel Used?

🛒 E-commerce Websites Online shops with product listings, carts, and payment systems.
🏫 School Management Systems Student records, attendance, grades, and portals.
🏢 Company Portals Internal tools, HR systems, dashboards.
🔌 APIs & Web Services Backend services that power mobile apps or third-party integrations.
✅ Summary: Laravel makes PHP development faster, easier, and more secure — making it one of the most popular PHP frameworks in the world.
📗 Lecture 13 — Routing in Laravel

What is Routing?

Core Concept: Routing is the process of accepting a request (a URL + HTTP Method) and directing it to the appropriate code — either a Controller or a Closure (an inline function).

Think of it like this:

👤 User types a URL in the browser
URL Request
→
Routerweb.php
→
Controller
or ClosureYour logic runs
→
ResponsePage / Data

🚦 The router is the "traffic controller" of your Laravel application.

Maps URLs to logic Decides which function or controller handles each URL.
Filters via Middleware Can block or allow requests (e.g., only logged-in users).
Clean URL structure Provides SEO-friendly, readable URLs for your app.
📁 Where do routes live? All web routes are defined in routes/web.php inside your Laravel project.

GET vs. POST — The Two Most Common HTTP Methods

These are the two most important HTTP methods you'll use when building web apps:

Feature GET POST
Purpose Requesting / viewing data Sending / submitting data
Where is data? Visible in the URL (e.g., ?name=John) Hidden — sent in the request body
Changes Database? ❌ Should NOT change data ✅ Yes — creates / updates records
CSRF Required? No ✅ Yes — always add @csrf
Example use Viewing a page, searching Submitting a form, logging in

Setting Up a Basic GET Route

A GET route displays a page when the user visits a URL. Here's how to set one up:

  1. Create a Blade view file named contact.blade.php inside the resources/views/ folder.
  2. Open routes/web.php and add the following route:
// routes/web.php
Route::get('/contact', function () {
    return view('contact'); // returns the contact.blade.php view
});
  1. Open your browser and visit yoursite.test/contact — the contact page will appear.
How it works: When a user visits /contact, Laravel matches the URL to this route and returns the contact.blade.php view file.

Route Parameters — Dynamic URLs

Sometimes you need to capture a value from the URL itself (like a user ID, or a name). These are called route parameters.

Syntax: Wrap the parameter name in curly braces: {parameterName}

Required Parameters

These must be present in the URL. If missing, Laravel shows a 404 error.

// Captures the {id} from the URL
Route::get('/user/{id}', function ($id) {
    return 'User ID is: ' . $id;
});

Example: Visiting /user/42 will display "User ID is: 42"

Multiple Parameters

You can capture more than one value from the URL:

// Captures two values from the URL
Route::get('/contact/{firstname}/{lastname}', function ($firstname, $lastname) {
    return $firstname . ' ' . $lastname;
});

Example: Visiting /contact/John/Doe displays "John Doe"

Named Routes

A named route gives a route a unique nickname. Instead of hardcoding URLs in your views, you use the route's name — so if the URL ever changes, you only update it in one place.

Step 1 — Define the named route in web.php

Route::get('/test', function () {
    return 'This is the test page';
})->name('testpage');

Step 2 — Use the name in your Blade view

<!-- home.blade.php -->
<a href="{{ route('testpage') }}">
    Go to Test Page
</a>
✅ Why use named routes? If you change /test to /testing, you only update web.php. Your Blade views with route('testpage') continue to work automatically — no need to find and update every link.

Route Groups & Organization

When you have many routes that share something in common (like all needing a logged-in user), you can group them together instead of repeating the same settings on each route.

Prefix Group routes under a shared URL segment, e.g., all admin routes starting with /admin.
Middleware Apply a filter to a whole group of routes at once, e.g., only allow logged-in users.

Example — Protecting routes with auth middleware:

// Only logged-in users can access /dashboard and /account
Route::middleware(['auth'])->group(function () {
    Route::get('/dashboard', ...);
    Route::get('/account', ...);
});
Think of it this way: The auth middleware is like a security guard at the door. Only users who are logged in get past. Any route inside the group automatically gets checked.

Setting Up a POST Route (Forms)

A POST route is used when a user submits a form. Data travels securely in the request body (not in the URL).

Step 1 — Create the HTML form (home.blade.php)

<form action="{{ route('formsubmitted') }}" method="POST">
    @csrf  <!-- CSRF token — required for all POST forms -->

    <label>Name:</label>
    <input type="text" name="user_name" required>

    <label>Email:</label>
    <input type="email" name="user_email" required>

    <button type="submit">Send Message</button>
</form>

Step 2 — Handle the form data in web.php

Route::post('/formsubmitted', function (Request $request) {
    $fname = $request->input('user_name');
    $email = $request->input('user_email');

    return "Your name is {$fname} and your email is {$email}";
})->name('formsubmitted');
How $request->input() works: It reads a value from the form by its name attribute. input('user_name') reads the value of the field with name="user_name".

CSRF Protection

⚠️ What is CSRF? Cross-Site Request Forgery — a type of attack where a malicious website tricks a user's browser into submitting a form to your site without their knowledge.
The Problem A hacker's website could secretly submit a form to your app using a logged-in user's browser session.
Laravel's Solution Laravel automatically generates a secret CSRF token for each session and verifies it on every POST, PUT, PATCH, and DELETE request.
Your Job Always include @csrf inside every HTML form. This inserts a hidden token field Laravel can verify. 
<!-- Always add @csrf inside your form tags -->
<form method="POST" action="/submit">
    @csrf
    ...
</form>
✅ Rule: Every form with method="POST" in Laravel must include @csrf — otherwise Laravel will reject the request with a 419 error.

Inserting Images in Laravel (using asset())

In Laravel, public files like images are stored in the public/ folder. To reference them in Blade views, use the asset() helper — it generates the correct full URL automatically.

  1. Inside the public/ folder of your Laravel project, create a folder called images/.
  2. Place your image file inside that folder (e.g., hi.avif).
  3. In your Blade view, use the asset() helper to reference the image:
<!-- Blade view -->
<img src="{{ asset('images/hi.avif') }}" alt="My Image">
Why use asset()? It always generates the correct full URL regardless of your server setup — e.g., http://yoursite.test/images/hi.avif. Hardcoding paths like /images/hi.avif can break in some environments.

Building CRUD Operations in Laravel

CRUD stands for Create, Read, Update, Delete — the four basic operations of any data-driven web app.

🔧 Laravel makes CRUD easy using a Model, Migration, Controller, and a Resource Route. You generate most of the scaffolding with a single Artisan command.

Step 1 — Generate Model, Migration & Controller

# Creates Model + Migration + Controller (with resource methods)
php artisan make:model Post --mcr

This creates three files for you: app/Models/Post.php, a migration file in database/migrations/, and app/Http/Controllers/PostController.php.

Step 2 — Update the Migration File

Open the generated migration file and define your table columns inside the up() method:

public function up(): void
{
    Schema::create('posts', function (Blueprint $table) {
        $table->id();                  // auto-increment primary key
        $table->string('title');      // VARCHAR column
        $table->text('body');         // TEXT column
        $table->timestamps();         // created_at and updated_at
    });
}

Step 3 — Run the Migration

# Creates the 'posts' table in your database
php artisan migrate
Rollback last migration php artisan migrate:rollback
Rollback specific file php artisan migrate:rollback --path=database/migrations/filename
Re-run all migrations php artisan migrate:refresh

Step 4 — Update the Model (Mass Assignment)

Open app/Models/Post.php and tell Laravel which fields are allowed to be filled:

protected $fillable = ['title', 'body'];
Why? The $fillable property protects you from accidentally saving unwanted data. Only fields listed here can be mass-assigned (e.g., via Post::create($data)).

Step 5 — Create a Resource Route

In routes/web.php, one line creates all 7 CRUD routes automatically:

Route::resource('posts', PostController::class);
✅ This single line generates all the routes you need: GET /posts, GET /posts/create, POST /posts, GET /posts/{id}, GET /posts/{id}/edit, PUT /posts/{id}, DELETE /posts/{id}.

The 7 Controller Functions (Resource Controller)

When you open PostController.php, you'll find these 7 methods already scaffolded. Each one handles a specific CRUD action:

index()

Shows a list of all posts.

create()

Shows the form to create a new post.

store()

Saves the new post to the database.

show()

Displays a single post by ID.

edit()

Shows the form to edit an existing post.

update()

Saves the edited post to the database.

destroy()

Deletes a post from the database.

index() — List all posts

public function index()
{
    $posts = Post::all();  // Get all records from 'posts' table
    return view('posts.index', compact('posts')); // Pass to view
}

→ Create folder resources/views/posts/ and file index.blade.php

create() — Show the create form

public function create()
{
    return view('posts.create');
}

→ Create file resources/views/posts/create.blade.php

store() — Save new post to database

public function store(Request $request)
{
    $validated = $request->validate([
        'title' => 'required',  // title must not be empty
        'body'  => 'required',
    ]);
    Post::create($validated);  // Insert into database
    return redirect()->route('posts.index')
        ->with('success', 'Post created successfully');
}

edit() — Show the edit form

public function edit($id)
{
    $post = Post::findOrFail($id);  // Find post or show 404
    return view('posts.edit', compact('post'));
}

→ Create file resources/views/posts/edit.blade.php

update() — Save edited post

public function update(Request $request, $id)
{
    $validated = $request->validate([
        'title' => 'required',
        'body'  => 'required',
    ]);
    $post = Post::findOrFail($id);    // Find the post
    $post->update($validated);          // Update with new data
    return redirect()->route('posts.index')
        ->with('success', 'Post updated successfully');
}

destroy() — Delete a post

public function destroy($id)
{
    $post = Post::findOrFail($id);  // Find the post
    $post->delete();                  // Delete it
    return redirect()->route('posts.index')
        ->with('success', 'Post deleted successfully');
}
